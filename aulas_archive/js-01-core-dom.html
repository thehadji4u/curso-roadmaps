// motivação

Por que isso importa?

JavaScript é a única linguagem nativa dos navegadores. Não é possível escapar dela: todo app web moderno — React, Vue, Angular, Next.js — compila para JavaScript. Entender o core da linguagem é a diferença entre copiar código do Stack Overflow às cegas e realmente saber o que está acontecendo.

Nesta aula vamos cobrir os pilares que sustentam tudo: como JS gerencia tipos e variáveis, como funções funcionam de verdade (incluindo hoisting e closures), e como o DOM permite que código JS manipule o HTML da página em tempo real.

💼 Por que um engenheiro sênior usa isso hoje
Sêniors não "só usam React". Eles entendem que React é uma camada sobre JS puro. Quando algo quebra — um event listener não dispara, uma variável tem valor inesperado, um loop se comporta estranhamente — o debug exige conhecimento profundo do core da linguagem. Hoisting, coerção de tipos, propagação de eventos: esses são os "fantasmas" que assombram bugs de produção.
// analogia

Antes do código: uma analogia

🌍 Mundo Real → Tecnologia
Imagine uma peça de teatro. O HTML é o roteiro e o cenário — define o que existe, onde os objetos estão. O CSS é o figurino e a iluminação — define a aparência. O JavaScript é o ator e o diretor ao mesmo tempo: pode ler e alterar o roteiro ao vivo, mover objetos do cenário, responder às reações do público (eventos), e até criar novos atores em cena (criar elementos DOM dinamicamente).

O DOM (Document Object Model) é o mapa do palco: uma representação em árvore de todos os elementos da peça, que o JS pode navegar e modificar durante a apresentação.
🗺️ Mapa do palco → DOM (Document Object Model) 👋 Reação do público (aplausos) → Evento (click, input, scroll) 🎭 Ator que muda o cenário → JavaScript manipulando o DOM 📋 Roteiro guardado nos bastidores → Variáveis no escopo fechado (closure)

Tipos & Variáveis

JavaScript é uma linguagem de tipagem dinâmica: o tipo de uma variável é determinado em runtime, não na declaração. Isso traz flexibilidade e também armadilhas.

Existem 8 tipos primitivos: string, number, bigint, boolean, undefined, null, symbol — e o tipo object (que inclui arrays, funções e objetos literais).

// tipos e declarações
// ─── As 3 formas de declarar variáveis ───────────────────
var  legado = 'evitar em código novo';  // function-scoped, sofre hoisting
let  mutavel = 42;                      // block-scoped, reatribuível
const fixo = 'não pode ser reatribuído'; // block-scoped, obrigatório na maioria dos casos

// ─── Tipos primitivos ─────────────────────────────────────
let texto    = "Olá mundo";          // string
let numero   = 3.14;                 // number (int e float são o mesmo tipo!)
let enorme   = 9007199254740993n;    // bigint (sufixo n)
let ativo    = true;                 // boolean
let vazio    = null;                 // null (ausência intencional de valor)
let naoInit;                         // undefined (variável sem valor)

// ─── typeof ───────────────────────────────────────────────
console.log(typeof "JS");   // "string"
console.log(typeof 42);     // "number"
console.log(typeof null);   // "object" ← bug histórico do JS! null NÃO é objeto
console.log(typeof []);     // "object" ← arrays também são objects

// ─── Coerção de tipos (armadilha clássica!) ───────────────
console.log("5" + 3);    // "53"  (+ com string concatena)
console.log("5" - 3);    // 2     (- converte string para number)
console.log(true + 1);   // 2     (true vira 1)
console.log(null + 1);   // 1     (null vira 0)

// ─── Comparação: == vs === ─────────────────────────────────
console.log(5 == "5");   // true  (== faz coerção — EVITE)
console.log(5 === "5");  // false (=== sem coerção — USE SEMPRE)
⚠️ Regra de ouro
Sempre use const por padrão. Use let apenas quando precisar reatribuir. Nunca use var em código novo — seu escopo de função e hoisting causam bugs sutis e difíceis de rastrear.

var vs let vs const

Declaração Escopo Hoisting Reatribuível Usar quando
var Função Inicializado como undefined ✅ Sim Nunca em código novo
let Bloco {} Declarado mas não init (TDZ) ✅ Sim Contadores, acumuladores
const Bloco {} Declarado mas não init (TDZ) ❌ Não Padrão — use sempre
💡 TDZ — Temporal Dead Zone
Com let e const, a declaração existe no escopo mas não pode ser acessada antes da linha de declaração. Isso é chamado de Temporal Dead Zone e gera um ReferenceError. Com var, a variável existe (como undefined) desde o topo da função — o que é confuso.

Funções — muito além do def

Em JavaScript, funções são cidadãos de primeira classe: podem ser atribuídas a variáveis, passadas como argumentos e retornadas de outras funções. Isso habilita padrões funcionais poderosos como callbacks, higher-order functions e closures.

Declaration

Function Declaration

Sofre hoisting completo — pode ser chamada antes da definição no código.

Expression

Arrow Function

Sintaxe concisa. Não tem próprio this — usa o do contexto externo.

Closure

Closures

Função que "lembra" as variáveis do escopo onde foi criada, mesmo após esse escopo terminar.

HOF

Higher-order Functions

Funções que recebem ou retornam outras funções. Base de map, filter, reduce.

// todos os tipos de função
// ─── 1. Function Declaration (hoisting completo) ────────────
saudar("Ana");   // funciona! hoisting eleva a função inteira

function saudar(nome) {
  return `Olá, ${nome}!`;
}

// ─── 2. Function Expression (não sofre hoisting) ──────────
// saudarEx("Ana");  // ❌ TypeError: saudarEx is not a function

const saudarEx = function(nome) {
  return `Olá, ${nome}!`;
};

// ─── 3. Arrow Function ────────────────────────────────────
const dobrar = (n) => n * 2;                     // implícito: return
const somar  = (a, b) => a + b;
const info   = nome => ({ name: nome, ts: Date.now() }); // objeto → parênteses

// ─── 4. Closures na prática ───────────────────────────────
function criarContador(inicio = 0) {
  let count = inicio;            // esta variável "fica presa" na closure
  return {
    incrementar: () => ++count,
    resetar: () => { count = inicio; },
    valor: () => count,
  };
}

const c = criarContador(10);
console.log(c.incrementar()); // 11
console.log(c.incrementar()); // 12
console.log(c.resetar());     // undefined
console.log(c.valor());       // 10

// ─── 5. Higher-order functions ─────────────────────────────
const numeros = [1, 2, 3, 4, 5];

const dobrados    = numeros.map(n => n * 2);       // [2, 4, 6, 8, 10]
const pares       = numeros.filter(n => n % 2 === 0); // [2, 4]
const soma        = numeros.reduce((acc, n) => acc + n, 0); // 15

Escopo & Hoisting

Escopo define onde uma variável ou função pode ser acessada. JavaScript tem escopo global, de função e de bloco (introduzido com let/const no ES6).

Hoisting é o comportamento do motor JS de "elevar" declarações para o topo do escopo antes de executar o código. function declaration sobe completa. var sobe mas inicializado como undefined. let/const entram na TDZ.

// escopo e a armadilha clássica do var em loop
// ─── Problema com var em loops (escopo de função) ────────────
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}
// Imprime: 3, 3, 3 ← porque var é compartilhado!

// ─── Solução com let (escopo de bloco) ───────────────────────
for (let j = 0; j < 3; j++) {
  setTimeout(() => console.log(j), 100);
}
// Imprime: 0, 1, 2 ✅ cada iteração tem seu próprio j

// ─── Escopo léxico e closure ──────────────────────────────────
function externa() {
  const mensagem = "Olá do escopo externo";

  function interna() {
    console.log(mensagem); // acessa via closure
  }

  return interna;
}

const fn = externa();
fn(); // "Olá do escopo externo" — mesmo após externa() ter terminado!

O DOM — Document Object Model

O DOM é a representação em árvore do HTML que o browser cria na memória. Cada tag HTML vira um nó (Node). O JavaScript pode navegar, ler e modificar essa árvore em tempo real, fazendo a página "ganhar vida".

// selecionar, ler e modificar o DOM
// ─── Seleção de elementos ────────────────────────────────────
const titulo = document.getElementById('titulo');
const btn    = document.querySelector('#btn');           // CSS selector
const itens  = document.querySelectorAll('.lista li');   // NodeList (todos)
const inputs = document.getElementsByTagName('input');   // HTMLCollection

// ─── Lendo e modificando conteúdo ────────────────────────────
console.log(titulo.textContent);       // texto puro (sem HTML)
console.log(titulo.innerHTML);         // texto com tags HTML
titulo.textContent = 'Novo Título';

titulo.style.fontSize  = '2rem';

const novoItem = document.createElement('li');

novoItem.remove(); // ES6 — simples e direto

Eventos — a linguagem da interação

Eventos são o mecanismo pelo qual o browser comunica interações do usuário (cliques, teclado, formulários) ao JavaScript. O modelo de eventos do DOM funciona em três fases: captura, alvo e bubbling.

// addEventListener — a forma correta
// ─── addEventListener (prefira sempre sobre onclick = ...) ───
const btn = document.querySelector('#btn');

btn.addEventListener('click', function(event) {
  console.log('Clicado!', event.target); // event.target = elemento clicado
});

// Arrow function — NÃO tem 'this' próprio (cuidado se precisar de this)
btn.addEventListener('click', (e) => {
  e.preventDefault();  // evita comportamento padrão (ex: form submit)
  e.stopPropagation(); // impede o evento subir na árvore (bubbling)
});

// ─── Eventos de teclado ───────────────────────────────────────
document.addEventListener('keydown', (e) => {
  if (e.key === 'Enter') console.log('Enter pressionado!');
  if (e.ctrlKey && e.key === 's') {
    e.preventDefault();
    salvar();
  }
});

// ─── Eventos de formulário ────────────────────────────────────
const form  = document.querySelector('form');
const input = document.querySelector('#nome');

form.addEventListener('submit', (e) => {
  e.preventDefault(); // impede reload da página
  const valor = input.value.trim();
  if (!valor) return console.error('Campo obrigatório!');
  processarDados(valor);
});

input.addEventListener('input', (e) => {
  console.log('Digitando:', e.target.value);
});

// ─── Event Delegation — técnica sênior ───────────────────────
// Em vez de adicionar listener em cada 
  • , adicionar no pai
const lista = document.querySelector('.lista');

lista.addEventListener('click', (e) => {
  if (e.target.tagName === 'LI') {
    e.target.classList.toggle('selecionado');
    console.log('Item selecionado:', e.target.textContent);
  }
});
// Funciona inclusive para elementos adicionados dinamicamente depois!
    • 🔥 Pro tip: Event Delegation
    Event Delegation é um dos padrões mais usados em produção. Em vez de adicionar event listeners em múltiplos filhos, adicione um só no elemento pai e use event.target para identificar qual filho foi clicado. Isso economiza memória, é mais performático e funciona para elementos criados dinamicamente.

    // ─── Estado da aplicação ──────────────────────────────────────
const state = {
  tasks: [],
  nextId: 1,
};

// ─── Seleção de elementos ──────────────────────────────────────
const input    = document.getElementById('task-input');
const addBtn   = document.getElementById('add-btn');
const taskList = document.getElementById('task-list');

// ─── Renderizar lista ──────────────────────────────────────────
function render() {
  taskList.innerHTML = ''; // limpa a lista

  state.tasks.forEach(task => {
    const li = document.createElement('li');
    li.className = `task ${task.done ? 'done' : ''}`;
    li.dataset.id = task.id; // guarda o id no elemento

    li.innerHTML = `
      
      ${task.text}
      
    `;

    taskList.appendChild(li);
  });
}

// ─── Adicionar tarefa ──────────────────────────────────────────
function addTask() {
  const text = input.value.trim();
  if (!text) return;

  state.tasks.push({ id: state.nextId++, text, done: false });
  input.value = '';
  render();
}

// ─── Event Delegation na lista ─────────────────────────────────
taskList.addEventListener('click', (e) => {
  const li = e.target.closest('li');
  if (!li) return;

  const id = Number(li.dataset.id);
  const task = state.tasks.find(t => t.id === id);

  if (e.target.matches('input[type="checkbox"]')) {
    task.done = !task.done;
    render();
  }

  if (e.target.matches('.del-btn')) {
    state.tasks = state.tasks.filter(t => t.id !== id);
    render();
  }
});

// ─── Event listeners ──────────────────────────────────────────
addBtn.addEventListener('click', addTask);
input.addEventListener('keydown', (e) => {
  if (e.key === 'Enter') addTask();
});

    ⚠️ Armadilhas Comuns

    1. Usar == ao invés de ===

    O operador == faz coerção de tipo e produz resultados surpreendentes. 0 == false é true. "" == false é true. Sempre use === e !==.

    2. var dentro de loops com setTimeout

    Clássico: usar var i em um loop e criar funções que referenciam i. Todas as funções compartilham a mesma variável. A solução é usar let.

    3. Usar innerHTML com dados do usuário

    Se você inserir input do usuário via innerHTML, abre brecha para XSS (Cross-site Scripting). Use textContent para texto puro ou sanitize os dados antes.

    4. Adicionar event listeners em loop sem delegation

    Adicionar addEventListener em cada item de uma lista pode criar centenas de listeners desnecessários. Use Event Delegation — um único listener no pai.

    5. typeof null === "object"

    Bug histórico do JavaScript que nunca foi corrigido por compatibilidade. Para verificar se algo é realmente null, use value === null, não typeof.

    Termos desta Aula

    DOM (Document Object Model)
    Representação em árvore do HTML na memória do browser, que pode ser lida e modificada via JavaScript.
    Hoisting
    Comportamento do motor JS de processar declarações antes de executar o código, "elevando-as" ao topo do escopo.
    Closure
    Função que "lembra" e acessa variáveis do escopo externo onde foi criada, mesmo após esse escopo ter terminado.
    Event Bubbling
    Propagação de um evento do elemento-alvo para os ancestrais na árvore DOM, até o document.
    Event Delegation
    Padrão que usa um único event listener no elemento pai para gerenciar eventos de múltiplos filhos via event.target.
    TDZ (Temporal Dead Zone)
    Zona onde variáveis let/const existem no escopo mas não podem ser acessadas — entre o início do bloco e a linha de declaração.
