// motivação

Por que isso importa?

Objetos e arrays são as estruturas de dados mais usadas em JavaScript. Quase tudo no ecossistema JS — respostas de API, estado de componentes React, configurações de build — é um objeto ou array. Dominar como navegar, transformar e compor essas estruturas é o que separa quem escreve código funcional de quem escreve código elegante e sustentável.

O ES6 (lançado em 2015) e as versões subsequentes trouxeram açúcares sintáticos que revolucionaram a forma de escrever JS moderno: destructuring, spread operator, template literals, classes, modules e muito mais. Todo codebase profissional usa esses recursos no dia a dia.

💼 Por que um engenheiro sênior usa isso hoje
Ao revisar um PR, sêniors reconhecem imediatamente código "pré-ES6" — cheio de var, concatenação de strings e loops for manuais — como sinal de inexperiência. Além de mais legível, o código moderno é menos propenso a bugs. map/filter/reduce, destructuring e spread são a linguagem comum dos times de alto nível; sem eles, você não vai entender nem escrever React de forma fluente.

Antes do código: uma analogia

🌍 Mundo Real → Tecnologia
Imagine uma ficha de cadastro de funcionário. A ficha tem campos nomeados (nome, cargo, salário) — isso é um objeto. A lista de todos os funcionários da empresa é um array de objetos.

O destructuring é como arrancar só os campos que você precisa da ficha, em vez de carregar o formulário inteiro. O spread operator é como fotocopiar uma ficha e adicionar novos campos à cópia sem alterar o original. Os modules são como departamentos da empresa: cada um tem suas próprias responsabilidades e expõe só o que precisa para os outros.
📋 Ficha de funcionário → Objeto JavaScript { chave: valor } 📂 Lista de funcionários → Array de objetos [{}, {}, {}] ✂️ Arrancar só nome e cargo → Destructuring const { nome, cargo } = func 🗂️ Departamentos isolados → ES Modules (import/export)

Objetos — a estrutura universal do JS

Um objeto em JavaScript é uma coleção de pares chave-valor. Chaves são strings (ou Symbols), valores podem ser qualquer tipo — incluindo funções (que então se chamam métodos) e outros objetos (objetos aninhados).

// criação, acesso, modificação e métodos úteis
// ─── Criação de objetos ──────────────────────────────────────
const usuario = {
  nome: 'Ana Silva',
  idade: 28,
  cargo: 'Engenheira Sênior',
  ativo: true,
  enderecos: { cidade: 'São Paulo', estado: 'SP' },    // aninhado
  saudar() {                                            // método (ES6 shorthand)
    return `Olá, sou ${this.nome}`;
  },
};

// ─── Acesso a propriedades ────────────────────────────────────
console.log(usuario.nome);            // dot notation
console.log(usuario['cargo']);        // bracket notation (útil com variáveis)
console.log(usuario.enderecos.cidade);// encadeamento
console.log(usuario.telefone);        // undefined (sem erro)
console.log(usuario.telefone ?? 'N/A');  // nullish coalescing — default para null/undefined

// ─── Modificação e adição ─────────────────────────────────────
usuario.idade = 29;                   // modifica
usuario.email = 'ana@dev.com';        // adiciona nova chave
delete usuario.ativo;                 // remove

// ─── Métodos essenciais de Object ────────────────────────────
const chaves   = Object.keys(usuario);    // ['nome', 'idade', ...]
const valores  = Object.values(usuario);  // ['Ana Silva', 29, ...]
const entradas = Object.entries(usuario); // [['nome','Ana'], ['idade',29], ...]

// Iterar com entries é ouro:
for (const [chave, valor] of Object.entries(usuario)) {
  console.log(`${chave}: ${valor}`);
}

// ─── Checando existência de chave ─────────────────────────────
console.log('nome' in usuario);               // true
console.log(usuario.hasOwnProperty('nome'));  // true
🔥 Pro tip: Optional Chaining (?.)
Ao acessar objetos aninhados ou propriedades que podem não existir, use obj?.prop?.subprop em vez de verificar cada nível com if. Se qualquer nível for null ou undefined, a expressão retorna undefined silenciosamente, sem jogar TypeError.

console.log(usuario?.enderecos?.bairro); → undefined (sem erro)

Destructuring — extraia o que precisa

Destructuring é uma sintaxe que permite extrair valores de objetos e arrays para variáveis de forma concisa. É um dos recursos ES6 mais usados no dia a dia, especialmente ao trabalhar com parâmetros de função e respostas de API.

❌ Antes do ES6
const nome  = usuario.nome;
const cargo = usuario.cargo;
const cidade = usuario.enderecos.cidade;

// arrays
const primeiro = arr[0];
const segundo  = arr[1];
✅ Com Destructuring
const { nome, cargo } = usuario;
const { enderecos: { cidade } } = usuario;

// arrays
const [primeiro, segundo] = arr;
const [, , terceiro] = arr; // pula posições
// destructuring avançado — renomear, defaults, rest
// ─── Renomear ao desestruturar ─────────────────────────────────
const { nome: nomeCompleto, cargo: funcao = 'Dev' } = usuario;
//      ↑ extrai 'nome' e salva em 'nomeCompleto'
//                              ↑ valor default se a chave não existir

// ─── Destructuring em parâmetros de função ─────────────────────
function exibirUsuario({ nome, cargo, ativo = true }) {
  console.log(`${nome} — ${cargo} (${ativo ? 'ativo' : 'inativo'})`);
}
exibirUsuario(usuario);

// ─── Rest em destructuring ─────────────────────────────────────
const { nome: n, ...restante } = usuario;
// n = 'Ana Silva'
// restante = { cargo, enderecos, ... } (tudo menos nome)

// ─── Array destructuring com swap ──────────────────────────────
let a = 1, b = 2;
[a, b] = [b, a];   // troca de valores sem variável temporária!
console.log(a, b); // 2, 1

// ─── Destructuring com loops ──────────────────────────────────
const usuarios = [
  { nome: 'Ana', cargo: 'Sênior' },
  { nome: 'Bob', cargo: 'Júnior' },
];

for (const { nome, cargo } of usuarios) {
  console.log(`${nome}: ${cargo}`);
}

Spread & Rest — expandir e coletar

O operador ... tem dois comportamentos dependendo do contexto. Como spread, ele expande um array ou objeto. Como rest, ele coleta o que sobrou em uma variável.

// spread e rest na prática
// ─── SPREAD: expandir arrays ───────────────────────────────────
const frutas = ['maçã', 'banana'];
const mais   = ['uva', ...frutas, 'pera'];   // ['uva', 'maçã', 'banana', 'pera']
const copia  = [...frutas];                  // cópia rasa (shallow clone)

Math.max(...[1, 5, 3, 9, 2]);              // spread em chamada de função → 9

// ─── SPREAD: merge e override de objetos ──────────────────────
const base = { cor: 'azul', tamanho: 'M', estoque: 10 };
const override = { tamanho: 'G', desconto: 0.1 };

// Última propriedade vence — padrão para "merge com override"
const produto = { ...base, ...override };
// { cor: 'azul', tamanho: 'G', estoque: 10, desconto: 0.1 }

// Imutabilidade: atualizar objeto sem mutar o original ✅
const usuarioAtualizado = { ...usuario, idade: 30, email: 'novo@dev.com' };

// ─── REST: coletar argumentos ─────────────────────────────────
function somar(...numeros) {            // rest em parâmetro
  return numeros.reduce((a, n) => a + n, 0);
}
console.log(somar(1, 2, 3, 4, 5)); // 15

function log(nivel, ...mensagens) {    // primeiro fixo, resto coletado
  console.log(`[${nivel}]`, ...mensagens);
}
log('INFO', 'Servidor', 'iniciado', 'na porta 3000');

// ─── Clonar vs referência ─────────────────────────────────────
// ⚠️ Spread faz SHALLOW clone (apenas 1 nível de profundidade)
const original = { nome: 'Ana', endereco: { cidade: 'SP' } };
const clone    = { ...original };

clone.nome = 'Bob';                  // ✅ não afeta original
clone.endereco.cidade = 'RJ';       // ❌ AFETA original! (mesmo objeto aninhado)

// Deep clone: use structuredClone() (moderno) ou JSON.parse(JSON.stringify())
const deepClone = structuredClone(original);
⚠️ Shallow vs Deep Clone
O spread { ...obj } e [...arr] fazem shallow clone — copiam apenas o primeiro nível. Objetos aninhados ainda são compartilhados por referência. Para clonar estruturas profundas, use structuredClone(obj) (nativo desde 2022) ou bibliotecas como Lodash's _.cloneDeep().

ES6+ — Sintaxe moderna essencial

Além de destructuring e spread, o ES6+ trouxe vários recursos que aparecem em todo codebase moderno. Vamos cobrir os mais frequentes:

Template Literals

Strings com superpoderes

Interpolação com ${expr}, multi-linha nativa, tagged templates para formatação customizada.

Shorthand Properties

Menos código, mesma intenção

Quando chave e variável têm o mesmo nome: { nome } em vez de { nome: nome }.

Optional Chaining

?. para navegação segura

Evita TypeError ao acessar propriedades em objetos que podem ser null ou undefined.

Nullish Coalescing

?? para defaults precisos

Retorna o lado direito apenas se o esquerdo for null ou undefined (diferente de || que rejeita qualquer falsy).

// ES6+ na prática
// ─── Template Literals ────────────────────────────────────────
const nome = 'Ana';
const cargo = 'Sênior';

// Antes:
const msg1 = 'Olá, ' + nome + '! Você é ' + cargo + '.';

// ES6+:
const msg2 = `Olá, ${nome}! Você é ${cargo}.`;

// Expressões completas dentro:
const preco = 1500;
const label = `Preço: R$ ${(preco * 1.1).toFixed(2)} (com 10% de taxa)`;

// Multi-linha real:
const html = `
  

    
${nome}

    
${cargo}

  

`.trim();

// ─── Shorthand property names ──────────────────────────────────
const x = 10, y = 20;
const ponto = { x, y };              // { x: 10, y: 20 }

// Computed property names:
const campo = 'email';
const obj = { [campo]: 'ana@dev.com' }; // { email: 'ana@dev.com' }

// ─── Optional Chaining ?. ─────────────────────────────────────
const config = { db: { host: 'localhost' } };
console.log(config?.db?.host);        // 'localhost'
console.log(config?.cache?.host);     // undefined (sem erro!)
console.log(config?.metodo?.());      // undefined (sem erro ao chamar método)

// ─── Nullish Coalescing ?? vs || ─────────────────────────────
// O problema com ||:
const porta1 = 0 || 3000;         // 3000 ← ERRADO! 0 é falsy, mas pode ser válido
const porta2 = 0 ?? 3000;         // 0    ← CORRETO! ?? só ativa em null/undefined

const usuario = { nome: '', admin: false };
const nome1 = usuario.nome || 'Anônimo';  // 'Anônimo' ← errado se '' é válido
const nome2 = usuario.nome ?? 'Anônimo'; // '' ← mantém string vazia

// ─── Logical Assignment (ES2021) ─────────────────────────────
let config2 = null;
config2 ??= { theme: 'dark' };    // atribui APENAS se for null/undefined
config2.debug ||= false;          // atribui APENAS se for falsy
config2.level &&= config2.level + 1; // atribui APENAS se for truthy

Prototype Chain & Classes ES6

JavaScript usa herança prototípica: cada objeto tem um prototype (protótipo), e ao acessar uma propriedade, o motor JS sobe a cadeia de protótipos até encontrá-la ou chegar em null. As classes ES6 são açúcar sintático sobre esse mecanismo — não mudam o modelo interno, apenas tornam o código mais legível.

// classes ES6 — sintaxe moderna de OOP em JS
// ─── Classe base ──────────────────────────────────────────────
class Pessoa {
  #cpf;  // campo privado (ES2022 — # = privado de verdade)

  constructor(nome, idade, cpf) {
    this.nome  = nome;
    this.idade = idade;
    this.#cpf  = cpf;
  }

  // Getter (acesso como propriedade)
  get info() {
    return `${this.nome} (${this.idade} anos)`;
  }

  // Método de instância
  aniversario() {
    this.idade++;
    return this;  // permite encadeamento: pessoa.aniversario().aniversario()
  }

  // Método estático (chamado na classe, não na instância)
  static comparar(a, b) {
    return a.idade - b.idade;
  }

  toString() {
    return this.info;
  }
}

// ─── Herança ──────────────────────────────────────────────────
class Funcionario extends Pessoa {
  #salario;

  constructor(nome, idade, cpf, cargo, salario) {
    super(nome, idade, cpf);  // obrigatório chamar super() antes de usar this
    this.cargo   = cargo;
    this.#salario = salario;
  }

  get salario() { return this.#salario; }

  // Override do método pai
  get info() {
    return `${super.info} — ${this.cargo}`;  // super acessa o método do pai
  }

  promover(novoCargo, aumento) {
    this.cargo     = novoCargo;
    this.#salario *= (1 + aumento);
    return this;
  }
}

// ─── Uso ──────────────────────────────────────────────────────
const ana = new Funcionario('Ana', 28, '111.222.333-44', 'Desenvolvedora', 8000);
console.log(ana.info);            // 'Ana (28 anos) — Desenvolvedora'
ana.promover('Sênior', 0.2).aniversario();
console.log(ana.salario);         // 9600
console.log(ana instanceof Funcionario); // true
console.log(ana instanceof Pessoa);      // true

// ─── Verificar prototype ─────────────────────────────────────
console.log(Object.getPrototypeOf(ana) === Funcionario.prototype); // true

ES Modules — import / export

Modules são a forma nativa de JavaScript de organizar código em arquivos separados, com escopo isolado. Cada módulo é um arquivo; o que não for exportado é privado. Isso é a base de todo projeto Node.js e frontend moderno.

// math.js — módulo que exporta utilidades
// ─── Exportações nomeadas (named exports) ─────────────────────
export const PI = 3.14159;

export function soma(a, b) { return a + b; }
export function media(...nums) { return nums.reduce((a, n) => a + n, 0) / nums.length; }

// ─── Export default (um por arquivo) ─────────────────────────
export default class Calculadora {
  historico = [];  // class field (ES2022)

  calcular(op, a, b) {
    const resultado = op === '+' ? a + b : op === '-' ? a - b
                    : op === '*' ? a * b : a / b;
    this.historico.push({ op, a, b, resultado });
    return resultado;
  }
}
// app.js — importando o módulo
// ─── Import named ────────────────────────────────────────────
import { PI, soma, media } from './math.js';

// ─── Import default ──────────────────────────────────────────
import Calculadora from './math.js';

// ─── Ambos juntos ────────────────────────────────────────────
import Calculadora, { PI, soma } from './math.js';

// ─── Renomear ao importar ─────────────────────────────────────
import { soma as somar, media as calcMedia } from './math.js';

// ─── Importar tudo como namespace ─────────────────────────────
import * as MathUtils from './math.js';
console.log(MathUtils.PI);    // 3.14159
console.log(MathUtils.soma(2, 3)); // 5

// ─── Dynamic import (lazy loading) ──────────────────────────
// Carrega o módulo somente quando necessário (retorna Promise)
const { soma: s } = await import('./math.js');
console.log(s(10, 20)); // 30
💡 CommonJS vs ES Modules
Node.js historicamente usava require() / module.exports (CommonJS). ES Modules (import/export) são o padrão moderno e nativo do browser. Em projetos Node modernos, use "type": "module" no package.json ou extensão .mjs. Bundlers como Vite e esbuild lidam com a compatibilidade automaticamente.

// ─── produto.js ──────────────────────────────────────────────
export class Produto {
  #id = crypto.randomUUID();

  constructor(nome, preco, categoria, estoque = 0) {
    this.nome      = nome;
    this.preco     = preco;
    this.categoria = categoria;
    this.estoque   = estoque;
  }

  get id()          { return this.#id; }
  get disponivel()  { return this.estoque > 0; }
  get precoFmt()    { return `R$ ${this.preco.toFixed(2)}`; }

  aplicarDesconto(pct) {
    this.preco *= (1 - pct);
    return this;
  }
}

export const filtrarPorCategoria = (produtos, cat) =>
  produtos.filter(p => p.categoria === cat);

export const totalEmEstoque = produtos =>
  produtos.reduce((acc, p) => acc + p.preco * p.estoque, 0);

// ─── main.js ─────────────────────────────────────────────────
import { Produto, filtrarPorCategoria, totalEmEstoque } from './produto.js';

const catalogo = [
  new Produto('Notebook', 4500, 'tech', 3),
  new Produto('Mouse', 150, 'tech', 50),
  new Produto('Cadeira', 1200, 'moveis', 0),
].map(p => p.categoria === 'tech' ? p.aplicarDesconto(0.1) : p);

const tech = filtrarPorCategoria(catalogo, 'tech');
console.log('Produtos tech com desconto:', tech.map(p => `${p.nome}: ${p.precoFmt}`));
console.log('Total em estoque: R$', totalEmEstoque(catalogo).toFixed(2));

⚠️ Armadilhas Comuns

1. sort() muta o array original

[...arr].sort(compareFn) — sempre faça uma cópia com spread antes de ordenar para evitar efeitos colaterais. O mesmo vale para splice() e reverse().

2. Comparação de objetos com ===

Objetos são comparados por referência, não por valor. { a: 1 } === { a: 1 } é false. Para comparar conteúdo use JSON.stringify() (simples) ou bibliotecas como Lodash's _.isEqual() (robusto).

3. || como default quando 0 ou "" são valores válidos

Use ?? (nullish coalescing) quando o valor padrão deve ser aplicado apenas para null/undefined, não para qualquer falsy (0, "", false).

4. map() quando você quer efeito colateral

map() é para transformar dados — ele sempre cria um novo array. Se você só quer executar código para cada item (ex: console.log, chamada de API), use forEach(). Retornar undefined implicitamente dentro de um map é sinal de uso errado.

5. Esquecer de retornar valor em reduce()

O acumulador do reduce() é o que você retorna a cada iteração. Esquecer o return (ou usar {} sem parênteses em arrow function) resulta em undefined acumulado.

// ❌ Bug: sem return
arr.reduce((acc, n) => { acc + n }, 0); // acumulador vira undefined!

// ✅ Correto:
arr.reduce((acc, n) => acc + n, 0);
arr.reduce((acc, n) => { return acc + n; }, 0); // ou com chaves + return explícito

Termos desta Aula

Destructuring
Sintaxe ES6 que extrai valores de objetos ou arrays para variáveis individuais em uma única expressão.
Spread Operator (...)
Expande um iterável (array, objeto) em seus elementos individuais, usado para clonar, mesclar e passar argumentos.
Rest Parameters (...)
Coleta múltiplos argumentos de função em um único array. Mesmo símbolo do spread, contexto diferente.
Prototype Chain
Mecanismo de herança do JS onde objetos delegam propriedades não encontradas ao seu protótipo, subindo a cadeia até null.
Named Export / Default Export
Named exports exportam múltiplas coisas por nome; default export exporta uma coisa principal por arquivo.
Nullish Coalescing (??)
Operador que retorna o lado direito apenas se o lado esquerdo for null ou undefined (não qualquer falsy).
Optional Chaining (?.)
Acessa propriedades aninhadas de forma segura — retorna undefined se qualquer nível da cadeia for null/undefined.
