Animações, Tipografia & Efeitos

⏱ 35–45 min de leitura 🎯 Intermediário → Avançado 🔧 Demos interativas incluídas
transition @keyframes animation transform 3D glassmorphism gradient text filter will-change
// motivação

Por que isso importa?

Interfaces modernas não são estáticas. Micro-interações — o hover de um botão, o loading spinner, o card que levanta — comunicam estado, guiam a atenção e criam sensação de qualidade. A diferença entre um produto que parece profissional e um que parece amador é frequentemente feita em CSS.

CSS moderno — com transition, @keyframes e transform — faz animações performáticas sem JavaScript. A chave: animar apenas opacity e transform (GPU), nunca width, height ou margin (CPU).

💼 Por que um engenheiro sênior usa isso hoje

Skeleton loaders substituem spinners em aplicações modernas (Meta, Spotify, LinkedIn). Glassmorphism e gradient text são o padrão visual de 2024-25 em produtos como Arc, Linear e Vercel. @starting-style e transition-behavior agora permitem animar display: none → block nativamente. Entender o compositor do browser — GPU vs CPU — é o que separa animações fluidas de animações que "travam".

// analogia

Antes do código: uma analogia

🌍 Mundo Real → Tecnologia
Pense em um teatro de marionetes. Transitions são movimentos suaves de um estado para outro (a marionete levanta o braço gradualmente). Keyframes são coreografias — o roteirista define exatamente o que acontece em cada momento do espetáculo (0%, 50%, 100%). O easing é o ritmo — movimentos acelerados no início e desacelerados no fim parecem mais naturais. E o compositor do browser é o palco — operações que acontecem nele (transform, opacity) são fluidas; operações que exigem recálculo do cenário inteiro (width, layout) são lentas.
Movimento suave de A→B → transition Coreografia com roteiro → @keyframes + animation Ritmo do movimento → timing-function / easing Palco (GPU) vs bastidores (CPU) → compositor vs layout

Transition vs Animation

Dois mecanismos distintos com casos de uso diferentes:

// quando usar cada um
flowchart TD Q{Precisa de animação?} --> A{Disparada por\nevent/state?} A -->|Sim hover focus class| T[transition\nSimples A→B\nCSS puro] A -->|Não — roda sozinha| K[@keyframes + animation\nCoreografia complexa\nloop possível] T --> T1[hover, focus, :checked\nclass toggle, checked state] K --> K1[spinner, shimmer, pulse\nentrada de elementos\nseq complexa] style Q fill:#1e1e2e,stroke:#ef4444,color:#e8e8f0 style T fill:#16161f,stroke:#10b981,color:#34d399 style K fill:#16161f,stroke:#ef4444,color:#f87171
transition

Transição de estado

Define como uma propriedade muda de um valor a outro. Requer um gatilho: hover, focus, ou adição de classe via JS. transition: property duration easing delay

@keyframes

Animação autônoma

Define uma coreografia completa com múltiplos estágios. Pode loopar, pausar, reverter. Independente de estado do usuário.

GPU-safe

Animar apenas opacity e transform

Essas propriedades rodam no compositor (GPU) — 60fps garantidos. Evite animar: width, height, top, left, margin, padding.

will-change

Dica de performance ao browser

Diz ao browser para preparar a GPU antecipadamente. Use com moderação: will-change: transform em elementos que animam frequentemente.

/* ── Botão com transition multi-propriedade ─────── */
.btn {
  padding: 12px 28px;
  background: #6366f1;
  color: white;
  border: none;
  border-radius: 8px;
  font-size: 15px;
  cursor: pointer;
  position: relative;
  overflow: hidden;

  /* Transição em múltiplas propriedades */
  transition:
    transform 0.2s cubic-bezier(0.34, 1.56, 0.64, 1), /* spring */
    box-shadow 0.2s ease,
    background-color 0.2s ease;
}

.btn:hover {
  background: #4f46e5;
  transform: translateY(-3px);
  box-shadow: 0 12px 32px rgba(99, 102, 241, 0.5);
}

.btn:active {
  transform: translateY(0) scale(0.98);
  box-shadow: none;
}

/* Ripple effect com ::after */
.btn::after {
  content: '';
  position: absolute;
  inset: 0;
  background: radial-gradient(circle, rgba(255,255,255,0.3) 0%, transparent 70%);
  opacity: 0;
  transition: opacity 0.4s;
}
.btn:active::after { opacity: 1; }

Exemplo completo: Sistema de animações com prefers-reduced-motion

Nunca force animações em usuários com sensibilidade a movimento:

acessibilidade.csscss
/* ── Sistema de animações acessível ─────────────── */

/* Define as animações normalmente */
@keyframes slideIn {
  from { opacity: 0; transform: translateX(-24px); }
  to   { opacity: 1; transform: translateX(0); }
}

.animated { animation: slideIn 0.4s ease-out both; }

/* OBRIGATÓRIO — respeita a preferência do usuário */
@media (prefers-reduced-motion: reduce) {
  /* Remove animações para usuários com epilepsia ou tontura */
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}

/* ── Scroll-driven animations (Chrome 115+) ─────── */
/* Anima baseado na posição do scroll — sem JS! */
@keyframes revealOnScroll {
  from { opacity: 0; transform: translateY(40px); }
  to   { opacity: 1; transform: translateY(0); }
}
.scroll-reveal {
  animation: revealOnScroll linear both;
  animation-timeline: view();
  animation-range: entry 0% entry 30%;
}

/* ── Loading state completo ─────────────────────── */
.btn-loading {
  position: relative;
  pointer-events: none;
  color: transparent !important;
}
.btn-loading::after {
  content: '';
  position: absolute;
  inset: 0;
  margin: auto;
  width: 18px; height: 18px;
  border: 2px solid rgba(255,255,255,0.3);
  border-top-color: white;
  border-radius: 50%;
  animation: spin 0.6s linear infinite;
}

/* Toggle via JS: btn.classList.toggle('btn-loading') */

Erros que arruínam a performance

⚠️ Animar propriedades que causam layout reflow

Animar width, height, top, left, margin ou padding força o browser a recalcular o layout de toda a página a cada frame — o que causa jank (frames perdidos). Regra de ouro: anime apenas transform e opacity. Use transform: translateX() em vez de left.

⚠️ will-change em tudo

will-change: transform cria uma nova camada na GPU para o elemento — excelente para animações frequentes, mas se usado em excesso, consome memória VRAM. Use apenas em elementos que realmente animam, e remova após a animação via JS se necessário.

⚠️ Esquecer prefers-reduced-motion

Para usuários com epilepsia fotossensível, doenças vestibulares ou enjoo de movimento, animações podem ser debilitantes. É obrigação de acessibilidade incluir @media (prefers-reduced-motion: reduce).

⚠️ backdrop-filter sem fallback

backdrop-filter não funciona se o elemento pai tiver overflow: hidden. Sempre forneça um fallback de fundo sólido semi-transparente para browsers que não suportam.

  • Cada card tem um efeito de hover com transition (elevação + sombra colorida)
  • Um skeleton loader aparece antes dos cards com animação shimmer
