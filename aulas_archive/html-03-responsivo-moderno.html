🌐 HTML/CSS Aula 08

CSS Responsivo & Moderno

⏱ 30–40 min de leitura 🎯 Intermediário 🔧 Hands-on incluído
@media query clamp() custom properties container queries :is() :where() mobile-first fluid typography
// motivação

Por que isso importa?

Mais de 60% do tráfego web global vem de dispositivos móveis. Um site bonito no desktop que quebra no celular é um produto ruim. CSS moderno — com clamp(), custom properties e container queries — permite criar layouts que se adaptam fluidamente sem media queries excessivas.

Além disso, Custom Properties (variáveis CSS) transformam a manutenção de design systems: altere uma variável e todo o tema muda. É o que frameworks como Tailwind, MUI e Radix UI fazem internamente.

💼 Por que um engenheiro sênior usa isso hoje

Design systems modernos (Linear, Vercel, Stripe) são construídos com CSS custom properties para tokens de design. Container queries substituem media queries baseadas na tela por queries baseadas no componente — mudança de paradigma que permite componentes verdadeiramente reutilizáveis. clamp() elimina breakpoints manuais para tipografia e espaçamento. Isso é o CSS de 2025.

// analogia

Antes do código: uma analogia

🌍 Mundo Real → Tecnologia
Pense em um arquiteto que projeta um espaço multiuso. Em vez de construir paredes fixas, ele usa divisórias móveis que se adaptam ao espaço disponível. O design base funciona para uma sala pequena, e as divisórias se reorganizam quando o espaço cresce. As cores e materiais são definidos uma vez em um catálogo central — troca o catálogo, troca tudo. Media queries são as instruções de reorganização para cada tamanho de sala.
Divisórias móveis → Layout fluido / flexível Catálogo central de materiais → CSS Custom Properties Instruções por tamanho de sala → @media queries Sala pequena como base → Mobile-first design Espaço que se ajusta automaticamente → clamp() / fluid typography
// conceito

Mobile-first: a estratégia correta

Mobile-first significa escrever o CSS base para telas pequenas e usar min-width para expandir o layout — não o contrário. Isso resulta em CSS mais limpo, menor e performático:

Custom Props

Variáveis CSS nativas

Diferente das variáveis Sass, Custom Properties são dinâmicas — podem ser alteradas com JavaScript em tempo real, e herdam através do DOM.

clamp()

Valores fluidos sem breakpoints

clamp(min, ideal, max) cria valores que crescem proporcionalmente entre um mínimo e máximo. Adeus, media queries para tipografia!

Container Queries

Responsividade por componente

Enquanto media queries respondem à viewport, container queries respondem ao tamanho do container pai — perfeito para componentes reutilizáveis.

:is() :where()

Seletores modernos

Agrupam seletores com especificidade inteligente. :is() mantém a especificidade mais alta do grupo; :where() sempre tem especificidade zero.

Mãos na massa

Crie um arquivo responsivo.html e construa um design system responsivo do zero:

1
Configure o design system com Custom Properties
Tokens de design centralizados — base para temas claro/escuro:
style.csscss
/* ── Tokens de design ─────────────────────────── */
:root {
  /* Cores */
  --color-bg: #ffffff;
  --color-surface: #f8fafc;
  --color-text: #0f172a;
  --color-muted: #64748b;
  --color-accent: #6366f1;
  --color-accent-hover: #4f46e5;
  --color-border: #e2e8f0;

  /* Tipografia fluida */
  --text-sm: clamp(0.8rem, 0.17vw + 0.76rem, 0.89rem);
  --text-base: clamp(1rem, 0.34vw + 0.91rem, 1.19rem);
  --text-lg: clamp(1.25rem, 0.61vw + 1.1rem, 1.58rem);
  --text-xl: clamp(1.56rem, 1vw + 1.31rem, 2.11rem);
  --text-2xl: clamp(1.95rem, 1.56vw + 1.56rem, 2.81rem);
  --text-3xl: clamp(2.44rem, 2.38vw + 1.85rem, 3.75rem);

  /* Espaçamento fluido */
  --space-xs: clamp(0.25rem, 0.5vw, 0.5rem);
  --space-sm: clamp(0.5rem, 1vw, 1rem);
  --space-md: clamp(1rem, 2vw, 2rem);
  --space-lg: clamp(1.5rem, 3vw, 3rem);
  --space-xl: clamp(2rem, 5vw, 5rem);

  /* Layout */
  --container-sm: 640px;
  --container-md: 768px;
  --container-lg: 1024px;
  --container-xl: 1280px;

  /* Bordas */
  --radius-sm: 4px;
  --radius-md: 8px;
  --radius-lg: 16px;
  --radius-full: 9999px;
}

/* Tema escuro automático */
@media (prefers-color-scheme: dark) {
  :root {
    --color-bg: #0f172a;
    --color-surface: #1e293b;
    --color-text: #f1f5f9;
    --color-muted: #94a3b8;
    --color-border: #334155;
  }
}
2
Layout mobile-first com Grid
Comece com 1 coluna, expanda progressivamente:
style.csscss
/* ── Reset e base ─────────────────────────────── */
*, *::before, *::after { box-sizing: border-box; margin: 0; }
body { font-family: system-ui, sans-serif; font-size: var(--text-base); color: var(--color-text); background: var(--color-bg); line-height: 1.6; }

/* ── Container ────────────────────────────────── */
.container { width: 100%; max-width: var(--container-xl); margin-inline: auto; padding-inline: var(--space-md); }

/* ── Grid responsivo — Mobile-first ─────────────── */
.grid {
  display: grid;
  gap: var(--space-md);
  grid-template-columns: 1fr; /* mobile: 1 coluna */
}

@media (min-width: 640px) {
  .grid { grid-template-columns: repeat(2, 1fr); }
}

@media (min-width: 1024px) {
  .grid { grid-template-columns: repeat(3, 1fr); }
}

@media (min-width: 1280px) {
  .grid { grid-template-columns: repeat(4, 1fr); }
}

/* ── Auto-responsive sem media queries ─────────── */
.grid-auto {
  display: grid;
  gap: var(--space-md);
  /* auto-fill: quantas colunas couberem com min 280px */
  grid-template-columns: repeat(auto-fill, minmax(280px, 1fr));
}
3
Container Queries — componente inteligente
O card se adapta ao seu container, não à viewport:
style.csscss
/* ── Container Queries ────────────────────────── */

/* 1. Defina o container no pai */
.card-wrapper {
  container-type: inline-size;
  container-name: card;
}

/* 2. Escreva regras para o componente filho */
.card {
  display: grid;
  grid-template-columns: 1fr; /* empilhado por padrão */
  gap: var(--space-sm);
  padding: var(--space-md);
  background: var(--color-surface);
  border: 1px solid var(--color-border);
  border-radius: var(--radius-md);
}

/* Quando o CARD tiver +400px, vai lado a lado */
@container card (min-width: 400px) {
  .card {
    grid-template-columns: 120px 1fr;
    align-items: center;
  }
}

/* Tipografia que responde ao container */
@container card (min-width: 600px) {
  .card-title { font-size: var(--text-xl); }
}
4
Tipografia fluida com clamp()
Sem breakpoints, o texto cresce proporcionalmente:
style.csscss
/* ── Tipografia fluida ─────────────────────────── */

/* clamp(MÍNIMO, VALOR-FLUIDO, MÁXIMO) */
/* O valor fluido cresce com a viewport */

h1 {
  /* 28px → 56px entre 320px e 1280px de viewport */
  font-size: clamp(1.75rem, 4vw + 0.5rem, 3.5rem);
  line-height: 1.1;
}

h2 { font-size: clamp(1.25rem, 2.5vw + 0.5rem, 2rem); }
p  { font-size: clamp(1rem, 1vw + 0.5rem, 1.125rem); }

/* Largura do texto — sempre legível */
.prose {
  max-width: 65ch; /* 65 caracteres — legibilidade ideal */
}

/* Espaçamento fluido */
.section {
  padding-block: clamp(3rem, 8vw, 8rem);
}

/* ── Seletores modernos ────────────────────────── */

/* :is() — substitui lista repetitiva */
:is(h1, h2, h3, h4, h5, h6) {
  font-weight: 700;
  line-height: 1.2;
}

/* :where() — especificidade ZERO, fácil de sobrescrever */
:where(ul, ol) { padding-left: 1.5rem; }

/* :has() — o selector pai CSS */
.form-group:has(input:invalid) label { color: #ef4444; }
5
Toggle de tema claro/escuro com CSS puro
style.css + index.htmlcss
/* Tema escuro via classe no :root */
:root[data-theme="dark"] {
  --color-bg: #0f172a;
  --color-surface: #1e293b;
  --color-text: #f1f5f9;
  --color-muted: #94a3b8;
  --color-border: #334155;
}

/* Transição suave entre temas */
*, *::before, *::after {
  transition: background-color 0.2s, color 0.2s, border-color 0.2s;
}
// Toggling com JavaScript
const btn = document.querySelector('#theme-toggle');
btn.addEventListener('click', () => {
  const current = document.documentElement.getAttribute('data-theme');
  document.documentElement.setAttribute(
    'data-theme', current === 'dark' ? 'light' : 'dark'
  );
});

Erros comuns em CSS responsivo

⚠️ Usar px em vez de rem para tipografia

Pixels ignoram as preferências de acessibilidade do usuário. Se o usuário definiu 18px como tamanho padrão no browser, 16px ignora isso; 1rem respeita. Use rem para texto e espaçamento, px só para bordas e sombras.

⚠️ Desktop-first (max-width) gera CSS mais difícil de manter

Media queries com max-width (desktop-first) adicionam exceções ao código base. Mobile-first com min-width progride naturalmente — é mais fácil adicionar do que sobrescrever.

⚠️ Usar vh no mobile sem dvh

Em mobile, 100vh inclui a barra de endereço, causando overflow. Use 100dvh (dynamic viewport height) para elementos fullscreen no mobile. Suporte: ~95% dos browsers modernos.

🏁 Checkpoint — valide seu aprendizado

Crie uma landing page responsiva completa usando apenas CSS moderno (sem frameworks):

  • Defina pelo menos 10 custom properties para cores, tipografia e espaçamento
  • Implemente tipografia 100% fluida com clamp() — sem media queries para font-size
  • Use auto-fill / auto-fit + minmax() para pelo menos um grid que não precise de media queries
  • Adicione pelo menos uma Container Query em um componente card
  • Implemente toggle de tema claro/escuro modificando as custom properties
  • Teste no celular — garanta que não há overflow horizontal e que o texto é legível
💬 Dica: use o site clamp.font-size.app para calcular valores de clamp() com precisão. Para testar responsividade, use o DevTools do Chrome em modo Device — teste em 320px, 768px e 1440px.
