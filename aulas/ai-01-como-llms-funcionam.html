01 · Motivação

Por que entender como LLMs funcionam?

Você pode usar GPT-4 sem entender como ele funciona — assim como pode dirigir um carro sem saber mecânica. Mas quando o modelo alucina, recusa ou se comporta de forma inesperada, entender a mecânica interna é o que separa quem debugga em minutos de quem fica horas tentando.

🎯

Prompts melhores

Saber como o modelo processa tokens ajuda a escrever prompts que realmente funcionam.

💰

Custo de API

Tokens custam dinheiro. Saber como contar tokens evita surpresas na fatura.

🔧

Parâmetros certos

Temperature, top-p e top-k controlam criatividade vs. precisão. Saber usar é essencial.

💡 Por que um engenheiro sênior precisa saber isso?
Em 2024–2025, engenheiros que integram LLMs em produtos são os mais valorizados do mercado. Mas quem entende a mecânica constrói sistemas mais robustos, econômicos e previsíveis — não só "chama a API e torce".
02 · Analogia

O Escritor Estatístico Ultra-Sofisticado

✍️ Analogia
Imagine um escritor que leu toda a internet e internalizou os padrões estatísticos de como as palavras se relacionam. Quando você diz "Complete: O céu é...", ele não sabe o significado de "azul" — ele sabe que, estatisticamente, depois de "O céu é" a palavra mais provável é "azul". Mas esse escritor tem uma memória de trabalho limitada (context window), se confunde com textos muito longos, e às vezes "inventa" com confiança quando não tem certeza. A temperatura é o quanto esse escritor está "bêbado" — 0.0 é extremamente conservador e previsível, 1.5 é criativo e caótico.
📚 Leu toda a internet
→ Pré-treinamento em trilhões de tokens
🔤 Palavras que combinam
→ Próximo token mais provável
🧠 Memória de trabalho
→ Context window (tokens)
🍺 Nível de "bebedeira"
→ Temperature (0.0 – 2.0)
📝 Revisão por humano
→ RLHF — feedback humano
03 · Conceito Central

Tokens — A Unidade Fundamental dos LLMs

LLMs não processam palavras — processam tokens. Um token é um pedaço de texto, geralmente uma palavra ou parte dela. O modelo usa um vocabulário fixo (ex: GPT-4 tem ~100k tokens) e representa todo texto como sequência de números inteiros.

Como o texto é tokenizado

Docker é uma plataforma de contêinerização

"Dockerização" vira 3 tokens. "é" = 1 token. Espaços geralmente fazem parte do token seguinte. Palavras raras são divididas em sub-palavras (BPE — Byte Pair Encoding).

⚠️ Implicações práticas dos tokens
  • Inglês é mais eficiente que português: "containerization" = 3 tokens; "containerização" = 4–5 tokens
  • Código é tokenizado de forma especial — identação, símbolos e palavras reservadas têm tokens próprios
  • GPT-4: ~$0.03/1k tokens input. Um documento de 10 páginas ≈ 3.000–5.000 tokens
  • Regra prática: 1 token ≈ 0.75 palavra em inglês; ≈ 0.6 palavra em português
# Contando tokens com tiktoken (biblioteca da OpenAI)
import tiktoken

enc = tiktoken.encoding_for_model("gpt-4")

texto = "Docker é uma plataforma de containerização"
tokens = enc.encode(texto)

print(f"Texto: {texto}")
print(f"Número de tokens: {len(tokens)}")  # ex: 9
print(f"IDs dos tokens: {tokens}")          # ex: [40, 82, 374, ...]

# Decodificar tokens individuais
for token_id in tokens:
    print(repr(enc.decode([token_id])))
04 · Arquitetura

Transformers — A Arquitetura por Trás de Tudo

Todos os LLMs modernos (GPT, Claude, Gemini, Llama) são baseados na arquitetura Transformer, proposta no paper "Attention Is All You Need" (Google, 2017). O componente chave é o mecanismo de atenção.

Self-Attention — O Coração do Transformer

O mecanismo de atenção permite que o modelo "preste atenção" em diferentes partes do contexto ao processar cada token. Para a palavra "banco" na frase "Fui ao banco pescar", o modelo aprende a prestar mais atenção em "pescar" para entender que é margem de rio, não instituição financeira.

🔥 Por que "Attention is All You Need"?
Antes dos Transformers, usávamos RNNs (redes recorrentes) que processavam sequências palavra por palavra — lento e sem paralelismo. O Transformer processa todos os tokens em paralelo usando atenção, permitindo treinar em escala muito maior com GPUs.
05 · Embeddings

Embeddings — Significado como Números

Um embedding é a representação de um token (ou texto) como um vetor numérico de alta dimensão (ex: 1536 dimensões no text-embedding-ada-002 da OpenAI). Textos semanticamente similares ficam próximos no espaço vetorial.

Espaço de embeddings — similaridade semântica

# Gerando embeddings com a API da OpenAI
from openai import OpenAI
import numpy as np

client = OpenAI()

def get_embedding(text: str) -> list[float]:
    response = client.embeddings.create(
        input=text,
        model="text-embedding-3-small"  # 1536 dimensões, barato
    )
    return response.data[0].embedding

def cosine_similarity(v1: list, v2: list) -> float:
    """Mede similaridade entre dois vetores (0 = diferente, 1 = idêntico)"""
    a, b = np.array(v1), np.array(v2)
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

# Comparar similaridade semântica
e1 = get_embedding("cachorro")
e2 = get_embedding("cão")
e3 = get_embedding("automóvel")

print(cosine_similarity(e1, e2))  # ~0.92 — muito similar
print(cosine_similarity(e1, e3))  # ~0.15 — bem diferente
06 · Parâmetros de Geração

Context Window, Temperature & Sampling

Context Window — A Memória do Modelo

A context window é o número máximo de tokens que o modelo pode processar de uma vez (input + output combinados). Tudo fora desse limite é simplesmente ignorado — o modelo não tem acesso.

from openai import OpenAI
client = OpenAI()

def gerar_resposta(prompt: str, temperature: float = 0.7, top_p: float = 1.0):
    """
    temperature: 0.0 = determinístico, 2.0 = caótico
    top_p: 0.9 = considera apenas os 90% de probabilidade acumulada
    top_k: não disponível na API OpenAI, mas presente em Anthropic/Ollama
    max_tokens: limite de tokens na resposta
    """
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": prompt}],
        temperature=temperature,
        top_p=top_p,
        max_tokens=1000,
    )
    return response.choices[0].message.content

# Para código: temperature baixa (previsível)
codigo = gerar_resposta(
    "Escreva uma função Python para ordenar uma lista",
    temperature=0.1
)

# Para criativo: temperature alta
historia = gerar_resposta(
    "Escreva o início de um conto de ficção científica",
    temperature=1.0
)

Pré-treinamento, Fine-tuning e RLHF

📚

Pré-treinamento

Modelo aprende a prever o próximo token em trilhões de tokens. Semanas/meses em milhares de GPUs. Custo: dezenas de milhões de dólares.

🎯

SFT (Supervised Fine-tuning)

Fine-tuning em pares (instrução → resposta ideal) criados por humanos. Ensina o modelo a seguir instruções, não só completar texto.

👥

RLHF

Humanos rankeiam respostas → treina um reward model → usa RL (PPO) para otimizar o LLM. É o que faz modelos serem "úteis e seguros".

09 · Armadilhas

⚠️ Conceitos que Geram Confusão

⚠️ LLMs não "sabem" — eles predizem

LLMs não têm conhecimento ou entendimento real. Eles geram o texto estatisticamente mais provável dado o contexto. É por isso que "alucinam" com confiança — o modelo não sabe que está errado, só sabe que aquela sequência de tokens é plausível.

⚠️ Context window ≠ memória persistente

O modelo não lembra de conversas anteriores. Cada chamada à API começa do zero. O que parece "memória" no ChatGPT é apenas o histórico da conversa sendo reenviado a cada mensagem — consumindo tokens.

⚠️ Temperature 0.0 não é sempre melhor

Temperature=0.0 é determinístico mas pode ser repetitivo e limitado. Para código e fatos: use 0.0–0.3. Para escrita criativa ou brainstorming: 0.7–1.0. Não existe "temperatura ideal" universal.

⚠️ Tamanho do modelo ≠ qualidade sempre

Um modelo menor com fine-tuning específico para sua tarefa frequentemente supera um modelo maior generalista. Llama 3.1 8B fine-tunado para SQL pode ser melhor que GPT-4 para queries SQL específicas.

Desafio: Explorar os Parâmetros de Geração

Use a API da OpenAI ou Anthropic (ambas têm tier gratuito) para explorar os parâmetros de geração:

  • Instale openai com pip install openai e configure sua API key
  • Gere 5 respostas para o mesmo prompt com temperatures: 0.0, 0.3, 0.7, 1.0, 1.5
  • Use tiktoken para contar os tokens do seu prompt e calcular o custo estimado
  • Compare as respostas: qual temperature produziu o resultado mais útil?
  • Bônus: Gere embeddings de 5 frases e calcule a similaridade cosseno entre elas — visualize quais são semanticamente próximas
💬 Dica: Para experimentar sem gastar, use a API do Google Gemini (tem tier gratuito generoso) ou instale o Ollama localmente com ollama pull llama3.2:3b — roda em qualquer máquina com 4GB de RAM.
