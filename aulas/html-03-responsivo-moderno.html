CSS — Responsivo & Moderno

⏱ ~30 min de leitura 🎯 Intermediário 🔧 Hands-on incluído
@media min-width mobile-first breakpoints container queries clamp() viewport units
// motivação

Por que isso importa?

Mais de 60% do tráfego web global vem de dispositivos móveis. Se seu site não adapta seu layout ao tamanho da tela do usuário, você está essencialmente bloqueando mais da metade dos seus visitantes de ter uma boa experiência. Design responsivo não é opcional — é o mínimo esperado.

Mas responsividade moderna vai além de media queries tradicionais. Com container queries, clamp(), e unidades de viewport dinâmicas, você escreve CSS que se adapta não apenas ao tamanho da janela, mas ao contexto do próprio componente.

💼 Como engenheiros sênior usam isso hoje

Equipes de front-end em empresas como Nubank e iFood constroem sistemas de design com breakpoints padronizados e tokens de tipografia fluida usando clamp(). Um componente de card, por exemplo, reage ao espaço disponível em seu container (container queries) — não ao viewport inteiro. Isso permite reutilizar o mesmo componente em uma sidebar estreita ou em um grid de tela cheia sem alterar o HTML.

// analogia

Antes do código: uma analogia

🌍 Mundo Real → Tecnologia
Imagine uma peça de roupa com tecnologia de compressão adaptável — ela se ajusta automaticamente ao corpo de quem a veste: fica mais larga para alguém grande, mais compacta para alguém pequeno, mas sempre com o mesmo design elegante. Você não precisa de versões separadas da roupa para cada tamanho.

Em CSS, as media queries são esse mecanismo de adaptação. Você escreve um único conjunto de regras que o navegador aplica de forma diferente dependendo do "tamanho" (viewport) do dispositivo. Com clamp(), é como se a roupa se ajustasse continuamente em vez de em saltos.
👗 Tamanho da roupa (P/M/G/GG) → Breakpoints (480px / 768px / 1024px / 1280px) 🪡 Tecido elástico que se ajusta → clamp(min, ideal, max) 📐 Medida a partir do corpo (não da vitrine) → Container queries 👕 Começa pelo menor tamanho → Mobile-first approach

Media Queries — o coração da responsividade

Uma media query é uma condição que o CSS avalia antes de aplicar um bloco de estilos. A sintaxe básica é:

syntax.css CSS 
/* ─── Sintaxe básica ────────────────────────────── */
@media (condição) {
  /* CSS aplicado apenas quando condição for verdadeira */
}

/* ─── Exemplos reais ─────────────────────────────── */

/* Mobile-first: estilos base para mobile */
.card {
  flex-direction: column;
  padding: 16px;
}

/* Tablet: muda quando tela tem pelo menos 768px */
@media (min-width: 768px) {
  .card {
    flex-direction: row;
    padding: 24px;
  }
}

/* Desktop: muda quando tela tem pelo menos 1024px */
@media (min-width: 1024px) {
  .card {
    padding: 32px;
    gap: 32px;
  }
}

/* Orientação: regras para tela em paisagem */
@media (orientation: landscape) and (max-height: 500px) {
  .hero { min-height: 60vh; }
}

/* Preferência do sistema: modo escuro/claro */
@media (prefers-color-scheme: dark) {
  :root { --bg: #0a0a0f; }
}

/* Redução de movimento: acessibilidade */
@media (prefers-reduced-motion: reduce) {
  * { animation-duration: 0.01ms !important; }
}

Mãos na massa

Crie um arquivo responsivo.html e implemente um layout completo mobile-first:

1
Configure o viewport e estrutura base
Sem a meta tag viewport, as media queries não funcionam em dispositivos móveis reais.
responsivo.html HTML 
<!DOCTYPE html>
<html lang="pt-BR">
<head>
  <meta charset="UTF-8">
  <!-- CRÍTICO: sem isso, mobile ignora suas media queries! -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Layout Responsivo</title>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <header class="header">
    <div class="logo">DevBlog</div>
    <nav class="nav">
      <a href="#">Início</a>
      <a href="#">Artigos</a>
      <a href="#">Sobre</a>
    </nav>
    <button class="menu-toggle">☰</button>
  </header>

  <main class="layout">
    <section class="content">
      <h1>Artigo Principal</h1>
      <div class="cards-grid">
        <div class="card">Card 1</div>
        <div class="card">Card 2</div>
        <div class="card">Card 3</div>
        <div class="card">Card 4</div>
      </div>
    </section>
    <aside class="sidebar">
      <h2>Sidebar</h2>
      <p>Conteúdo complementar</p>
    </aside>
  </main>
</body>
</html>
2
Estilos base (mobile-first)
Comece pelo menor. Estilos sem media query são o ponto de partida para mobile.
styles.css — parte 1: base mobile CSS 
/* ─── RESET & BASE ────────────────────────────── */
*, *::before, *::after { box-sizing: border-box; margin: 0; padding: 0; }

body {
  font-family: system-ui, sans-serif;
  font-size: 1rem; /* 16px base */
  line-height: 1.6;
  color: #333;
}

/* ─── HEADER — mobile: stacked ──────────────────── */
.header {
  display: flex;
  align-items: center;
  justify-content: space-between;
  padding: 16px;
  background: #1e293b;
  color: white;
}

/* Navegação OCULTA no mobile */
.nav {
  display: none;
  gap: 16px;
}

.nav a { color: white; text-decoration: none; }

/* Botão hamburguer VISÍVEL no mobile */
.menu-toggle { display: block; background: none; border: none; color: white; font-size: 24px; }

/* ─── LAYOUT — mobile: uma coluna ───────────────── */
.layout {
  display: flex;
  flex-direction: column; /* empilhado no mobile */
  gap: 24px;
  padding: 16px;
  max-width: 1200px;
  margin: 0 auto;
}

/* ─── CARDS GRID — mobile: 1 coluna ─────────────── */
.cards-grid {
  display: grid;
  grid-template-columns: 1fr; /* 1 coluna */
  gap: 16px;
}

.card {
  padding: 20px;
  background: #f1f5f9;
  border-radius: 8px;
  border-left: 4px solid #ef4444;
}

/* ─── TIPOGRAFIA FLUIDA com clamp() ─────────────── */
h1 {
  /* mínimo 24px, ideal 5vw, máximo 40px */
  font-size: clamp(1.5rem, 5vw, 2.5rem);
  margin-bottom: 24px;
}

h2 {
  font-size: clamp(1.2rem, 3.5vw, 1.8rem);
  margin-bottom: 16px;
}
3
Adicione os breakpoints progressivos
Cada breakpoint expande o layout — nunca redefinindo, apenas expandindo.
styles.css — parte 2: media queries CSS 
/* ─── TABLET (768px+) ──────────────────────────── */
@media (min-width: 768px) {
  /* Header: mostra nav, esconde botão */
  .nav { display: flex; }
  .menu-toggle { display: none; }

  /* Cards: 2 colunas */
  .cards-grid {
    grid-template-columns: repeat(2, 1fr);
  }

  /* Padding aumentado */
  .layout { padding: 24px; }
}

/* ─── DESKTOP (1024px+) ─────────────────────────── */
@media (min-width: 1024px) {
  /* Layout: conteúdo + sidebar lado a lado */
  .layout {
    flex-direction: row;
    align-items: flex-start;
  }

  .content { flex: 1; }    /* ocupa o espaço disponível */
  .sidebar { width: 280px; flex-shrink: 0; } /* largura fixa */

  /* Cards: 2 colunas (dentro do .content, não tela cheia) */
  .cards-grid {
    grid-template-columns: repeat(2, 1fr);
  }
}

/* ─── MONITOR GRANDE (1280px+) ──────────────────── */
@media (min-width: 1280px) {
  /* Cards: 4 colunas no desktop grande */
  .cards-grid {
    grid-template-columns: repeat(4, 1fr);
  }
}

/* ─── PREFERÊNCIA DE MOVIMENTO ──────────────────── */
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}

Container Queries — o futuro do CSS responsivo
Com container queries, o componente reage ao seu próprio container, não ao viewport.
container-query.css CSS 
/* ─── CONTAINER QUERIES (CSS moderno, 2023+) ──── */

/* 1. Declare o container que será monitorado */
.cards-grid {
  container-type: inline-size; /* monitora a largura */
  container-name: cards;       /* nome opcional */
}

/* 2. Regras para o FILHO baseadas no tamanho do CONTAINER */
@container cards (min-width: 400px) {
  .card {
    display: flex;
    gap: 16px;
  }
}

@container cards (min-width: 600px) {
  .card {
    flex-direction: column;
    align-items: center;
  }
}

/* ─── VANTAGEM: mesmo componente em sidebar (estreita)
   e em grid (largo) — comportamento diferente automaticamente!
   A sidebar pode ter 280px e o card fica empilhado,
   enquanto o grid tem 800px e o card fica lado a lado. ───── */

Técnicas modernas de tipografia fluida

clamp() é a forma mais elegante de criar tipografia que escala suavemente entre tamanhos de tela, sem nenhuma media query para fonte.

fluid-typography.css CSS 
/*
  Sistema de tipografia fluida — sem media queries!
  Execute no browser: abra DevTools → toggle device toolbar
  Redimensione a janela e veja as fontes se adaptarem suavemente.
*/

:root {
  /* ─── ESCALA FLUIDA ───────────────────────────────────────
     clamp(MÍNIMO, IDEAL, MÁXIMO)
     IDEAL: vw = viewport width, dvw = dynamic viewport width
     ─────────────────────────────────────────────────────── */

  --font-xs:   clamp(0.75rem,  1.5vw, 0.875rem);  /* 12px → 14px */
  --font-sm:   clamp(0.875rem, 2vw,   1rem);       /* 14px → 16px */
  --font-base: clamp(1rem,     2.5vw, 1.125rem);   /* 16px → 18px */
  --font-lg:   clamp(1.125rem, 3vw,   1.5rem);     /* 18px → 24px */
  --font-xl:   clamp(1.5rem,   4vw,   2rem);       /* 24px → 32px */
  --font-2xl:  clamp(2rem,     6vw,   3rem);       /* 32px → 48px */
  --font-3xl:  clamp(2.5rem,   8vw,   4.5rem);     /* 40px → 72px */

  /* ─── ESPAÇAMENTO FLUIDO ─────────────────────────────── */
  --space-sm:  clamp(8px,  2vw, 16px);
  --space-md:  clamp(16px, 3vw, 24px);
  --space-lg:  clamp(24px, 5vw, 48px);
  --space-xl:  clamp(48px, 8vw, 96px);
}

/* Uso: */
h1 { font-size: var(--font-2xl); margin-bottom: var(--space-md); }
h2 { font-size: var(--font-xl);  margin-bottom: var(--space-sm); }
p  { font-size: var(--font-base); line-height: 1.7; }

/* ─── UNIDADES MODERNAS DE VIEWPORT ──────────────────────
   Problema: 100vh em mobile inclui a barra de endereço.
   Solução: unidades dinâmicas (suporte 2022+)         */

.hero {
  min-height: 100dvh; /* dynamic viewport height */
  width: 100dvw;      /* dynamic viewport width */
}

/* svh = small viewport height (sem barras do browser) */
/* lvh = large viewport height (com barras do browser) */
/* dvh = dynamic (muda conforme barras aparecem/somem) */

.section-tall { min-height: 60svh; } /* sempre seguro */

Erros comuns em CSS responsivo

⚠️ Esquecer a meta tag viewport

Sem <meta name="viewport" content="width=device-width, initial-scale=1.0">, o navegador móvel renderiza a página como se tivesse 980px de largura e suas media queries para mobile nunca disparam. É o erro #1 de quem começa com responsividade.

⚠️ Desktop-first em vez de mobile-first

Usar max-width em vez de min-width força você a sobrescrever estilos de desktop para mobile. Mobile-first com min-width é mais limpo: você começa simples e adiciona complexidade para telas maiores. Estilos base para o menor dispositivo, expansão para telas maiores.

⚠️ Pixels fixos que quebram o layout

Usar width: 400px em um container que pode aparecer em telas de 320px. Prefira max-width: 400px; width: 100% ou unidades relativas. Nunca use height fixo para textos — deixe o conteúdo determinar a altura. Use min-height quando necessário.

🏁 Checkpoint — valide seu aprendizado

Construa uma página de portfólio responsiva do zero, com layout mobile-first. A página deve se adaptar corretamente em 3 breakpoints distintos:

  • ✓ Mobile (base): header empilhado com botão hamburguer visível, navegação oculta, seção de projetos em 1 coluna, tipografia base de 16px
  • ✓ Tablet (768px+): navegação exposta horizontalmente, grid de projetos em 2 colunas, sem botão hamburguer
  • ✓ Desktop (1024px+): sidebar "sobre mim" ao lado do conteúdo principal, projetos em 3 colunas, header com mais espaçamento
  • ✓ Use clamp() para pelo menos o tamanho do h1 e um elemento de espaçamento — confirme que a fonte escala suavemente ao redimensionar
  • ✓ Adicione @media (prefers-color-scheme: dark) com pelo menos 2 variáveis CSS alteradas para o modo escuro
💬 Dica: Comece pelo mobile sem nenhuma media query. Só adicione o breakpoint de 768px quando o mobile estiver perfeito. Use o Chrome DevTools (F12 → ícone de celular) para testar cada breakpoint. Confirme que a meta viewport está presente ou nada vai funcionar no mobile real.
