// 01 — motivação

Por que isso importa?

Em 2024, mais de 60% do tráfego web mundial vem de dispositivos móveis. Um layout que parece perfeito no seu monitor de 27" pode ser completamente inutilizável num celular. CSS responsivo não é mais um diferencial — é o requisito mínimo de qualquer produto profissional.

Mas responsividade vai além de media queries. Um engenheiro sênior hoje usa CSS Custom Properties para criar sistemas de design escaláveis, entende a Cascade em profundidade para evitar especificidade wars, e domina técnicas modernas como clamp(), container queries e @layer que mudaram completamente como CSS é escrito.

📱

Mobile-first obrigatório

Qualquer produto SaaS, e-commerce ou app precisa funcionar perfeitamente em qualquer tela. Media queries são a base.

🎨

Design Systems escaláveis

CSS Custom Properties permitem temas dinâmicos, dark mode e consistência visual em projetos grandes.

⚡

CSS moderno elimina JS

Container queries, :has(), clamp() eliminam lógica que antes precisava de JavaScript.

🔥

Cascade = poder e armadilha

Entender a cascade evita anos de dor com !important e especificidade guerras em projetos reais.

Pilar 1: Responsividade — Mobile First

A estratégia mobile-first significa escrever o CSS base para telas pequenas e usar min-width para expandir conforme a tela cresce. É o oposto do legado desktop-first com max-width.

< 640px
mobile
640–768
sm
768–1024
md
1024–1280
lg
> 1280px
xl
❌ Desktop-First (legado)
/* Começa no desktop */
.container { width: 1200px; }

/* "Conserta" o mobile depois */
@media (max-width: 768px) {
  .container { width: 100%; }
}
@media (max-width: 480px) {
  .container { padding: 8px; }
}
✅ Mobile-First (moderno)
/* Base = mobile */
.container { 
  width: 100%;
  padding: 16px;
}

/* Expande progressivamente */
@media (min-width: 768px) {
  .container { max-width: 720px; }
}
@media (min-width: 1024px) {
  .container { max-width: 1200px; }
}

Fluid Typography com clamp()

A função clamp(min, preferred, max) cria valores que escalam fluidamente entre um mínimo e um máximo, eliminando a necessidade de múltiplas media queries apenas para tipografia.

exemplo: tipografia fluida
/* Antes: media queries para cada tamanho */
h1 { font-size: 1.8rem; }
@media (min-width: 768px) { h1 { font-size: 2.4rem; } }
@media (min-width: 1200px) { h1 { font-size: 3rem; } }

/* Agora: uma linha só, escala suavemente */
h1 { font-size: clamp(1.8rem, 4vw + 1rem, 3rem); }

/* Como ler: mínimo 1.8rem, cresce com a viewport, máximo 3rem */
/* 4vw + 1rem = o valor "preferred" que cresce junto com a tela */
💡 Fórmula para clamp() responsivo

Use a fórmula clamp(min, {multiplier}vw + {offset}rem, max). O multiplier controla a velocidade de crescimento. Para títulos: 4–6vw. Para textos: 1–2vw.

Pilar 2: CSS Custom Properties (Variáveis CSS)

CSS Custom Properties são variáveis nativas do browser, declaradas com --nome e usadas com var(--nome). Diferente de variáveis SASS/LESS, elas existem em runtime e podem ser alteradas dinamicamente via JavaScript ou pela própria Cascade.

sistema de design com custom properties
/* ============================================
   Design System: tokens centralizados em :root
   ============================================ */
:root {
  /* Cores */
  --color-primary: #ef4444;
  --color-primary-dim: rgba(239, 68, 68, 0.12);
  --color-bg: #0a0a0f;
  --color-surface: #111118;
  --color-text: #e8e8f0;
  --color-muted: #8585a8;

  /* Tipografia */
  --font-body: 'Inter', sans-serif;
  --font-mono: 'JetBrains Mono', monospace;
  --text-sm: 0.875rem;
  --text-base: 1rem;
  --text-lg: 1.125rem;
  --text-xl: clamp(1.5rem, 3vw + 1rem, 2.5rem);

  /* Espaçamento */
  --space-sm: 8px;
  --space-md: 16px;
  --space-lg: 32px;
  --space-xl: 64px;

  /* Bordas */
  --radius-sm: 4px;
  --radius-md: 8px;
  --radius-lg: 12px;
  --border: 1px solid rgba(255, 255, 255, 0.08);
}

/* Dark mode automático via media query */
@media (prefers-color-scheme: dark) {
  :root {
    --color-bg: #0a0a0f;
    --color-text: #e8e8f0;
  }
}

/* Tema customizado via classe no elemento pai */
[data-theme="light"] {
  --color-bg: #ffffff;
  --color-text: #111118;
  --color-surface: #f5f5f5;
}

/* Uso: qualquer componente usa os tokens */
.button {
  background: var(--color-primary);
  padding: var(--space-sm) var(--space-md);
  border-radius: var(--radius-md);
  font-family: var(--font-body);
  font-size: var(--text-sm);
}

/* Escopo local: sobrescrita numa seção específica */
.sidebar {
  --color-primary: #3b82f6; /* azul só na sidebar */
}
/* Botões dentro da sidebar herdam automaticamente o azul */
🔥 Pro Tip: CSS Variables são herdadas pela Cascade

Ao contrário de variáveis SASS, CSS Custom Properties participam da herança. Um --color-primary definido num elemento pai é herdado por todos os filhos. Você pode sobrescrever localmente em qualquer escopo.

Pilar 3: A Cascade Avançada

A Cascade (cascata) é o algoritmo que o browser usa para determinar qual regra CSS prevalece quando há conflito. Em projetos grandes, não entender a cascade resulta em um caos de !important e especificidade wars.

ordem de prioridade (maior → menor)
1
Transições e Animações CSS
Temporariamente sobrepõem tudo
↓
2
!important do Usuário
Configurações de acessibilidade do browser/SO
↓
3
!important do Autor
Seu CSS com !important — evitar quando possível
↓
4
Regras do Autor (CSS normal)
Seu CSS sem !important — especificidade decide o vencedor
↓
5
Estilos padrão do Browser
User agent stylesheet — base que tudo herda

Especificidade — O Sistema de Pontuação

Quando duas regras do mesmo nível da cascade se conflitam, a especificidade decide qual vence. É calculada como três números separados: (A, B, C).

Seletor Cálculo (A, B, C) Peso Total Exemplo
Inline style (1,0,0,0) 1000 style="color:red"
ID (0,1,0,0) 100 #header
Classe / Atributo / Pseudo-classe (0,0,1,0) 10 .nav, [type], :hover
Elemento / Pseudo-elemento (0,0,0,1) 1 div, ::before
Universal / Combinadores (0,0,0,0) 0 *, +, ~, >
calculando especificidade
/* Especificidade: (0,0,1,1) = 11 */
.nav a { color: blue; }

/* Especificidade: (0,0,2,0) = 20 — VENCE */
.nav .link { color: red; }

/* Especificidade: (0,0,0,2) = 2 */
ul li { color: green; }

/* Especificidade: (0,1,0,0) = 100 — VENCE TUDO acima */
#main { color: purple; }

/* ⚠️ Não use IDs no CSS — cria problemas de especificidade */
/* Use classes duplas para aumentar especificidade quando necessário */
.button.button { /* (0,0,2,0) = 20 */ }
.button:not(.disabled) { /* (0,0,2,0) = 20 */ }

@layer — O Futuro da Cascade

@layer é uma feature moderna (2022+) que permite criar camadas de cascade explícitas. Você define a ordem de prioridade no topo do arquivo e qualquer regra numa camada de menor prioridade nunca vence uma de maior prioridade — independente da especificidade.

@layer para design systems profissionais
/* Define a ordem das camadas (primeiro = menor prioridade) */
@layer reset, base, components, utilities;

/* Reset: lowest priority */
@layer reset {
  *, *::before, *::after { box-sizing: border-box; }
  body { margin: 0; }
}

/* Base: estilos fundamentais */
@layer base {
  a { color: var(--color-primary); }
  h1 { font-size: var(--text-xl); }
}

/* Components: blocos reutilizáveis */
@layer components {
  .button {
    padding: var(--space-sm) var(--space-md);
    border-radius: var(--radius-md);
    /* Aqui posso usar seletores simples sem medo: */
    /* As utilities sempre vão vencer mesmo com menor especificidade */
  }
}

/* Utilities: highest priority — sempre vencem */
@layer utilities {
  .hidden { display: none; }
  .text-center { text-align: center; }
  .mt-8 { margin-top: 8px; }
}

/* Regras fora de @layer têm prioridade máxima (cuidado!) */

Container Queries — Responsividade Contextual

Media queries respondem ao tamanho da viewport. Container queries respondem ao tamanho do container pai. Isso resolve o problema clássico de componentes que precisam se adaptar dependendo de onde são usados.

container queries em ação
/* 1. Defina o container */
.card-wrapper {
  container-type: inline-size;
  container-name: card; /* opcional, mas recomendado */
}

/* 2. Escreva queries baseadas no container, não na viewport */
.card {
  display: flex;
  flex-direction: column; /* padrão: vertical (container pequeno) */
  gap: 12px;
}

/* Quando o CONTAINER (não a viewport) for maior que 400px */
@container card (min-width: 400px) {
  .card {
    flex-direction: row; /* horizontal quando há espaço */
    align-items: center;
  }

  .card-image {
    width: 120px;
    flex-shrink: 0;
  }
}

/* O mesmo componente .card funciona diferente em cada contexto:
   - Na sidebar (container estreito): layout vertical
   - No grid principal (container largo): layout horizontal
   Sem duplicar código ou usar classes diferentes! */
💡 Container Queries vs Media Queries

Use media queries para layout global (header, grid, sidebar). Use container queries para componentes isolados que precisam se adaptar ao contexto onde são inseridos (cards, widgets, listas).

⚠️ Pitfalls — Erros Frequentes

❌ Esquecer a meta viewport

Sem <meta name="viewport" content="width=device-width, initial-scale=1.0">, nenhuma media query funciona no mobile. O browser assume que a página tem 980px de largura e "encolhe" para caber na tela.

❌ Usar px fixo para tudo

Usar font-size: 14px ignora a preferência de tamanho do usuário no SO. Use rem para textos (relativo ao root), em para espaçamento interno de componentes, e px apenas para bordas finas (1px, 2px).

❌ !important como solução padrão

Cada !important que você usa gera a necessidade de mais !important no futuro. É um ciclo vicioso. Use @layer ou reestruture a especificidade em vez de usar !important. 

/* ❌ Ruim: cascata de !important */
.card { color: red !important; }
.card.featured { color: blue !important; } /* não funciona! */

/* ✅ Bom: especificidade correta */
.card { color: red; }
.card.featured { color: blue; } /* vence por especificidade */
❌ CSS Custom Properties sem fallback

Em browsers antigos ou quando a variável não está definida, o valor é inválido. Sempre forneça um fallback no segundo argumento de var().

/* ❌ Quebra se --primary não estiver definida */
color: var(--primary);

/* ✅ Com fallback seguro */
color: var(--primary, #ef4444);
❌ Confundir container-type com display

container-type: inline-size cria um contexto de containment mas não muda o display do elemento. Adicione ao wrapper, não diretamente no elemento que você quer estilizar.

Glossário da Aula

Breakpoint
Largura de tela específica onde o layout muda. Ex: 768px é o breakpoint padrão entre mobile e tablet na maioria dos frameworks.
Mobile-First
Estratégia de desenvolvimento onde o CSS base é escrito para mobile e incrementado com min-width media queries para telas maiores.
CSS Custom Property (variável CSS)
Variável nativa do CSS declarada com --nome: valor e usada com var(--nome). Participa da cascade e herança, diferente de variáveis SASS.
Cascade
Algoritmo do browser que determina qual regra CSS prevalece quando há conflito, baseado em: origem + importância → camada @layer → especificidade → ordem de declaração.
Especificidade
Sistema de pontuação que determina qual seletor CSS "vence" um conflito. Calculado como (inline, IDs, classes, elementos) — ex: #main .card a = (0,1,1,1).
@layer
Feature CSS que cria camadas de cascade explícitas. Regras numa camada de maior prioridade vencem as de menor, independente de especificidade.
Container Query
Query que dispara baseada no tamanho do container pai do elemento, não da viewport. Usa container-type no pai e @container nas regras.
clamp(min, preferred, max)
Função CSS que retorna o valor "preferred" clampado entre "min" e "max". Ideal para tipografia fluida: clamp(1rem, 2.5vw + 0.5rem, 2rem).
Fluid Typography
Técnica onde os tamanhos de fonte crescem suavemente com a viewport usando clamp() ou vw, sem "saltos" em breakpoints.
