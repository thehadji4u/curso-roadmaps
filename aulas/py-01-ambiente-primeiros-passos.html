Por que isso importa?

Python se tornou a linguagem mais popular do mundo por razões concretas: sintaxe limpa e legível, ecossistema vasto (ciência de dados, IA, web, automação) e curva de aprendizado suave que não sacrifica poder. Hoje Python roda os modelos de linguagem mais avançados do planeta, os pipelines de dados de grandes empresas e os backends de serviços de alto tráfego.

Antes de escrever qualquer linha de código Python, precisamos de um ambiente funcionando corretamente. Isso significa a versão certa do Python, isolamento de dependências com ambientes virtuais, e um editor configurado. Esses detalhes fazem toda a diferença entre um setup profissional e horas perdidas com conflitos de pacotes.

💼 Por que um engenheiro sênior se preocupa tanto com o ambiente
"Funciona na minha máquina" é uma das frases mais temidas em times de engenharia. Ambientes virtuais (venv), arquivos requirements.txt e ferramentas como pyenv existem exatamente para eliminar esse problema. Um setup profissional é reproduzível, isolado e documentado — qualquer colega pode clonar o repo e rodar o projeto em minutos.
// analogia

Antes do código: uma analogia

🌍 Mundo Real → Tecnologia
Imagine um chef de cozinha profissional. Antes de cozinhar, ele não usa uma cozinha compartilhada bagunçada — ele tem a própria bancada organizada (ambiente virtual), com seus utensílios específicos (pacotes instalados), em versões que ele sabe que funcionam para as suas receitas. Se outro chef usa a mesma cozinha mas com diferentes utensílios, eles não se atrapalham.

O Python instalado no sistema é a cozinha do prédio. Cada projeto com seu venv é como ter a própria bancada isolada. O pip é o sistema de delivery que traz os ingredientes (pacotes). O requirements.txt é a lista de compras que qualquer pessoa pode usar para montar a mesma bancada em qualquer lugar.
🏠 Cozinha do prédio (compartilhada) → Python do sistema 🍳 Bancada própria isolada → Ambiente virtual (venv) 🚚 Delivery de ingredientes → pip install pacote 🛒 Lista de compras da receita → requirements.txt

Ambientes Virtuais com venv

Um ambiente virtual cria uma cópia isolada do Python com seu próprio conjunto de pacotes instalados. Isso garante que projeto A usando requests==2.28 não conflite com projeto B usando requests==2.31.

⚠️ Regra de ouro: SEMPRE use venv
Nunca instale pacotes no Python do sistema com pip install global. Sempre crie um venv por projeto. Este é o primeiro hábito que diferencia um dev profissional de um iniciante.
Terminal — fluxo completo de um projeto
$ # 1. Criar pasta do projeto e entrar nela
$ mkdir meu-projeto && cd meu-projeto

$ # 2. Criar o ambiente virtual (pasta .venv dentro do projeto)
$ python3 -m venv .venv
# Cria: .venv/bin/python, .venv/bin/pip, etc.

$ # 3. ATIVAR o ambiente virtual
$ source .venv/bin/activate # macOS/Linux
> .venv\Scripts\activate # Windows PowerShell

# O prompt muda — sinal de que o venv está ativo:
(.venv) $ which python
/home/user/meu-projeto/.venv/bin/python

(.venv) $ # 4. Instalar pacotes — vão para o .venv, não o sistema!
(.venv) $ pip install requests pandas

(.venv) $ # 5. Salvar dependências (compartilhar com o time)
(.venv) $ pip freeze > requirements.txt

(.venv) $ # Para desativar quando terminar
(.venv) $ deactivate
$ # Prompt volta ao normal

$ # Outro dev clona o repo e recria o ambiente:
$ python3 -m venv .venv && source .venv/bin/activate
(.venv) $ pip install -r requirements.txt
Successfully installed requests-2.31.0 pandas-2.1.0 ...
🔥 Pro tip: .gitignore o .venv
Nunca commite a pasta .venv no Git — ela contém binários específicos do sistema e pode ter centenas de MB. Adicione .venv/ ao seu .gitignore. O que vai para o repo é apenas o requirements.txt.

pip — gerenciando pacotes

// comandos pip essenciais
# Instalar um pacote
pip install requests

# Instalar versão específica
pip install requests==2.31.0

# Instalar versão mínima
pip install "requests>=2.28,<3.0"

# Instalar múltiplos pacotes de uma vez
pip install requests pandas numpy matplotlib

# Atualizar pacote
pip install --upgrade requests

# Remover pacote
pip uninstall requests

# Ver pacotes instalados
pip list

# Ver informações de um pacote
pip show requests

# Salvar dependências exatas
pip freeze > requirements.txt

# Instalar a partir do arquivo
pip install -r requirements.txt

# Buscar pacotes disponíveis
pip index versions requests  # ver todas as versões disponíveis

Tipos & Variáveis em Python

Python é dinamicamente tipado (o tipo é inferido em runtime) mas também fortemente tipado (não converte automaticamente entre tipos incompatíveis). Isso é diferente do JavaScript — Python não vai somar uma string com um número silenciosamente.

int
42 · -7 · 1_000_000
Inteiro de precisão arbitrária. Underscores para legibilidade.
float
3.14 · -0.5 · 1.5e10
Ponto flutuante de 64 bits (IEEE 754). Cuidado com imprecisão!
str
"olá" · 'mundo' · """multi"""
String imutável. Aspas simples ou duplas (escolha uma e seja consistente).
bool
True · False
Subclasse de int — True == 1, False == 0. Maiúscula obrigatória!
NoneType
None
Ausência de valor. Equivalente ao null de outras linguagens.
complex
3 + 4j
Número complexo nativo. Usado em computação científica.
// variáveis, tipagem e operações
# ─── Declaração de variáveis (sem palavra-chave!) ────────────
nome = "Ana Silva"
idade = 28
altura = 1.72
ativo = True
cargo = None   # ainda não definido

# ─── Convenções de nomenclatura (PEP 8) ──────────────────────
nome_completo = "snake_case"      # variáveis e funções: snake_case
TAXA_IMPOSTO = 0.15               # constantes: UPPER_SNAKE_CASE
class MinhaClasse: pass           # classes: PascalCase
_privado = "convencão"            # privado por convenção (não é enforçado)

# ─── Type hints (Python 3.5+) — documente sem sacrificar dinamismo ──
nome: str = "Ana"
idade: int = 28
preco: float = 99.90
ativo: bool = True
dados: list[str] = ["a", "b"]
config: dict[str, int] = {"timeout": 30}
resultado: str | None = None      # Python 3.10+ — Union com |

# ─── Verificação de tipo em runtime ─────────────────────────
type(nome)          # 
isinstance(idade, int)    # True
isinstance(idade, (int, float))  # True — verifica múltiplos tipos

# ─── Conversão explícita (casting) ───────────────────────────
int("42")           # 42
float("3.14")       # 3.14
str(100)            # "100"
bool(0)             # False
bool("")            # False — strings vazias são falsy
bool("qualquer")    # True  — strings não vazias são truthy
list("abc")         # ['a', 'b', 'c']

# ─── Operadores ───────────────────────────────────────────────
10 + 3    # 13   (adição)
10 - 3    # 7    (subtração)
10 * 3    # 30   (multiplicação)
10 / 3    # 3.333... (divisão SEMPRE retorna float)
10 // 3   # 3    (divisão inteira — floor division)
10 % 3    # 1    (módulo — resto da divisão)
10 ** 3   # 1000 (exponenciação)

# ─── Operadores de comparação ─────────────────────────────────
5 == 5     # True
5 != 4     # True
5 > 3      # True
5 >= 5     # True
5 is None  # False — 'is' compara identidade, não igualdade!
None is None  # True

# ─── Valores truthy e falsy ───────────────────────────────────
# Falsy: None, False, 0, 0.0, "", [], {}, set()
# Todo o resto é truthy

if []:
    print("nunca executa")  # lista vazia é falsy

if [0]:
    print("executa!")       # lista com elemento (mesmo que 0) é truthy

# ─── Múltipla atribuição e swap ───────────────────────────────
a, b, c = 1, 2, 3           # unpacking
a, b = b, a                 # swap sem variável temporária
primeiro, *resto = [1, 2, 3, 4, 5]  # starred assignment
# primeiro = 1, resto = [2, 3, 4, 5]

Strings & Formatação

Strings em Python são objetos imutáveis com um arsenal de métodos embutidos. A forma moderna e preferida de formatação é a f-string — introduzida no Python 3.6 e adotada universalmente por ser a mais legível e performática.

❌ Formas antigas
# Concatenação (verbosa)
msg = "Olá, " + nome + "!"

# % formatting (estilo C)
msg = "Olá, %s! Você tem %d anos." % (nome, idade)

# .format() (mais legível, mas verboso)
msg = "Olá, {}! Você tem {} anos.".format(nome, idade)
✅ f-strings (use sempre)
# f-string — prefixo f antes das aspas
msg = f"Olá, {nome}! Você tem {idade} anos."

# Expressões completas dentro das {}
msg = f"Preço: R$ {preco * 1.1:.2f}"
msg = f"{'ATIVO' if ativo else 'INATIVO'}"
msg = f"Hoje: {datetime.now():%d/%m/%Y}"
// strings — tudo que você precisa saber
# ─── Criação ──────────────────────────────────────────────────
s1 = 'aspas simples'
s2 = "aspas duplas"                    # equivalentes
s3 = """string
multi-linha"""                          # preserva quebras de linha
s4 = r"C:\Users\ana\Desktop"           # raw string — ignora escape sequences
s5 = b"bytes"                          # bytes literal

# ─── f-strings com formatação de números ─────────────────────
pi = 3.14159265
preco = 1500000.50
pct = 0.856

f"{pi:.2f}"          # "3.14"       — 2 casas decimais
f"{pi:10.3f}"        # "    3.142"  — largura 10, 3 decimais
f"{preco:,.2f}"      # "1,500,000.50" — separador de milhares
f"{pct:.1%}"         # "85.6%"      — porcentagem
f"{42:05d}"          # "00042"      — preenche com zeros
f"{42:b}"            # "101010"     — binário
f"{42:#x}"           # "0x2a"       — hexadecimal com prefixo
f"{'center':^20}"    # "     center      " — centralizado em 20 chars
f"{'left':<20}"      # "left                " — alinhado à esquerda

# ─── Métodos essenciais de string ────────────────────────────
texto = "  Olá, Mundo!  "

texto.strip()               # "Olá, Mundo!" — remove espaços dos lados
texto.lstrip()              # "Olá, Mundo!  " — só da esquerda
texto.rstrip()              # "  Olá, Mundo!" — só da direita
texto.lower()               # "  olá, mundo!  "
texto.upper()               # "  OLÁ, MUNDO!  "
texto.title()               # "  Olá, Mundo!  "
texto.capitalize()          # "  olá, mundo!  " → "  Olá, mundo!  "
texto.replace("Mundo", "Python")  # "  Olá, Python!  "
texto.count("o")            # 2 (case-sensitive)
texto.find("Mundo")         # 6 (índice) ou -1 se não encontrar
texto.startswith("  Olá")  # True
texto.endswith("!  ")       # True

# Split e join — operações essenciais
csv_linha = "Ana,28,Engenheira,São Paulo"
campos = csv_linha.split(",")
# ['Ana', '28', 'Engenheira', 'São Paulo']

palavras = ["Python", "é", "incrível"]
" ".join(palavras)    # "Python é incrível"
", ".join(palavras)   # "Python, é, incrível"

# ─── Slicing — fatiamento ─────────────────────────────────────
s = "Python"
#    0 1 2 3 4 5   (índices positivos)
#   -6-5-4-3-2-1   (índices negativos)

s[0]      # 'P'
s[-1]     # 'n'
s[1:4]    # 'yth'    — do índice 1 até 3 (4 exclusivo)
s[:3]     # 'Pyt'    — do início até 3
s[3:]     # 'hon'    — do 3 até o fim
s[::2]    # 'Pto'    — passo 2
s[::-1]   # 'nohtyP' — string invertida!

# ─── Verificações ─────────────────────────────────────────────
"py" in "python"         # True
"123".isdigit()          # True
"abc".isalpha()          # True
"  ".isspace()           # True
"Ana Silva".split()      # ['Ana', 'Silva']
len("Python")            # 6

mensagem = (

Crie analisador.py
"""
Analisador de Texto — demonstra tipos, variáveis e strings em Python.
"""

# ─── Dados de entrada ────────────────────────────────────────────
texto: str = """
Python é uma linguagem de programação de alto nível, interpretada e de
propósito geral. Foi criada por Guido van Rossum e lançada em 1991.
Python enfatiza a legibilidade do código e permite que programadores
expressem conceitos em menos linhas de código do que outras linguagens.
"""

autor: str = "Análise automática"
versao: float = 1.0
exportar: bool = True

# ─── Processamento — usando métodos de string ─────────────────────
texto_limpo: str = texto.strip()
palavras: list[str] = texto_limpo.split()
caracteres: int = len(texto_limpo)
linhas: list[str] = texto_limpo.split("\n")
sentencas: list[str] = texto_limpo.replace("?", ".").replace("!", ".").split(".")
sentencas = [s.strip() for s in sentencas if s.strip()]  # lista por compreensão!

total_palavras: int = len(palavras)
total_caracteres: int = sum(len(p) for p in palavras)
media_caracteres: float = total_caracteres / total_palavras if total_palavras else 0

# Palavra mais longa
palavra_mais_longa: str = max(palavras, key=len)

# Frequência de palavras (case-insensitive)
frequencia: dict[str, int] = {}
for palavra in palavras:
    chave = palavra.lower().strip(".,!?;:")  # remove pontuação
    frequencia[chave] = frequencia.get(chave, 0) + 1

top_5 = sorted(frequencia.items(), key=lambda x: x[1], reverse=True)[:5]

# ─── Relatório com f-strings ──────────────────────────────────────
print("=" * 55)
print(f"{'RELATÓRIO DE ANÁLISE DE TEXTO':^55}")
print("=" * 55)
print(f"  Autor    : {autor}")
print(f"  Versão   : {versao:.1f}")
print(f"  Exportar : {'✅ Sim' if exportar else '❌ Não'}")
print("-" * 55)
print(f"  Palavras        : {total_palavras:>8,}")
print(f"  Caracteres      : {caracteres:>8,}")
print(f"  Sentenças       : {len(sentencas):>8,}")
print(f"  Média chars/pal : {media_caracteres:>8.1f}")
print(f"  Palavra mais longa: {palavra_mais_longa!r} ({len(palavra_mais_longa)} chars)")
print("-" * 55)
print(f"  TOP 5 PALAVRAS MAIS FREQUENTES:")
for i, (palavra, freq) in enumerate(top_5, start=1):
    barra = "█" * freq
    print(f"    {i}. {palavra:<20} {freq:>3}x  {barra}")
print("=" * 55)

# ─── Verificações de tipo ─────────────────────────────────────────
print(f"\n  type(texto)    : {type(texto)}")
print(f"  type(versao)   : {type(versao)}")
print(f"  type(palavras) : {type(palavras)}")
print(f"  type(frequencia): {type(frequencia)}")

⚠️ Armadilhas Comuns

1. Instalar pacotes sem ativar o venv

Se você rodar pip install sem o venv ativo, o pacote vai para o Python do sistema e pode causar conflitos. Sempre verifique se o prompt mostra (.venv) antes de instalar qualquer coisa.

2. Usar is para comparar valores (não identidade)

== compara valor (conteúdo). is compara identidade (se são o mesmo objeto na memória). Use == para comparar strings, números e coleções. Use is apenas para checar None, True e False.

a = "python"
b = "python"
a == b   # True  ✅
a is b   # True  (por otimização do Python — mas não confie nisso!)

x = [1, 2, 3]
y = [1, 2, 3]
x == y   # True  ✅ — mesmo conteúdo
x is y   # False ✅ — objetos diferentes na memória
3. Imprecisão de float

Floats em Python (como em toda linguagem com IEEE 754) têm imprecisão. Nunca use floats para cálculos monetários!

0.1 + 0.2          # 0.30000000000000004 — surpresa!

# Soluções:
from decimal import Decimal
Decimal("0.1") + Decimal("0.2")  # Decimal('0.3') ✅

round(0.1 + 0.2, 2)  # 0.3 — aceitável para display
4. Strings são imutáveis — não podem ser modificadas in-place

Métodos de string retornam uma NOVA string — não modificam a original. Se você não capturar o retorno, a operação é perdida.

nome = "ana silva"
nome.title()          # retorna "Ana Silva" mas não muda 'nome'
print(nome)           # "ana silva" — ainda minúsculo!

nome = nome.title()   # ✅ captura o resultado
print(nome)           # "Ana Silva"
5. Misturar Python 2 e Python 3

Em alguns sistemas, python ainda aponta para Python 2. Sempre use python3 explicitamente (ou configure o alias). Verifique com python --version e python3 --version.

Ambiente Python isolado por projeto, com seu próprio interpretador e pacotes independentes do sistema.
pip
Gerenciador de pacotes padrão do Python. "Pip Installs Packages" — busca e instala pacotes do PyPI (Python Package Index).
requirements.txt
Arquivo de texto com a lista de pacotes e versões exatas de um projeto. Gerado por pip freeze, reproduzido com pip install -r.
Type Hints
Anotações opcionais de tipo introduzidas no Python 3.5. Não são enforçadas em runtime, mas habilitam análise estática com Pylance/mypy.
f-string
String literal prefixada com f ou F que permite embutir expressões Python dentro de {}. Introduzida no Python 3.6.
REPL
Read-Eval-Print Loop — modo interativo do Python ativado com o comando python3. Executa código linha a linha imediatamente.
PEP 8
Guia de estilo oficial do Python. Define convenções de nomenclatura, indentação (4 espaços), comprimento de linha (79-99 chars) e muito mais.
Slicing
Sintaxe seq[start:stop:step] para extrair partes de strings, listas e outros iteráveis. O índice stop é sempre exclusivo.

🏁 Checkpoint — Desafio Final

Crie um script Python chamado gerador_relatorio.py que:

  • Defina um dicionário produto com ao menos 6 campos: nome (str), preco (float), estoque (int), categoria (str), ativo (bool) e desconto (float entre 0 e 1)
  • Calcule e armazene em variáveis com type hints: preço com desconto, margem de lucro (assuma custo = 60% do preço original), e classificação do estoque ("Crítico" < 5, "Baixo" < 20, "Normal" < 100, "Alto" ≥ 100)
  • Imprima um relatório formatado usando f-strings com: título centralizado em 50 chars, todos os valores monetários com 2 casas decimais e separador de milhar, percentuais com 1 casa e símbolo %, status do produto (ativo/inativo) com emoji, e a barra de estoque visual com █ proporcional (máx 20 blocos)
  • Ao final, use type() para imprimir o tipo Python de cada campo do dicionário
💬 Dica: para a barra de estoque, calcule blocos = min(20, estoque // 5) e faça "█" * blocos + "░" * (20 - blocos). Para classificação, use um dicionário de thresholds e itere sobre ele, ou use uma cadeia de if/elif.
