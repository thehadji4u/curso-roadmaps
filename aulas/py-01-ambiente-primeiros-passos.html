// 01 — motivação

Por que isso importa?

Python é a linguagem mais usada em ciência de dados, automação, back-end e inteligência artificial. A sintaxe limpa, o ecossistema gigante de bibliotecas e a facilidade de leitura tornam Python a melhor escolha para quem quer escrever código que outros humanos consigam entender.

Mas antes de escrever qualquer script útil, é preciso entender três coisas: como configurar o ambiente isolado para cada projeto (venv), quais são os tipos primitivos da linguagem (int, float, str, bool, None) e como formatar e manipular texto com f-strings e métodos de string. Esses são os tijolos de tudo que virá depois.

💼 Por que um engenheiro sênior usa isso hoje
Sêniors nunca instalam bibliotecas no Python global do sistema. Cada projeto tem seu próprio venv com dependências declaradas em requirements.txt ou pyproject.toml. Isso garante reprodutibilidade: qualquer colega (ou servidor de CI) consegue recriar o ambiente exato com um único comando.
// 02 — analogia

Antes do código: uma analogia

🌍 Mundo Real → Tecnologia
Imagine que você é um chef de cozinha. Você tem uma cozinha industrial compartilhada no restaurante — mas cada receita exige ingredientes específicos e versões diferentes de equipamentos. Se você usar a cozinha geral para tudo, os ingredientes de um prato vão contaminar outro.

A solução profissional é ter estações de trabalho individuais e isoladas — uma para confeitaria, outra para grelhados, outra para pratos frios. Cada estação tem exatamente o que precisa, sem interferir nas outras.

No Python, essa "estação isolada" se chama virtualenv (venv). O Python global do sistema operacional é a cozinha industrial — você nunca mexe nela diretamente. Para cada projeto, você cria um venv limpo.
Cozinha industrial global → Python do sistema (nunca toque) Estação de trabalho isolada → venv do projeto Lista de ingredientes da receita → requirements.txt Tipo de ingrediente (líquido, sólido…) → tipo de dado (int, str…)
// 03 — instalação & ambiente

Python 3, pip e venv

// 04 — tipos & variáveis

Tipos primitivos e tipagem dinâmica

Python é dinamicamente tipado: a variável não tem tipo — o objeto que ela referencia tem. Isso significa que x = 10 cria um objeto int e aponta x para ele. Em seguida, x = "hello" aponta x para um objeto str diferente. O tipo anterior simplesmente deixa de ser referenciado.

int
inteiro
42 · -7 · 0
float
ponto flutuante
3.14 · -0.5 · 1e10
str
string / texto
"olá" · 'mundo'
bool
booleano
True · False
None
ausência de valor
None (único valor)
📄 tipos.py Python 
# ─── Tipagem dinâmica: a variável aponta para um objeto ───────
x = 42             # x aponta para um int
print(type(x))     # <class 'int'>

x = 3.14           # agora aponta para um float
print(type(x))     # <class 'float'>

x = "Python"       # agora aponta para uma str
print(type(x))     # <class 'str'>

# ─── int ──────────────────────────────────────────────────────
idade = 25
ano = 2024
resultado = idade + ano          # 2049
grande = 1_000_000               # underscores para legibilidade

# Conversão (casting)
texto_numero = "42"
numero = int(texto_numero)       # "42" → 42
pi_inteiro = int(3.99)           # trunca (não arredonda): 3

# ─── float ────────────────────────────────────────────────────
preco = 29.90
taxa = 0.1
total = preco * (1 + taxa)       # 32.89

# ⚠️ Cuidado: ponto flutuante não é exato
print(0.1 + 0.2)                 # 0.30000000000000004
# Use Decimal para valores financeiros:
from decimal import Decimal
print(Decimal("0.1") + Decimal("0.2"))  # 0.3

# ─── bool ─────────────────────────────────────────────────────
ativo = True
desativado = False

# Em Python, bool é subclasse de int
print(int(True))   # 1
print(int(False))  # 0
print(True + True) # 2 (!!)

# Valores "falsy" (avaliam como False):
# False, 0, 0.0, "", [], {}, (), None
# Tudo o mais é "truthy"
if []:             # lista vazia → False
    print("nunca")
if [0]:            # lista com um elemento → True
    print("sempre")

# ─── None ─────────────────────────────────────────────────────
resultado = None   # variável sem valor definido ainda
print(resultado is None)  # True — use 'is', não '=='

# Função que não retorna nada explicitamente
def sem_retorno():
    pass

print(sem_retorno())  # None
🔥 Pro tip — type hints (desde Python 3.5+)
Python moderno usa anotações de tipo para documentar intenção e habilitar checagem estática com mypy ou Pylance. Não é obrigatório, mas é padrão em código profissional:

nome: str = "Ana"
idade: int = 30
preco: float = 9.99
ativo: bool = True
resultado: int | None = None  ← union type — pode ser int ou None
// 05 — strings & formatação

Strings — f-strings, métodos e slicing

Strings em Python são imutáveis e indexadas. Cada caractere tem uma posição (índice) começando do zero. Isso permite acessar partes específicas com a notação de slicing: texto[início:fim:passo].

Formas de criar strings

📄 strings_basico.py Python 
# Aspas simples ou duplas — equivalentes
nome = 'Ana'
sobrenome = "Silva"

# Triple quotes: strings multilinhas
descricao = """
Engenheira de Software
São Paulo, SP
"""

# ─── Concatenação vs f-string ─────────────────────────────────
# ❌ Evite concatenação com +
mensagem_ruim = "Olá, " + nome + "! Você tem " + str(30) + " anos."

# ✅ Prefira f-string (desde Python 3.6)
mensagem = f"Olá, {nome}! Você tem {30} anos."

# f-strings aceitam expressões inteiras
preco = 49.9
print(f"Preço: R$ {preco:.2f}")          # Preço: R$ 49.90
print(f"Dobro: R$ {preco * 2:.2f}")      # Dobro: R$ 99.80
print(f"{'centralizado':^20}")            # texto centralizado em 20 chars
print(f"{1_000_000:,}")                   # 1,000,000 (separador de milhar)

# Diagnóstico rápido com f-string (Python 3.8+)
x = 42
print(f"{x = }")   # x = 42 (útil para debug)

Slicing — fatiando strings

📄 slicing.py Python 
texto = "Python 3.12"
#        0123456789...
#       -11-10-9...  (índices negativos)

# Sintaxe: texto[início:fim:passo]
# fim é EXCLUSIVO — texto[0:6] pega índices 0,1,2,3,4,5

print(texto[0])        # 'P'          — primeiro caractere
print(texto[-1])       # '2'          — último caractere
print(texto[0:6])      # 'Python'     — do 0 ao 5 (fim=6 excl.)
print(texto[7:])       # '3.12'       — do 7 até o fim
print(texto[:6])       # 'Python'     — do início até 5
print(texto[::2])      # 'Pto .2'     — de 2 em 2
print(texto[::-1])     # '21.3 nohtyP' — invertido

# Casos práticos de slicing
cpf = "123.456.789-00"
apenas_digitos = cpf.replace(".", "").replace("-", "")  # "12345678900"
primeiros_3 = cpf[:3]  # "123"

url = "https://exemplo.com/path"
sem_https = url[8:]    # "exemplo.com/path"

Métodos essenciais de string

Método O que faz Exemplo
.upper() / .lower() Converte maiúscula/minúscula "Ana".upper() → "ANA"
.strip() Remove espaços (e \n, \t) no início e fim " oi ".strip() → "oi"
.split(sep) Divide em lista pelo separador "a,b,c".split(",") → ["a","b","c"]
.join(lista) Une lista em string com separador ", ".join(["a","b"]) → "a, b"
.replace(old, new) Substitui todas as ocorrências "aabaa".replace("a","x") → "xxbxx"
.startswith() / .endswith() Verifica prefixo/sufixo "foto.png".endswith(".png") → True
.find(sub) Retorna índice da primeira ocorrência (ou -1) "abcabc".find("c") → 2
.count(sub) Conta quantas vezes sub aparece "banana".count("a") → 3
.zfill(width) Preenche à esquerda com zeros "42".zfill(5) → "00042"
.isdigit() True se todos os chars são dígitos "123".isdigit() → True
📄 strings_metodos.py Python 
# ─── Pipeline de limpeza de dados (padrão real) ───────────────
entrada_usuario = "  ana.silva@GMAIL.COM  "

email_limpo = (entrada_usuario
    .strip()           # remove espaços externos
    .lower()           # normaliza para minúsculo
)
print(email_limpo)     # "ana.silva@gmail.com"

# ─── Split e Join — transformar estrutura ────────────────────
csv_linha = "Ana,30,São Paulo,Engenheira"
campos = csv_linha.split(",")
# ['Ana', '30', 'São Paulo', 'Engenheira']

# Reconstruir como TSV (tab-separated)
tsv_linha = "\t".join(campos)

# ─── Verificações práticas ────────────────────────────────────
extensoes_permitidas = (".jpg", ".jpeg", ".png", ".webp")
arquivo = "foto_perfil.PNG"

if arquivo.lower().endswith(extensoes_permitidas):
    print("Arquivo de imagem válido")

# ─── Formatação de números com f-string ──────────────────────
populacao = 215_300_000
pib = 2_081_235_000_000.50

print(f"Brasil: {populacao:,} habitantes")     # 215,300,000
print(f"PIB: R$ {pib:,.2f}")                  # R$ 2,081,235,000,000.50
print(f"Crescimento: {0.0312:.1%}")            # 3.1%
⚠️ Strings são imutáveis — não se alteram no lugar
Todos os métodos de string retornam uma nova string. O original nunca muda. Um erro clássico:

texto = " olá "
texto.strip() # ← não faz nada! resultado é descartado
texto = texto.strip() # ← correto: reatribua

Crie main.py e rode com o Python do venv:
📄 main.pyPython
from rich import print as rprint
from rich.table import Table

# ─── Tipos primitivos em ação ──────────────────────────────
nome: str   = "Ana Silva"
idade: int  = 28
altura: float = 1.65
ativo: bool = True
cargo: str | None = None   # ainda não definido

# ─── f-strings ─────────────────────────────────────────────
rprint(f"[bold blue]Usuário:[/] {nome}")
rprint(f"Idade: [green]{idade}[/] anos | Altura: {altura:.2f}m")
rprint(f"Ativo: {'✅' if ativo else '❌'}")
rprint(f"Cargo: {cargo if cargo is not None else '[dim]não definido[/]'}")

# ─── Manipulação de strings ─────────────────────────────────
email_bruto = "  ANA.SILVA@EMPRESA.COM  "
email = email_bruto.strip().lower()
rprint(f"\nEmail normalizado: [cyan]{email}[/]")

usuario, dominio = email.split("@")
rprint(f"Usuário: {usuario} | Domínio: {dominio}")

# ─── Slicing ────────────────────────────────────────────────
rprint(f"\nPrimeiro nome: [yellow]{nome[:3]}[/]")   # "Ana"
rprint(f"Invertido: {nome[::-1]}")

# ─── Tabela com rich ────────────────────────────────────────
tabela = Table(title="Resumo do perfil")
tabela.add_column("Campo",  style="cyan")
tabela.add_column("Valor",  style="white")
tabela.add_column("Tipo",   style="dim")

campos = [
    ("nome",   nome,         type(nome).__name__),
    ("idade",  str(idade),   type(idade).__name__),
    ("altura", str(altura),  type(altura).__name__),
    ("ativo",  str(ativo),   type(ativo).__name__),
    ("cargo",  str(cargo),   type(cargo).__name__),
]

for campo, valor, tipo in campos:
    tabela.add_row(campo, valor, tipo)

rprint(tabela)
5
Execute e observe
terminalbash 
python main.py
# Você verá uma tabela colorida no terminal com os tipos
# e valores de cada variável

// 07 — armadilhas

Erros que todo mundo comete

⚠️ Usar pip install sem ativar o venv
Se você esquecer de ativar o venv e rodar pip install, a biblioteca vai para o Python global do sistema. Seu projeto vai funcionar na sua máquina, mas não em outro ambiente. Sempre verifique se o prompt mostra (.venv) antes de instalar.
⚠️ Comparar None com ==
Use sempre x is None, não x == None. A razão: == chama o método __eq__ do objeto, que pode ser sobrescrito. is compara identidade de objeto — mais seguro e mais idiomático em Python.
⚠️ Aritmética de ponto flutuante para dinheiro
0.1 + 0.2 != 0.3 em Python (e em qualquer linguagem que usa IEEE 754). Para valores financeiros, use sempre from decimal import Decimal e passe os valores como strings: Decimal("0.10") + Decimal("0.20").
⚠️ Strings mutáveis — armadilha clássica
nome = "ana"
nome.capitalize() # retorna "Ana" mas NÃO altera nome
print(nome) # ainda "ana"!

Correto: nome = nome.capitalize()

🏁 Checkpoint — valide seu aprendizado

Crie um script perfil.py que simula o cadastro e exibição de um perfil de usuário:

  • Configure um venv isolado, instale a biblioteca rich e salve as dependências em requirements.txt
