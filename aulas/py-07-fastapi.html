FastAPI — API REST do Zero

// motivação

Por que FastAPI mudou o back-end Python?

Durante anos, Flask e Django dominaram o back-end Python. Flask era minimalista mas sem tipagem; Django era completo mas pesado. Em 2018, Sebastián Ramírez lançou o FastAPI e reescreveu as regras: velocidade comparável ao Node.js e Go, validação automática de dados, documentação Swagger gerada sem esforço e suporte nativo a async/await.

Hoje, FastAPI é usado por empresas como Microsoft, Uber e Netflix em produção. Qualquer API moderna em Python — seja um microsserviço, um backend de aplicação ou um endpoint de IA — é candidata a ser construída com FastAPI.

💼 Por que um engenheiro sênior usa FastAPI hoje

A combinação de tipagem estática + validação automática via Pydantic elimina uma classe inteira de bugs de runtime. O Swagger gerado automaticamente serve como contrato vivo da API para o time de front-end. E async/await nativo permite lidar com dezenas de requisições simultâneas sem overhead de threads — essencial em APIs que consomem outros serviços ou bancos de dados assíncronos.

// conceito

Como o FastAPI funciona internamente

FastAPI é construído sobre dois pilares: Starlette (framework ASGI assíncrono) e Pydantic (validação de dados via type hints). Quando você define uma rota com anotações de tipo, o FastAPI automaticamente: valida a entrada, converte os tipos, gera a documentação e serializa a resposta.

Os três conceitos-chave

01

Path Operations

Decorators @app.get, @app.post, @app.put, @app.delete mapeiam URLs para funções Python. Cada função recebe parâmetros tipados do path, query string ou body.

02

Pydantic Models

Classes que herdam de BaseModel definem o schema dos dados. FastAPI usa essas classes para validar entrada, serializar saída e gerar a documentação OpenAPI automaticamente.

03

Dependency Injection

Sistema elegante para compartilhar lógica entre rotas: conexões com banco, usuário autenticado, configurações. Use Depends() para injetar dependências sem acoplamento.

04

Async por padrão

Handlers podem ser async def (recomendado para I/O) ou def (executado em thread pool). Nunca misture chamadas bloqueantes dentro de async def.

Mãos na massa — sua primeira API em 10 minutos

Vamos construir uma API de lista de tarefas (To-Do) do zero. Cada passo é executável no seu terminal.

1
Criar ambiente virtual e instalar dependências
Sempre use um venv para isolar as dependências do projeto.

$ python -m venv .venv
$ source .venv/bin/activate # Windows: .venv\Scripts\activate
$ pip install fastapi uvicorn[standard]

Criar o arquivo principal main.py
Uma API mínima funcional com uma rota GET.

from fastapi import FastAPI

app = FastAPI(
    title="Todo API",
    description="API de tarefas construída com FastAPI",
    version="1.0.0"

@app.get("/")
async def root():
    return {"message": "Todo API funcionando!"}

@app.get("/health")
async def health():
    return {"status": "ok"}

API completa com Pydantic, CRUD e Dependency Injection

Veja o padrão de uma API real: modelos Pydantic separados para criação/leitura, IDs gerados automaticamente, erros HTTP tratados com HTTPException e uma dependência compartilhada.

// main.py — API completa
from fastapi import FastAPI, HTTPException, Depends, status
from pydantic import BaseModel, Field
from typing import Optional
from datetime import datetime
import uuid

app = FastAPI(title="Todo API", version="1.0.0")

# ── MODELOS PYDANTIC ──────────────────────────────────────
# Schema de criação — só o que o usuário envia
class TodoCreate(BaseModel):
    title: str = Field(..., min_length=1, max_length=200,
                       description="Título da tarefa")
    description: Optional[str] = Field(None, max_length=1000)
    priority: int = Field(default=1, ge=1, le=5,
                          description="Prioridade de 1 a 5")

# Schema de atualização — todos opcionais
class TodoUpdate(BaseModel):
    title: Optional[str] = Field(None, min_length=1, max_length=200)
    description: Optional[str] = None
    priority: Optional[int] = Field(None, ge=1, le=5)
    completed: Optional[bool] = None

# Schema de resposta — o que a API retorna
class TodoResponse(BaseModel):
    id: str
    title: str
    description: Optional[str]
    priority: int
    completed: bool
    created_at: datetime

    class Config:
        # Permite criar a partir de objetos ORM futuramente
        from_attributes = True

# ── "BANCO DE DADOS" em memória (substituir por SQLAlchemy na Aula 27)
todos_db: dict[str, dict] = {}

# ── DEPENDENCY — simula uma verificação de API key ────────
def get_current_user(x_api_key: str = None):
    """
    Em produção, aqui ficaria autenticação JWT.
    Por ora, aceita qualquer chave para demonstrar o padrão.
    """
    return {"user_id": "demo-user", "role": "user"}

# ── ROTAS ─────────────────────────────────────────────────

@app.get("/todos", response_model=list[TodoResponse], tags=["todos"])
async def list_todos(
    completed: Optional[bool] = None,    # query param opcional
    priority: Optional[int] = None,       # filtro por prioridade
    user=Depends(get_current_user)        # dependency injection
):
    """Lista todas as tarefas com filtros opcionais."""
    items = list(todos_db.values())

    if completed is not None:
        items = [t for t in items if t["completed"] == completed]
    if priority is not None:
        items = [t for t in items if t["priority"] == priority]

    return items

@app.post("/todos", response_model=TodoResponse,
          status_code=status.HTTP_201_CREATED, tags=["todos"])
async def create_todo(todo: TodoCreate, user=Depends(get_current_user)):
    """Cria uma nova tarefa. Pydantic valida automaticamente os campos."""
    new_id = str(uuid.uuid4())
    todo_dict = {
        "id": new_id,
        **todo.model_dump(),   # converte Pydantic → dict
        "completed": False,
        "created_at": datetime.utcnow()
    }
    todos_db[new_id] = todo_dict
    return todo_dict

@app.get("/todos/{todo_id}", response_model=TodoResponse, tags=["todos"])
async def get_todo(todo_id: str):
    """Busca uma tarefa pelo ID. Retorna 404 se não encontrada."""
    if todo_id not in todos_db:
        raise HTTPException(
            status_code=status.HTTP_404_NOT_FOUND,
            detail=f"Tarefa '{todo_id}' não encontrada"
        )
    return todos_db[todo_id]

@app.patch("/todos/{todo_id}", response_model=TodoResponse, tags=["todos"])
async def update_todo(todo_id: str, updates: TodoUpdate):
    """Atualização parcial (PATCH). Só altera os campos enviados."""
    if todo_id not in todos_db:
        raise HTTPException(status_code=404, detail="Tarefa não encontrada")

    todo = todos_db[todo_id]
    # exclude_unset=True ignora campos não enviados
    update_data = updates.model_dump(exclude_unset=True)
    todo.update(update_data)
    todos_db[todo_id] = todo
    return todo

@app.delete("/todos/{todo_id}", status_code=status.HTTP_204_NO_CONTENT,
            tags=["todos"])
async def delete_todo(todo_id: str):
    """Remove uma tarefa. Retorna 204 No Content em caso de sucesso."""
    if todo_id not in todos_db:
        raise HTTPException(status_code=404, detail="Tarefa não encontrada")
    del todos_db[todo_id]
    # 204 não tem body — não retornar nada

Erros comuns e como evitar

⚠️ Misturar código bloqueante em async def

Chamar funções bloqueantes (como requests.get() ou operações síncronas de banco) dentro de async def trava o event loop inteiro. Use httpx (async) em vez de requests, e SQLAlchemy async em vez do síncrono.

# ❌ ERRADO — bloqueia o event loop
@app.get("/data")
async def get_data():
    response = requests.get("https://api.externa.com")  # bloqueante!
    return response.json()

# ✅ CORRETO — usa cliente async
import httpx
@app.get("/data")
async def get_data():
    async with httpx.AsyncClient() as client:
        response = await client.get("https://api.externa.com")
    return response.json()
⚠️ Reutilizar o mesmo modelo para criação e resposta

Sempre crie modelos separados: XCreate (entrada sem id/created_at), XResponse (saída completa). Nunca exponha campos como password ou hashed_password no modelo de resposta.

⚠️ Não usar response_model

Sem response_model, o FastAPI retorna todos os campos do dict/objeto — incluindo dados sensíveis. Sempre declare o modelo de resposta explicitamente para ter controle total do output.
