// motivação

JavaScript no servidor — por que Node.js importa

Antes do Node.js, JavaScript vivia confinado ao browser. Em 2009, Ryan Dahl mudou tudo ao usar o engine V8 do Chrome para criar um runtime JavaScript no servidor. A proposta era ousada: um único processo assíncrono que lida com milhares de conexões simultâneas sem criar threads para cada uma.

Hoje, Node.js roda em 85% das empresas listadas na Forbes 500 que usam JavaScript no back-end. Express — seu framework web mais popular — tem mais de 30 milhões de downloads semanais. E Fastify cresce rapidamente como alternativa de alta performance. Conhecer esse ecossistema é obrigatório para qualquer dev JavaScript moderno.

💼 Por que um engenheiro sênior usa Node.js hoje

Microsserviços leves, APIs de edge computing, BFFs (Backend for Frontend) e ferramentas de linha de comando são casos de uso naturais para Node.js. A vantagem de compartilhar código (types, validação, utilitários) entre front e back — possível só com JavaScript — é um diferencial real em times full-stack.

// analogia

Antes do código: uma analogia

🌍 Mundo Real → Tecnologia
Imagine um call center com um único atendente muito eficiente. Em vez de colocar uma pessoa para cada ligação (modelo threaded), esse atendente recebe uma ligação, diz "aguarde, vou verificar", coloca em espera, atende outra, volta para a primeira quando a resposta chegou. Nunca fica bloqueado esperando — sempre produtivo. Isso é o Event Loop do Node.js. O Express é o roteiro de atendimento: para cada tipo de chamada (rota), há um script pronto. O middleware é o checklist que todo atendente segue antes de transferir: verificou identidade? Logou o horário? Autorizou o acesso?
📞 Atendente único eficiente → Event Loop (single thread) 📋 Roteiro de atendimento → Express Router ✅ Checklist pré-atendimento → Middleware pipeline ⏸️ "Aguarde, verificando..." → async/await / callbacks
// conceito 1

Node.js — o runtime e seu Event Loop

Node.js usa um modelo de I/O não-bloqueante baseado em eventos. Operações lentas (disco, rede, banco de dados) não travam o processo — Node delega ao SO e continua executando outras tarefas. Quando a operação termina, o callback/promise é colocado na fila e processado.

Módulos nativos essenciais

fs

File System

Leitura e escrita de arquivos. Use sempre a versão fs/promises (async) — nunca fs.readFileSync em servidor.

http

HTTP Server

Cria servidores HTTP sem framework. Express é construído sobre ele. Raramente usado diretamente, mas importante entender.

path

Path

Manipulação de caminhos de arquivo cross-platform. Use path.join() — nunca concatene strings de path manualmente.

events

EventEmitter

Base do padrão Observer no Node.js. Streams, HTTP, WebSockets — todos herdam de EventEmitter. Entender isso desvenda o Node.

// conceito 2

Express — o framework minimalista

Express adiciona uma camada de abstração sobre o http nativo: roteamento declarativo, sistema de middleware e utilitários de resposta. É minimalista por design — você escolhe o que adicionar.

O pipeline de middleware

Toda requisição no Express passa por uma cadeia de funções middleware antes de chegar ao handler. Cada middleware pode processar, modificar, ou encerrar a requisição.

1
morgan — Logger
Loga cada requisição: método, URL, status, tempo. Imprescindível para debug em produção.
2
cors — Cross-Origin
Permite requisições de outros domínios (ex: front-end em outra porta/domínio). Sem isso, o browser bloqueia.
3
express.json() — Body Parser
Converte o body JSON da requisição em req.body. Sem ele, req.body é undefined.
4
Auth Middleware — JWT
Verifica o token Bearer, popula req.user. Aplicado seletivamente por rota.
5
Route Handler
Finalmente executa a lógica da rota e envia a resposta.
// server.js — setup completo Express
import express from 'express'
import cors from 'cors'
import morgan from 'morgan'
import dotenv from 'dotenv'
import { todosRouter } from './routes/todos.js'
import { authRouter } from './routes/auth.js'
import { errorHandler } from './middleware/errorHandler.js'

dotenv.config()  // carrega .env para process.env

const app = express()
const PORT = process.env.PORT || 3000

// ── MIDDLEWARE GLOBAL ──────────────────────────────────
app.use(morgan('dev'))                    // log de requisições
app.use(cors({ origin: process.env.CORS_ORIGIN }))
app.use(express.json({ limit: '10kb' })) // evita payloads gigantes

// ── ROTAS ─────────────────────────────────────────────
app.use('/api/todos', todosRouter)
app.use('/api/auth', authRouter)

// Rota de health check — usada por Docker/k8s
app.get('/health', (req, res) => {
  res.json({ status: 'ok', uptime: process.uptime() })
})

// ── ERROR HANDLER (deve ser o último middleware) ───────
app.use(errorHandler)

app.listen(PORT, () => {
  console.log(`🚀 Server running on http://localhost:${PORT}`)
})

CRUD completo com Express Router

Express Router permite modularizar as rotas em arquivos separados. Cada recurso tem seu próprio router, com handlers async envoltos em try/catch.

// routes/todos.js — Router com CRUD completo
import { Router } from 'express'
import { authenticate } from '../middleware/auth.js'
import { validate } from '../middleware/validate.js'
import { createTodoSchema, updateTodoSchema } from '../schemas/todo.js'

export const todosRouter = Router()

// Storage em memória (substituir por ORM depois)
const todos = new Map()

// GET /api/todos — lista com filtros
todosRouter.get('/', authenticate, async (req, res, next) => {
  try {
    const { completed, priority } = req.query
    let items = [...todos.values()]

    if (completed !== undefined)
      items = items.filter(t => t.completed === (completed === 'true'))
    if (priority)
      items = items.filter(t => t.priority === Number(priority))

    res.json({ data: items, total: items.length })
  } catch (err) {
    next(err)  // passa para o errorHandler
  }
})

// POST /api/todos — criação com validação
todosRouter.post('/', authenticate, validate(createTodoSchema), async (req, res, next) => {
  try {
    const todo = {
      id: crypto.randomUUID(),
      ...req.body,
      completed: false,
      ownerId: req.user.id,
      createdAt: new Date().toISOString()
    }
    todos.set(todo.id, todo)
    res.status(201).json(todo)
  } catch (err) {
    next(err)
  }
})

// GET /api/todos/:id — busca por ID
todosRouter.get('/:id', authenticate, async (req, res, next) => {
  try {
    const todo = todos.get(req.params.id)
    if (!todo) {
      return res.status(404).json({ error: 'Tarefa não encontrada' })
    }
    res.json(todo)
  } catch (err) {
    next(err)
  }
})

// PATCH /api/todos/:id — atualização parcial
todosRouter.patch('/:id', authenticate, validate(updateTodoSchema), async (req, res, next) => {
  try {
    const todo = todos.get(req.params.id)
    if (!todo) return res.status(404).json({ error: 'Não encontrada' })

    const updated = { ...todo, ...req.body }
    todos.set(req.params.id, updated)
    res.json(updated)
  } catch (err) {
    next(err)
  }
})

// DELETE /api/todos/:id
todosRouter.delete('/:id', authenticate, async (req, res, next) => {
  try {
    if (!todos.has(req.params.id)) {
      return res.status(404).json({ error: 'Não encontrada' })
    }
    todos.delete(req.params.id)
    res.status(204).send()
  } catch (err) {
    next(err)
  }
})
// middleware/auth.js — verificação JWT
import jwt from 'jsonwebtoken'

export function authenticate(req, res, next) {
  const auth = req.headers.authorization
  if (!auth?.startsWith('Bearer ')) {
    return res.status(401).json({ error: 'Token não fornecido' })
  }

  try {
    const token = auth.slice(7)  // remove "Bearer "
    req.user = jwt.verify(token, process.env.JWT_SECRET)
    next()  // passa para o próximo middleware/handler
  } catch {
    res.status(401).json({ error: 'Token inválido ou expirado' })
  }
}
// middleware/errorHandler.js — tratamento centralizado
/**
 * Error handler global — deve ser registrado APÓS todas as rotas.
 * Express identifica como error handler pela assinatura (err, req, res, next).
 */
export function errorHandler(err, req, res, next) {
  console.error(err.stack)

  // Erros de validação (Zod, Joi...)
  if (err.name === 'ZodError') {
    return res.status(422).json({
      error: 'Dados inválidos',
      details: err.errors
    })
  }

  // Erro genérico
  const status = err.status || err.statusCode || 500
  res.status(status).json({
    error: err.message || 'Erro interno do servidor'
  })
}

Validação com Zod

// schemas/todo.js — schemas de validação
import { z } from 'zod'

export const createTodoSchema = z.object({
  title: z.string().min(1).max(200),
  description: z.string().max(1000).optional(),
  priority: z.number().int().min(1).max(5).default(1)
})

export const updateTodoSchema = createTodoSchema.partial().extend({
  completed: z.boolean().optional()
})

// middleware/validate.js
export const validate = (schema) => (req, res, next) => {
  try {
    req.body = schema.parse(req.body)  // valida e transforma
    next()
  } catch (err) {
    next(err)  // passa ZodError para o errorHandler
  }
}

Erros comuns

⚠️ Esquecer next(err) no catch de handlers async

Em handlers assíncronos, erros dentro de try/catch devem ser passados para next(err) — caso contrário o Express não os captura e a requisição fica pendurada. Considere usar um wrapper asyncHandler para eliminar o boilerplate.

// Wrapper que encapsula qualquer handler async
const asyncHandler = fn => (req, res, next) => {
  Promise.resolve(fn(req, res, next)).catch(next)
}

// Uso:
router.get('/', asyncHandler(async (req, res) => {
  const data = await someAsyncOperation()
  res.json(data)
  // qualquer throw chega automaticamente no errorHandler
}))
⚠️ Registrar error handler antes das rotas

O error handler global DEVE ser o último middleware registrado — após todas as rotas. Express identifica error handlers pela assinatura de 4 parâmetros (err, req, res, next), mas a ordem de registro é o que determina se ele é chamado.

⚠️ CORS muito permissivo em produção

Nunca use cors() sem configuração em produção — isso permite qualquer origem. Configure explicitamente: cors({ origin: ['https://seuapp.com'] }). Em desenvolvimento, use uma variável de ambiente para o origin.
