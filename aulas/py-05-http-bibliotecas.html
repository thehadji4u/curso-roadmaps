// motivação

Por que isso importa?

Um desenvolvedor Python profissional passa boa parte do tempo fazendo três coisas: conversar com APIs externas, manipular arquivos e caminhos no sistema, e processar textos e datas. Essas quatro bibliotecas — requests, pathlib, datetime e re — cobrem exatamente isso.

Entender HTTP não é opcional. Todo sistema moderno se comunica via HTTP — seja integrando com um pagamento, consumindo dados de uma bolsa de valores, ou chamando uma IA. Python torna essa comunicação trivial, mas saber o que está acontecendo por baixo é o que separa quem conserta bugs rapidamente de quem fica perdido horas.

💼 Por que um engenheiro sênior domina essas libs
Sêniors sabem qual biblioteca usar para cada contexto e por quê. Eles não usam requests onde precisam de async (usam httpx ou aiohttp), não concatenam strings para construir paths (usam pathlib), e não parsam datas com regex (usam datetime). Conhecer bem o stdlib e as libs de fato do Python evita reescrever código e reduz bugs em produção.
// analogia

Antes do código: uma analogia

🌍 Mundo Real → Tecnologia
Pense num restaurante com serviço de entrega. Você (cliente) faz um pedido (HTTP Request) para o restaurante (servidor/API). O pedido contém o endereço (URL), o que você quer (endpoint + parâmetros), e às vezes um cartão de fidelidade para identificação (API key no header). O restaurante processa e te manda a comida de volta (HTTP Response) com um código de status — "Pronto! ✅" (200), "Não temos esse prato ❌" (404), ou "Cozinha pegando fogo 🔥" (500).

O requests é o garçom que faz tudo isso pra você sem você precisar ligar diretamente para a cozinha.
📞 Você ligando para o restaurante → HTTP Request (GET, POST...) 📋 O endereço e o pedido específico → URL + Endpoint + Params 🪪 Cartão de fidelidade → API Key / Authorization Header 🛵 Entregador com a comida + nota → HTTP Response + JSON body "Pronto!" / "Esgotado" / "Fechado" → Status Code (200 / 404 / 503)

requests — HTTP para humanos

requests é a biblioteca HTTP mais popular do Python e uma das mais baixadas do mundo. Ela abstrai toda a complexidade do protocolo e oferece uma API limpa e intuitiva. Seu slogan oficial é "HTTP for Humans" — e é exatamente isso.

💡 Não é stdlib!
requests não vem com o Python. É necessário instalar: pip install requests. Para projetos com async, prefira httpx (mesma API, suporta async/await).

GET — buscando dados

// requests — GET básico com a API pública do GitHub
import requests

# GET simples — busca informações do repositório cpython
url = "https://api.github.com/repos/python/cpython"
response = requests.get(url)

# Verificar status
print(response.status_code)      # 200
print(response.headers["Content-Type"])  # application/json; charset=utf-8

# Converter o body JSON para dict Python — requests faz isso automaticamente
data = response.json()
print(data["full_name"])         # python/cpython
print(data["stargazers_count"])  # estrelas do repo
print(data["open_issues_count"]) # issues abertas

GET com parâmetros e headers

// params e headers — busca paginada com autenticação
import requests
import os

# Parâmetros de query — requests monta a URL automaticamente
params = {
    "q": "language:python stars:>10000",  # query de busca
    "sort": "stars",
    "per_page": 5,
    "page": 1,
}

# Headers com autenticação via token (nunca hardcode o token!)
headers = {
    "Authorization": f"Bearer {os.environ['GITHUB_TOKEN']}",
    "Accept": "application/vnd.github.v3+json",
    "X-GitHub-Api-Version": "2022-11-28",
}

response = requests.get(
    "https://api.github.com/search/repositories",
    params=params,
    headers=headers,
    timeout=10,  # ← SEMPRE defina timeout! Sem isso, pode pendurar para sempre
)

# Verificação de erro explícita
response.raise_for_status()  # lança HTTPError se status >= 400

repos = response.json()["items"]
for repo in repos:
    print(f"{repo['full_name']:45} ⭐ {repo['stargazers_count']:,}")

POST — enviando dados (JSON)

// POST com JSON body
import requests

# Payload que será enviado como JSON no body da requisição
payload = {
    "title": "Bug encontrado na aula 15",
    "body": "Descrição detalhada do bug...",
    "labels": ["bug", "documentation"],
}

response = requests.post(
    "https://api.github.com/repos/meu-usuario/meu-repo/issues",
    json=payload,       # requests serializa para JSON e seta Content-Type automaticamente
    headers={"Authorization": "Bearer TOKEN"},
)

if response.status_code == 201:
    issue = response.json()
    print(f"Issue criada: #{issue['number']} — {issue['html_url']}")
else:
    print(f"Erro {response.status_code}: {response.text}")

Tratamento robusto de erros

// error handling — padrão de produção
import requests
from requests.exceptions import Timeout, ConnectionError, HTTPError

def buscar_usuario(user_id: int) -> dict | None:
    """Busca usuário com tratamento completo de erros."""
    try:
        response = requests.get(
            f"https://api.exemplo.com/users/{user_id}",
            timeout=(3.05, 27),  # (connect timeout, read timeout)
        )
        response.raise_for_status()
        return response.json()
    
    except Timeout:
        print(f"Timeout buscando usuário {user_id}")
        return None
    
    except ConnectionError:
        print("Sem conexão com o servidor")
        return None
    
    except HTTPError as e:
        status = e.response.status_code
        if status == 404:
            print(f"Usuário {user_id} não encontrado")
        elif status == 429:
            print("Rate limit atingido — tente novamente em breve")
        else:
            print(f"Erro HTTP {status}: {e.response.text}")
        return None
    
    except ValueError:
        print("Resposta não é JSON válido")
        return None

pathlib — caminhos de arquivo orientados a objeto

Antes do pathlib (Python 3.4+), manipular caminhos de arquivo era um pesadelo de concatenação de strings com os.path. Hoje, pathlib.Path oferece uma API orientada a objeto, legível e que funciona igual no Windows, Linux e macOS.

⚠️ Evite os.path.join em código novo
os.path.join("pasta", "arquivo.txt") funciona, mas é verboso e menos elegante. Em código novo, use sempre pathlib.Path. Reserve os para variáveis de ambiente e operações de sistema.
// pathlib — operações essenciais
from pathlib import Path

# Criando objetos Path
p = Path(".")               # diretório atual
home = Path.home()          # /home/usuario no Linux, C:\Users\... no Windows
projeto = Path("/projetos/meu-app")

# Navegação — o operador / funciona como separador de caminho
config = projeto / "config" / "settings.json"
print(config)               # /projetos/meu-app/config/settings.json
print(config.parent)        # /projetos/meu-app/config
print(config.name)          # settings.json
print(config.stem)          # settings  (sem extensão)
print(config.suffix)        # .json
print(config.suffixes)      # ['.json'] (ou ['.tar', '.gz'])

# Verificações
print(config.exists())      # True ou False
print(config.is_file())     # True se é arquivo
print(config.is_dir())      # True se é diretório

# Criação de diretórios
output = Path("output/reports/2025")
output.mkdir(parents=True, exist_ok=True)  # cria toda a árvore, não falha se já existe

# Leitura e escrita — clean API
texto = config.read_text(encoding="utf-8")

# Bytes
binario = Path("imagem.png").read_bytes()

# Listagem de arquivos
pasta = Path("src")
for arquivo in pasta.glob("**/*.py"):  # recursivo
    print(arquivo.relative_to(pasta))

# Apenas arquivos diretos (sem subpastas)

print(f"Encontrados {len(csvs)} arquivos CSV")

# Renomear e mover
old = Path("relatorio_v1.pdf")
new = old.rename(old.parent / "relatorio_v2.pdf")

# Informações do arquivo
stat = config.stat()
print(f"Tamanho: {stat.st_size:,} bytes")

# Resolução de path relativo → absoluto

datetime — manipulação de datas e horários

Datas são traiçoeiras. Fusos horários, horário de verão, formatos diferentes por país — é fácil introduzir bugs sérios sem perceber. O módulo datetime da stdlib resolve a maioria dos casos, e para casos avançados, a biblioteca arrow ou pendulum complementa.

// datetime — do básico ao avançado
from datetime import datetime, date, timedelta, timezone

# Agora — sem fuso (naive) vs com fuso (aware)
agora_naive = datetime.now()                          # local, sem info de fuso
agora_utc = datetime.now(tz=timezone.utc)             # UTC explícito — use SEMPRE em produção
print(agora_utc.isoformat())  # 2025-03-15T14:30:00+00:00

# Apenas a data de hoje
hoje = date.today()
print(hoje)              # 2025-03-15
print(hoje.year)         # 2025
print(hoje.month)        # 3
print(hoje.day)          # 15
print(hoje.weekday())    # 0=Segunda ... 6=Domingo

# Criando datas específicas
nascimento = date(1990, 7, 23)
reuniao = datetime(2025, 12, 31, 23, 59, 0, tzinfo=timezone.utc)

# Aritmética de datas com timedelta
daqui_30_dias = hoje + timedelta(days=30)
semana_passada = hoje - timedelta(weeks=1)
prazo = datetime(2025, 12, 31) - datetime.now()
print(f"Faltam {prazo.days} dias para o fim do ano")

# Formatação — strftime (datetime → string)
print(agora_utc.strftime("%d/%m/%Y %H:%M"))     # 15/03/2025 14:30
print(agora_utc.strftime("%A, %B %d, %Y"))      # Saturday, March 15, 2025
print(agora_utc.strftime("%Y%m%d_%H%M%S"))      # 20250315_143000 — bom para nomes de arquivo

# Parsing — strptime (string → datetime)
data_str = "15/03/2025 14:30:00"
dt = datetime.strptime(data_str, "%d/%m/%Y %H:%M:%S")
print(dt)  # 2025-03-15 14:30:00

# ISO 8601 — formato de APIs (fromisoformat)
iso_str = "2025-03-15T14:30:00+00:00"
dt_iso = datetime.fromisoformat(iso_str)  # Python 3.7+
print(dt_iso.tzinfo)  # UTC+00:00

# Conversão de timestamp Unix
timestamp = 1741968000
dt_from_ts = datetime.fromtimestamp(timestamp, tz=timezone.utc)
print(dt_from_ts)

# Diferença entre datas
delta = date(2025, 12, 31) - date.today()
print(f"{delta.days} dias para o ano acabar")

⚠️ Pitfall: datetime naive vs aware

Misturar datetimes com e sem fuso horário causa TypeError em comparações. Regra de ouro: sempre use UTC internamente e converta apenas na exibição ao usuário.

# ❌ ERRADO — comparação explode com TypeError
dt_naive = datetime.now()
dt_aware = datetime.now(tz=timezone.utc)
# dt_naive < dt_aware  → TypeError!

# ✅ CORRETO — tudo em UTC
dt_utc1 = datetime.now(tz=timezone.utc)
dt_utc2 = datetime(2025, 12, 31, tzinfo=timezone.utc)
print(dt_utc1 < dt_utc2)  # True ou False, sem erro

re — Expressões Regulares

Expressões regulares (regex) são uma mini-linguagem para encontrar padrões em texto. São incrivelmente poderosas para validação, extração e transformação de strings. O módulo re faz parte da stdlib do Python.

A curva de aprendizado existe, mas o investimento vale: regex aparece em logs, parsing de dados, web scraping, validação de formulários e muito mais.

Conceitos básicos de regex

\dDígito [0-9]\d+ → "123", "42"

\sEspaço em branco\s+ → " ", "\t"
^Início da string^https → começa com https

[abc]Um dos caracteres[aeiou] → vogais

Funções principais do módulo re

// re — do básico ao prático
import re

texto = "Email: joao@exemplo.com, maria@empresa.org — Data: 2025-03-15"

# re.search — encontra a PRIMEIRA ocorrência (retorna Match ou None)
match = re.search(r"\d{4}-\d{2}-\d{2}", texto)
if match:
    print(match.group())      # 2025-03-15
    print(match.start())      # posição inicial
    print(match.end())        # posição final

# re.findall — encontra TODAS as ocorrências (retorna lista de strings)
emails = re.findall(r"[\w.+-]+@[\w-]+\.[a-zA-Z]{2,}", texto)
print(emails)  # ['joao@exemplo.com', 'maria@empresa.org']

# re.match — verifica APENAS o início da string
padrao_cpf = r"^\d{3}\.\d{3}\.\d{3}-\d{2}$"
print(bool(re.match(padrao_cpf, "123.456.789-09")))   # True
print(bool(re.match(padrao_cpf, "abc.456.789-09")))   # False

# re.sub — substituição
resultado = re.sub(r"\s+", " ", "texto   com   espaços   extras")
print(resultado)  # "texto com espaços extras"

# Mascarar dados sensíveis
log = "Usuário 12345 transferiu R$1500 para conta 67890"
log_seguro = re.sub(r"\b\d{5}\b", "XXXXX", log)
print(log_seguro)  # "Usuário XXXXX transferiu R$1500 para conta XXXXX"

# re.compile — compila o padrão para reutilização (mais eficiente em loops)
email_pattern = re.compile(r"[\w.+-]+@[\w-]+\.[a-zA-Z]{2,}", re.IGNORECASE)

emails_encontrados = []
for linha in open("contatos.txt"):
    emails_encontrados.extend(email_pattern.findall(linha))

# Grupos de captura — extrair partes específicas
log_entry = "2025-03-15 14:30:22 ERROR app.py:42 — Connection timeout"
padrao_log = re.compile(
    r"(?P\d{4}-\d{2}-\d{2})\s"
    r"(?P\d{2}:\d{2}:\d{2})\s"
    r"(?P\w+)\s"
    r"(?P[\w.]+):(?P\d+)"
)
m = padrao_log.match(log_entry)
if m:
    print(m.group("date"))   # 2025-03-15
    print(m.group("level"))  # ERROR
    print(m.group("line"))   # 42
    print(m.groupdict())     # dict com todos os grupos nomeados
🔥 Pro tip — use sempre raw strings r""
Escreva padrões como r"\d+" em vez de "\\d+". A raw string evita que o Python interprete a barra invertida antes que chegue ao regex. Sem o r, você precisaria escrever "\\\\d+" para representar \\d.

Projeto Prático: Coletor de Dados de APIs

Vamos unir todas as bibliotecas num projeto real: um script que busca repositórios Python populares via API do GitHub, filtra com regex, manipula datas e salva os resultados em arquivo usando pathlib.

// projeto completo — github_collector.py
"""
github_collector.py
Coleta repositórios Python populares, filtra e salva em JSON.
"""
import re
import json
import requests
from pathlib import Path
from datetime import datetime, timezone


# ── Configuração ─────────────────────────────────────────────────────────────

OUTPUT_DIR = Path(__file__).parent / "output"
OUTPUT_DIR.mkdir(exist_ok=True)

API_BASE = "https://api.github.com"
PATTERN_VALID_REPO = re.compile(r"^[a-zA-Z0-9_.-]+/[a-zA-Z0-9_.-]+$")


# ── Funções ───────────────────────────────────────────────────────────────────

def buscar_repos_python(max_repos: int = 20) -> list[dict]:
    """Busca os repositórios Python mais populares via GitHub API."""
    params = {
        "q": "language:python stars:>5000",
        "sort": "stars",
        "order": "desc",
        "per_page": min(max_repos, 30),
    }
    headers = {"Accept": "application/vnd.github.v3+json"}

    try:
        resp = requests.get(
            f"{API_BASE}/search/repositories",
            params=params,
            headers=headers,
            timeout=15,
        )
        resp.raise_for_status()
        return resp.json().get("items", [])

    except requests.RequestException as e:
        print(f"Erro na API: {e}")
        return []


def processar_repo(repo: dict) -> dict | None:
    """Extrai e processa dados relevantes de um repositório."""
    nome = repo.get("full_name", "")

    # Filtra com regex — nome deve ter formato "dono/repo"
    if not PATTERN_VALID_REPO.match(nome):
        return None

    # Parsing de data ISO 8601 para datetime aware
    criado_str = repo.get("created_at", "")
    atualizado_str = repo.get("updated_at", "")

    try:
        criado = datetime.fromisoformat(criado_str.replace("Z", "+00:00"))
        atualizado = datetime.fromisoformat(atualizado_str.replace("Z", "+00:00"))
        agora = datetime.now(tz=timezone.utc)
        
        dias_sem_update = (agora - atualizado).days
        idade_anos = (agora - criado).days / 365
    except ValueError:
        return None

    return {
        "nome": nome,
        "descricao": (repo.get("description") or "")[:200],
        "estrelas": repo.get("stargazers_count", 0),
        "forks": repo.get("forks_count", 0),
        "linguagem": repo.get("language", ""),
        "criado_em": criado.strftime("%d/%m/%Y"),
        "atualizado_em": atualizado.strftime("%d/%m/%Y"),
        "dias_sem_atualizacao": dias_sem_update,
        "idade_anos": round(idade_anos, 1),
        "ativo": dias_sem_update < 180,  # menos de 6 meses sem update
        "url": repo.get("html_url", ""),
    }


def salvar_resultados(repos: list[dict]) -> Path:
    """Salva os repositórios processados em JSON com timestamp no nome."""
    timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")
    arquivo = OUTPUT_DIR / f"repos_python_{timestamp}.json"

    dados = {
        "gerado_em": datetime.now(tz=timezone.utc).isoformat(),
        "total": len(repos),
        "repositorios": repos,
    }

    arquivo.write_text(
        json.dumps(dados, ensure_ascii=False, indent=2),
        encoding="utf-8",
    )
    return arquivo


# ── Main ──────────────────────────────────────────────────────────────────────

def main():
    print("🔍 Buscando repositórios Python populares...")
    repos_raw = buscar_repos_python(max_repos=20)
    print(f"   Encontrados: {len(repos_raw)} repositórios")

    print("⚙️  Processando e filtrando...")
    repos = [r for repo in repos_raw if (r := processar_repo(repo))]
    
    # Filtra apenas repos ativos
    ativos = [r for r in repos if r["ativo"]]
    print(f"   Ativos (< 6 meses): {len(ativos)}/{len(repos)}")

    if repos:
        arquivo = salvar_resultados(repos)
        print(f"✅ Salvo em: {arquivo.relative_to(Path.cwd())}")

        print("\n📊 Top 5 por estrelas:")
        for i, repo in enumerate(sorted(repos, key=lambda x: x["estrelas"], reverse=True)[:5], 1):
            status = "✅" if repo["ativo"] else "💤"

⚠️ Armadilhas Comuns

❌ Não definir timeout nas requisições

Sem timeout, uma requisição pode ficar pendurada infinitamente. Em produção, isso pode travar seu serviço inteiro.

# ❌ Sem timeout — perigo!
requests.get("https://api.lenta.com/endpoint")

# ✅ Sempre com timeout
requests.get("https://api.lenta.com/endpoint", timeout=10)
requests.get("https://api.lenta.com/endpoint", timeout=(3.05, 27))  # connect, read
❌ API keys hardcoded no código

Nunca coloque tokens diretamente no código. Use variáveis de ambiente ou arquivos .env com python-dotenv.

# ❌ NUNCA faça isso — o token vai para o git!
headers = {"Authorization": "Bearer ghp_abc123seu_token_aqui"}

# ✅ Correto — token em variável de ambiente
import os
headers = {"Authorization": f"Bearer {os.environ['GITHUB_TOKEN']}"}
# No terminal: export GITHUB_TOKEN="ghp_..."
❌ Ignorar o status code

Chamar .json() sem verificar o status pode dar erros confusos se a resposta for um HTML de erro.

# ❌ Pode explodir com JSONDecodeError
data = requests.get(url).json()

# ✅ Verificar antes ou usar raise_for_status()
resp = requests.get(url)
resp.raise_for_status()  # lança HTTPError se 4xx ou 5xx
data = resp.json()
❌ Construir paths com f-strings ou + em strings

Concatenar caminhos com strings quebra no Windows (separador diferente) e é difícil de manter.

# ❌ Quebra no Windows
path = base_dir + "/output/" + filename

# ✅ Cross-platform e legível
from pathlib import Path
path = Path(base_dir) / "output" / filename

Glossário

REST API
Representational State Transfer — estilo arquitetural para APIs baseado em HTTP. Usa URLs para identificar recursos e verbos HTTP para ações.
Endpoint
URL específica de uma API que corresponde a um recurso ou operação. Ex: /api/v1/users/42
Rate Limiting
Limite de requisições por unidade de tempo imposto pela API para evitar abuso. Ex: 60 req/hora sem autenticação no GitHub.
Idempotente
Operação que produz o mesmo resultado independente de quantas vezes é executada. GET e DELETE são idempotentes; POST geralmente não é.
Datetime aware vs naive
Aware: datetime com informação de fuso horário (tzinfo). Naive: sem fuso. Misturar os dois causa TypeError.
Grupo de captura
Em regex, (padrão) cria um grupo cujo conteúdo pode ser extraído com .group(n). Grupos nomeados usam (?P<nome>padrão).
