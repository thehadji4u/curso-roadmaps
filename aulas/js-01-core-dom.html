Por que JavaScript importa?

JavaScript é a única linguagem nativa dos browsers. Não existe alternativa: se você quer que uma página web pense, reaja e se transforme — é JavaScript. Mas vai além: com Node.js, JS roda no servidor. Com React Native, no mobile. É literalmente a linguagem mais usada do planeta há mais de uma década seguida no Stack Overflow Survey.

Um engenheiro sênior usa JS todos os dias — seja escrevendo APIs em Node, componentizando UIs em React, ou automatizando pipelines de build. Entender o core da linguagem (tipos, escopo, closures, o event loop) é o que separa quem apenas "usa frameworks" de quem realmente entende o que está acontecendo.

🔥 Por que começar pelo Core e não pelo React?
Frameworks vêm e vão. O Core é permanente. Quem entende closures, hoisting e o event loop consegue aprender qualquer framework em dias — e debugar os problemas que os outros não conseguem.

var · let · const — Diferenças Críticas

var — Evite!
  • Escopo: função
  • Sofre hoisting
  • Re-declarável
  • Vaza blocos if/for
let — Use quando muda
  • Escopo: bloco {}
  • Temporal Dead Zone
  • Não re-declarável
  • Ideal para contadores
const — Padrão
  • Escopo: bloco {}
  • Não re-atribuível
  • Objetos: mutáveis
  • Use 90% das vezes
⚠️ const não significa imutável!
const arr = [1,2,3]; arr.push(4); — funciona! O binding é constante, não o valor. Se precisar de imutabilidade real, use Object.freeze().

// FUNÇÕES

Funções — Cidadãos de Primeira Classe

Em JavaScript, funções são valores — podem ser passadas como argumentos, retornadas por outras funções e atribuídas a variáveis. Isso é o que torna possível padrões como callbacks, higher-order functions e closures.

3 Formas de Declarar Funções

javascript — sintaxes comparadas
// 1. Declaração de função (function declaration)
// Sofre hoisting — pode ser chamada ANTES de ser declarada
function somar(a, b) {
  return a + b;
}

// 2. Expressão de função (function expression)
// NÃO sofre hoisting (a variável sofre, mas não a função)
const multiplicar = function(a, b) {
  return a * b;
};

// 3. Arrow function (ES6+) — forma mais moderna e concisa
// NÃO tem seu próprio 'this' — captura o this do contexto pai
const dividir = (a, b) => a / b;

// Arrow function com corpo (mais de uma linha)
const calcularIMC = (peso, altura) => {
  const imc = peso / (altura ** 2);
  return imc.toFixed(2);
};

// Funções como valores — passando como argumento
const numeros = [3, 1, 4, 1, 5, 9];
const ordenados = numeros.sort((a, b) => a - b);
// [1, 1, 3, 4, 5, 9]

Closures — O Superpoder Oculto

Uma closure é uma função que "lembra" do ambiente em que foi criada, mesmo após a função externa ter terminado. É o fundamento dos módulos, do encapsulamento e de pattern como debounce.

javascript — closure na prática
// Exemplo 1: Contador com estado privado
function criarContador() {
  let count = 0; // variável privada — não acessível de fora

  return {
    incrementar: () => ++count,
    decrementar: () => --count,
    valor: () => count
  };
}

const contador = criarContador();
contador.incrementar(); // 1
contador.incrementar(); // 2
contador.decrementar(); // 1
console.log(contador.valor()); // 1
// count não é acessível diretamente — está "fechado" na closure

// Exemplo 2: Factory function com closure
function criarMultiplicador(fator) {
  return (numero) => numero * fator; // 'fator' vive na closure
}

const dobrar = criarMultiplicador(2);
const triplicar = criarMultiplicador(3);

console.log(dobrar(5));    // 10
console.log(triplicar(5)); // 15
💡 Closures no mundo real
Event handlers em React são closures. Middlewares no Express são closures. O padrão de módulo IIFE é uma closure. Entender closures é entender 80% dos patterns do ecossistema JS.

O DOM — Sua Interface com o HTML

O DOM (Document Object Model) é a representação do HTML como uma árvore de objetos JavaScript. Quando o browser carrega uma página, ele parseia o HTML e cria essa árvore na memória. O JavaScript manipula essa árvore para mudar o que o usuário vê — sem precisar recarregar a página.

// Em vez de adicionar listener em cada item, adicione no pai
// Funciona para elementos criados dinamicamente também!
const lista = document.querySelector('#lista');
lista.addEventListener('click', (e) => {
  if (e.target.matches('li')) {

});

    Em vez de adicionar um listener para cada elemento filho, adicione um só no pai e use e.target.matches(). Mais performático, funciona com elementos adicionados dinamicamente e é o padrão em frameworks como jQuery e React (internamente).

    // todo.js — Todo App com closure e manipulação de DOM

// Estado encapsulado em closure (padrão módulo)
const todoApp = (() => {
  let tarefas = [];
  let proximoId = 1;

  // Funções privadas (acessíveis apenas dentro do closure)
  function criarTarefa(texto) {
    return { id: proximoId++, texto, concluida: false };
  }

  function atualizarStatus() {
    const status = document.querySelector('#status');
    const total = tarefas.length;
    const concluidas = tarefas.filter(t => t.concluida).length;
    status.textContent = `${concluidas}/${total} tarefas concluídas`;
  }

  function renderizarLista() {
    const lista = document.querySelector('#lista');
    lista.innerHTML = ''; // limpa a lista

    tarefas.forEach(tarefa => {
      const li = document.createElement('li');
      
      const span = document.createElement('span');
      span.textContent = tarefa.texto;
      if (tarefa.concluida) span.classList.add('concluido');
      span.style.cursor = 'pointer';
      span.addEventListener('click', () => alternarTarefa(tarefa.id));

      const btn = document.createElement('button');
      btn.textContent = '🗑';
      btn.addEventListener('click', () => removerTarefa(tarefa.id));

      li.appendChild(span);
      li.appendChild(btn);
      lista.appendChild(li);
    });

    atualizarStatus();
  }

  // Funções públicas (retornadas pelo módulo)
  function adicionarTarefa(texto) {
    if (!texto.trim()) return;
    tarefas.push(criarTarefa(texto));
    renderizarLista();
  }

  function removerTarefa(id) {
    tarefas = tarefas.filter(t => t.id !== id);
    renderizarLista();
  }

  function alternarTarefa(id) {
    const tarefa = tarefas.find(t => t.id === id);
    if (tarefa) tarefa.concluida = !tarefa.concluida;
    renderizarLista();
  }

  return { adicionarTarefa, removerTarefa, alternarTarefa };
})(); // IIFE — executa imediatamente


// Wiring: conectar o formulário ao módulo
document.addEventListener('DOMContentLoaded', () => {

    Exemplo Completo: Buscador Dinâmico com Debounce

    Vamos implementar um buscador que filtra uma lista em tempo real conforme o usuário digita — mas com debounce para não executar a busca a cada tecla pressionada (padrão usado em todos os apps profissionais).

    javascript — buscador com debounce e closure
    // ============================================================
// Debounce — atrasa execução até o usuário parar de digitar
// Clássico exemplo de closure + setTimeout
// ============================================================
function debounce(fn, delay) {
  let timer; // timer vive na closure — persiste entre chamadas
  
  return function(...args) { // retorna nova função
    clearTimeout(timer);      // cancela o timer anterior
    timer = setTimeout(() => { // agenda nova execução
      fn.apply(this, args);   // chama a função original
    }, delay);
  };
}


// ============================================================
// Dados — lista de linguagens de programação
// ============================================================
const linguagens = [
  { nome: 'JavaScript', ano: 1995, criador: 'Brendan Eich' },
  { nome: 'Python', ano: 1991, criador: 'Guido van Rossum' },
  { nome: 'TypeScript', ano: 2012, criador: 'Microsoft' },
  { nome: 'Rust', ano: 2010, criador: 'Mozilla' },
  { nome: 'Go', ano: 2009, criador: 'Google' },
  { nome: 'Kotlin', ano: 2011, criador: 'JetBrains' },
  { nome: 'Swift', ano: 2014, criador: 'Apple' },
];


// ============================================================
// Função de busca — filtra e renderiza resultados
// ============================================================
function buscar(termo) {
  const container = document.querySelector('#resultados');
  
  if (!termo.trim()) {
    container.innerHTML = '
    Digite para buscar...
    ';
    return;
  }
  
  const termoNormalizado = termo.toLowerCase().trim();
  
  // Filtra as linguagens que casam com o termo
  const resultados = linguagens.filter(lang =>
    lang.nome.toLowerCase().includes(termoNormalizado) ||
    lang.criador.toLowerCase().includes(termoNormalizado)
  );
  
  if (resultados.length === 0) {
    container.innerHTML = '
    Nenhum resultado encontrado.
    ';
    return;
  }
  
  // Template literals para gerar HTML dinâmico
  container.innerHTML = resultados.map(lang => `
    <div class="resultado-item">
      <strong>${lang.nome}</strong> · ${lang.ano}
      <span style="color: #888">por ${lang.criador}</span>
    </div>
  `).join('');
}


// ============================================================
// Wiring — conectar o input ao buscador com debounce
// ============================================================
document.addEventListener('DOMContentLoaded', () => {
  const inputBusca = document.querySelector('#busca');
  
  // Cria versão "debounced" da função buscar
  // Só executa 300ms após o usuário parar de digitar
  const buscarComDebounce = debounce(buscar, 300);
  
  inputBusca.addEventListener('input', (e) => {
    buscarComDebounce(e.target.value);
  });
});

    ⚠️ Pitfalls — Erros Clássicos de Iniciante

    🪤 var dentro de loop — comportamento inesperado

    for (var i = 0; i < 3; i++) { setTimeout(() => console.log(i)) } — imprime "3 3 3", não "0 1 2". Isso porque var tem escopo de função, não de bloco. Use let para obter o comportamento esperado.

    🪤 == vs === — coerção silenciosa

    0 == false → true. "5" == 5 → true. O operador == faz coerção de tipo. Sempre use === (igualdade estrita) — é o padrão da comunidade. O == gera bugs difíceis de rastrear.

    🪤 typeof null === "object" — bug histórico

    É uma falha de design que nunca foi corrigida por compatibilidade. Para checar se algo é null, use valor === null, não typeof valor === "object".

    🪤 innerHTML com input do usuário — XSS

    Nunca faça elemento.innerHTML = input.value. Se o usuário digitar <script>alert('hackeado')</script>, vai executar! Use textContent para texto puro, ou sanitize o HTML antes.

    🪤 this em arrow functions — sem binding próprio

    Arrow functions não têm seu próprio this. Em métodos de objeto que precisam referenciar o objeto, use function tradicional. Em callbacks, prefira arrow functions justamente por isso.

    🪤 Esquecer de DOMContentLoaded

    Se você coloca o <script> no <head> e tenta selecionar elementos antes de a página carregar, vai obter null. Solução: coloque o script antes do </body> ou use document.addEventListener('DOMContentLoaded', ...).

    • ✦ Mínimo 8 itens com ao menos 3 categorias diferentes
    • ✦ Input de busca com debounce (≥ 200ms) filtrando por nome
    • ✦ Botões de filtro por categoria (Todos / Cat1 / Cat2 / Cat3)
    • ✦ Estado gerenciado com closure (padrão módulo)
    • ✦ Cards criados via document.createElement ou template literals
    • ✦ Contagem dinâmica de itens visíveis ("X de Y itens")
    • ✦ Feedback visual quando não há resultados
    💬 Dica: Comece definindo o estado (array de itens + filtro ativo + termo de busca). A função de render aplica os dois filtros em sequência: primeiro categoria, depois busca. Toda interação chama render novamente.
