Por que isso importa?

Nem toda informação está disponível via API. Muitos sistemas legados, portais governamentais, plataformas de e-commerce e aplicações empresariais só expõem dados através da interface web. Automação de browser permite interagir com qualquer site exatamente como um humano faria — e processar os resultados programaticamente.

Playwright é a ferramenta moderna de referência: suporta Chromium, Firefox e WebKit, tem API async nativa, auto-waits inteligentes e é mantido pela Microsoft. Um engenheiro sênior usa Playwright para testes E2E, scraping estruturado, automação de tarefas repetitivas e integração com sistemas sem API.

🧪

Testes E2E

Simula o usuário real: clica, preenche formulários, navega — garante que o fluxo completo funciona.

📊

Scraping Estruturado

Extrai dados de sites dinâmicos (JavaScript-heavy) que ferramentas simples como requests não conseguem.

🤖

Automação de Tarefas

Preenche formulários, faz uploads, gera relatórios — tarefas manuais repetitivas eliminadas.

🔍

Monitoramento

Monitora preços, disponibilidade de produtos, mudanças em páginas com alertas automáticos.

Primeiro script — capturar título de uma página
Abre o browser, navega, extrai dados, fecha
python — primeiro_script.py
from playwright.sync_api import sync_playwright

with sync_playwright() as p:
    # Abre browser (headless=False para ver a janela)
    browser = p.chromium.launch(headless=True)
    page = browser.new_page()

    # Navega para a URL
    page.goto("https://example.com")

    # Extrai o título
    titulo = page.title()
    print(f"Título: {titulo}")

    # Extrai texto de um elemento
    texto = page.locator("h1").text_content()
    print(f"H1: {texto}")

    # Captura screenshot
    page.screenshot(path="screenshot.png")

    browser.close()

Versão Async (recomendada para produção)
Mais eficiente — permite múltiplas páginas em paralelo
python — async_playwright.py
import asyncio
from playwright.async_api import async_playwright

async def main():
    async with async_playwright() as p:
        browser = await p.chromium.launch(headless=True)
        page = await browser.new_page()

        await page.goto("https://quotes.toscrape.com")

        # Extrai todas as citações
        citacoes = await page.locator(".quote .text").all_text_contents()
        autores = await page.locator(".quote .author").all_text_contents()

        for citacao, autor in zip(citacoes, autores):
            print(f"{autor}: {citacao[:60]}...")

        await browser.close()

asyncio.run(main())

Seletores CSS e XPath

Para interagir com um elemento da página, você precisa localizá-lo primeiro. Playwright suporta CSS selectors, XPath, texto e locators semânticos (recomendado para testes).

CSS — por ID
#meu-botao
Elemento com id="meu-botao". Mais rápido e preciso.
CSS — por classe
.produto-card
Todos os elementos com class="produto-card".
CSS — atributo
input[name="email"]
Input com atributo name igual a "email".
CSS — filho direto
ul.menu > li:first-child
Primeiro li filho direto de ul.menu.
XPath — texto exato
//button[text()='Enviar']
Botão que contém exatamente o texto "Enviar".
XPath — contém texto
//a[contains(@href, '/produto')]
Links cujo href contém "/produto".
Playwright — por role
get_by_role("button", name="Login")
Semântico e resistente a mudanças de CSS. Preferido para testes.
Playwright — por texto
get_by_text("Ver mais detalhes")
Qualquer elemento com esse texto visível.
🔥 Como encontrar o seletor certo
No Chrome: botão direito no elemento → Inspecionar → botão direito no HTML → Copy → Copy selector. Para XPath: Copy → Copy XPath. Valide no console do DevTools com $$(".meu-seletor").

Ações Essenciais do Playwright

python — acoes_playwright.py
from playwright.sync_api import sync_playwright

with sync_playwright() as p:
    browser = p.chromium.launch(headless=False)
    page = browser.new_page()

    # === NAVEGAÇÃO ===
    page.goto("https://exemplo.com")
    page.go_back()
    page.go_forward()
    page.reload()

    # === CLIQUES ===
    page.click("#botao-login")
    page.locator("button.submit").click()
    page.get_by_role("button", name="Entrar").click()
    page.locator(".item").nth(2).click()       # terceiro item da lista
    page.locator(".link").first.click()        # primeiro link

    # === FORMULÁRIOS ===
    page.fill("#email", "usuario@email.com")
    page.fill("#senha", "minha_senha_123")
    page.press("#senha", "Enter")              # pressiona Enter

    page.select_option("#estado", "SP")        # dropdown por valor
    page.check("#aceito-termos")               # checkbox
    page.uncheck("#newsletter")

    # === AGUARDAR ELEMENTOS ===
    page.wait_for_selector(".resultado", state="visible")
    page.wait_for_load_state("networkidle")    # aguarda rede quieta
    page.wait_for_url("**/dashboard")         # aguarda URL mudar

    # === EXTRAÇÃO DE DADOS ===
    titulo = page.title()
    texto = page.locator("h1").text_content()
    href  = page.locator("a.voltar").get_attribute("href")
    html  = page.locator(".conteudo").inner_html()

    # Lista de elementos
    itens = page.locator(".produto").all()
    for item in itens:
        nome  = item.locator(".nome").text_content()
        preco = item.locator(".preco").text_content()
        print(f"{nome}: {preco}")

    # === SCROLL ===
    page.evaluate("window.scrollTo(0, document.body.scrollHeight)")
    page.locator(".carregar-mais").scroll_into_view_if_needed()

    # === SCREENSHOT E PDF ===
    page.screenshot(path="pagina.png", full_page=True)
    page.pdf(path="relatorio.pdf")

    browser.close()

    contexto = browser.new_context(

    aba1.fill("#pass", "senha")
    aba1.click("#entrar")
    aba1.wait_for_url("**/dashboard")

HTTP Intercept — Capturar e Modificar Requisições

Uma das funcionalidades mais poderosas do Playwright: interceptar as requisições de rede da página. Útil para capturar chamadas de API, bloquear recursos desnecessários (imagens, fontes) e acelerar o scraping.

// fluxo de interceptação de requisições
sequenceDiagram participant Script as Playwright Script participant Page as Página participant Net as Rede Script->>Page: page.route("**/api/**", handler) Page->>Net: fetch("/api/produtos") Net->>Script: interceptado pelo handler Script->>Script: analisa / modifica / captura Script->>Net: route.continue() ou route.fulfill() Net-->>Page: resposta (original ou mockada)
python — http_intercept.py
import json
from playwright.sync_api import sync_playwright

dados_api = []

def capturar_resposta(response):
    """Captura respostas JSON de chamadas à API."""
    if "/api/produtos" in response.url and response.status == 200:
        try:
            dados = response.json()
            dados_api.extend(dados.get("items", []))
            print(f"  Capturados {len(dados.get('items', []))} itens da API")
        except Exception:
            pass

with sync_playwright() as p:
    browser = p.chromium.launch(headless=True)

    # Contexto com bloqueio de recursos desnecessários (2-5x mais rápido)
    contexto = browser.new_context()
    page = contexto.new_page()

    # Bloquear imagens, fontes e CSS (não precisamos deles para dados)
    page.route("**/*.{png,jpg,jpeg,gif,webp,svg}", lambda r: r.abort())
    page.route("**/*.{woff,woff2,ttf,eot}", lambda r: r.abort())

    # Interceptar respostas da API
    page.on("response", capturar_resposta)

    # Navegar e interagir — as chamadas de API serão capturadas
    page.goto("https://site-exemplo.com/produtos")

    # Clicar em "próxima página" várias vezes para paginação
    for pagina in range(1, 6):
        page.wait_for_selector(".produto-card", state="visible")
        print(f"Página {pagina}: {len(page.locator('.produto-card').all())} cards visíveis")

        proximo = page.locator("button.proxima-pagina")
        if proximo.is_visible() and proximo.is_enabled():
            proximo.click()
            page.wait_for_load_state("networkidle")
        else:
            break

    print(f"\nTotal capturado via API: {len(dados_api)} produtos")
    for item in dados_api[:3]:
        print(f"  - {item.get('nome', 'N/A')}: R$ {item.get('preco', 'N/A')}")

    browser.close()

Scraping Estruturado — Exemplo Completo

Extraindo dados de um e-commerce com paginação, esperando carregamento dinâmico e exportando para JSON/CSV.

python — scraper_ecommerce.py
import asyncio, json, csv
from dataclasses import dataclass, asdict
from playwright.async_api import async_playwright

@dataclass
class Produto:
    nome: str
    preco: float
    avaliacao: float
    num_avaliacoes: int
    url: str

async def extrair_produtos_pagina(page) -> list[Produto]:
    """Extrai todos os produtos da página atual."""
    await page.wait_for_selector(".product-item", state="visible")

    items = await page.locator(".product-item").all()
    produtos = []

    for item in items:
        try:
            nome = await item.locator(".product-name").text_content()
            preco_texto = await item.locator(".price").text_content()
            preco = float(preco_texto.replace("R$", "").replace(".", "").replace(",", ".").strip())

            avaliacao_texto = await item.locator(".rating").get_attribute("data-score") or "0"
            num_aval_texto = await item.locator(".review-count").text_content() or "0"
            num_avaliacoes = int(''.join(filter(str.isdigit, num_aval_texto)) or 0)

            url_relativa = await item.locator("a").get_attribute("href")
            url = f"https://loja.exemplo.com{url_relativa}"

            produtos.append(Produto(
                nome=nome.strip(),
                preco=preco,
                avaliacao=float(avaliacao_texto),
                num_avaliacoes=num_avaliacoes,
                url=url
            ))
        except Exception as e:
            print(f"  ⚠ Erro ao extrair produto: {e}")

    return produtos

async def scrape_loja(url_base: str, max_paginas: int = 10) -> list[Produto]:
    todos_produtos = []

    async with async_playwright() as p:
        browser = await p.chromium.launch(headless=True)
        contexto = await browser.new_context(
            user_agent="Mozilla/5.0 (compatible; MeuBot/1.0)",
            viewport={"width": 1280, "height": 720}
        )

        # Bloquear mídia para acelerar
        page = await contexto.new_page()
        await page.route("**/*.{png,jpg,jpeg,gif,webp}", lambda r: r.abort())

        await page.goto(url_base)

        for num_pagina in range(1, max_paginas + 1):
            print(f"Processando página {num_pagina}...")
            produtos = await extrair_produtos_pagina(page)
            todos_produtos.extend(produtos)
            print(f"  {len(produtos)} produtos extraídos (total: {len(todos_produtos)})")

            # Verificar se existe próxima página
            proximo = page.locator("a.next-page, button[aria-label='Próxima página']")
            if not await proximo.is_visible() or not await proximo.is_enabled():
                print("Última página alcançada.")
                break

            await proximo.click()
            await page.wait_for_load_state("networkidle")
            await asyncio.sleep(1)  # Respeitar o servidor

        await browser.close()

    return todos_produtos

async def main():
    produtos = await scrape_loja("https://loja.exemplo.com/categoria/eletronicos")

    # Salvar como JSON
    with open("produtos.json", "w", encoding="utf-8") as f:
        json.dump([asdict(p) for p in produtos], f, ensure_ascii=False, indent=2)

    # Salvar como CSV
    with open("produtos.csv", "w", newline="", encoding="utf-8") as f:
        writer = csv.DictWriter(f, fieldnames=["nome", "preco", "avaliacao", "num_avaliacoes", "url"])
        writer.writeheader()
        writer.writerows([asdict(p) for p in produtos])

    print(f"\n✅ {len(produtos)} produtos salvos em produtos.json e produtos.csv")

asyncio.run(main())

Playwright vs Puppeteer vs Selenium

Feature Playwright Puppeteer Selenium
Browsers suportados Chromium, Firefox, WebKit Chromium (Chrome/Edge) Todos (via WebDriver)
Linguagens Python, JS, Java, C# JavaScript/TypeScript Python, Java, C#, Ruby...
Auto-wait Nativo e inteligente Básico Manual (WebDriverWait)
HTTP Intercept Completo Completo Limitado
Multi-tab/contexto Excelente Parcial Complexo
Manutenção Microsoft (ativa) Google (ativa) Selenium HQ (ativa)
Melhor para Testes E2E + scraping moderno Automação Chrome/Node.js Compatibilidade legada
💡 Recomendação 2025
Para novos projetos Python, use Playwright. Auto-wait inteligente elimina 80% dos problemas de flakiness. Para Node.js, Puppeteer ainda é uma boa escolha. Selenium só faz sentido para manter projetos legados ou quando precisa de suporte a browsers muito antigos.

⚠️ Armadilhas Comuns

⏱ Race conditions — clicar antes de carregar
page.click(".botao") imediatamente após page.goto() pode falhar se o botão ainda não existe no DOM. Use sempre page.wait_for_selector(".botao") ou confie no auto-wait nativo dos locators do Playwright.
🚫 Bloqueio por anti-bot
Sites com Cloudflare, Akamai ou PerimeterX detectam automação por: User-Agent genérico, ausência de cookies reais, velocidade sobre-humana e fingerprinting do browser. Soluções: user-agent realista, delays aleatórios, stealth plugins, playwright-stealth.
🔍 Seletor frágil baseado em posição
.container > div:nth-child(3) > span quebra com qualquer mudança de layout. Prefira IDs, atributos data-testid, roles ARIA ou texto visível — mais estáveis a mudanças de UI.
💾 Não fechar o browser em caso de erro
Se o script lança exceção antes de browser.close(), processos do Chromium ficam zumbis. Use sempre with sync_playwright() as p: (context manager) ou try/finally.
⚖️ Aspectos legais e éticos
Verifique o robots.txt e os Termos de Serviço do site antes de automatizar. Dados pessoais coletados via scraping podem violar a LGPD. Respeite rate limits e não sobrecarregue servidores.
