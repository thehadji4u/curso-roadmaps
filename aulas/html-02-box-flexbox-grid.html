CSS — Box Model, Flexbox & Grid

⏱ ~45 min de leitura 🎯 Fundamentos → Layout 🔧 Hands-on incluído
box model flexbox css grid cascade specificity display position unidades
// motivação

Por que isso importa?

CSS é a linguagem que transforma HTML estruturado em interfaces visuais. E os três pilares desta aula — Box Model, Flexbox e Grid — são a base de qualquer layout que você vai construir. Não importa se usa React, Vue ou HTML puro: todo componente é uma caixa com dimensões, e toda interface é um conjunto de caixas organizadas em linhas e colunas.

Entender o Box Model elimina 80% dos bugs de espaçamento. Dominar Flexbox e Grid elimina 90% das gambiarras de layout. Juntos, eles permitem construir layouts complexos e responsivos com código limpo e previsível.

💼 Por que um engenheiro sênior domina isso profundamente
Sêniors não adivinham valores de CSS — eles sabem exatamente por que um elemento tem determinado tamanho e posição. Isso torna o debug de layout 10x mais rápido. Além disso, o conhecimento profundo de Flexbox e Grid permite criar sistemas de design escaláveis e substituir dependências externas (como Bootstrap) por CSS nativo mais leve e controlado.
// analogia

Antes do código: uma analogia

🌍 Mundo Real → Tecnologia
Imagine montar uma exposição de arte em uma galeria. Cada obra tem seu espaço físico (content), uma moldura ao redor (border) e uma distância de proteção dos visitantes que a cercam (padding). A distância entre uma obra e a próxima parede ou obra adjacente é a margem (margin). Isso é o Box Model.

Agora, para organizar as obras: se você quer pendurar várias fotos em fila, horizontalmente ou verticalmente, com alinhamento flexível — você usa Flexbox, como um assistente que distribui as obras em uma única direção. Se você quer um layout em grade com linhas e colunas definidas precisamente — como as salas numeradas de um museu — você usa Grid.
🖼️ Espaço físico da obra → content (width/height) 🪟 Moldura da obra → border 📏 Distância de proteção interna → padding ↔️ Espaço entre obras adjacentes → margin 🪑 Cadeiras enfileiradas em 1 direção → Flexbox (1D) 🏛️ Salas numeradas em grade → CSS Grid (2D)

Cascade, Especificidade & Herança

CSS significa Cascading Style Sheets — e a cascata é o algoritmo que decide qual regra vence quando múltiplas regras se aplicam ao mesmo elemento. Entender isso elimina a necessidade de !important desesperado.

Especificidade — o sistema de pontuação

Cada tipo de seletor tem um peso. O browser soma os pontos e a regra com maior especificidade vence, independente da ordem no arquivo.

Estilos inline style="color:red"
0-1-0-0
ID #header { }
0-0-1-0
Classe, atributo, pseudo-classe .card { } [type="text"] { } :hover { }
0-0-0-1
Elemento, pseudo-elemento p { } h1 { } ::before { }
0-0-0-0
Seletor universal * { }
// especificidade na prática
/* Especificidade: 0-0-0-1 */
p { color: black; }

/* Especificidade: 0-0-1-0 — vence sobre p */
.destaque { color: blue; }

/* Especificidade: 0-0-1-1 — vence sobre .destaque */
p.destaque { color: green; }

/* Especificidade: 0-1-0-0 — vence sobre tudo acima */
#titulo { color: purple; }

/* !important — vence sobre TUDO (exceto outro !important mais específico) */
/* Use apenas como último recurso — sinal de problema arquitetural */
p { color: red !important; }

/* ── Seletores compostos e seus pesos ── */
/* 0-0-2-1 */
nav.menu a { }

/* 0-0-1-2 */
ul li a { }

/* 0-1-1-1 */
#header .nav a { }

/* ── Herança: propriedades que se propagam aos filhos ── */
body {
  font-family: 'JetBrains Mono', monospace; /* herdado por todos os filhos */
  color: #e8e8f0;                           /* herdado */
  font-size: 16px;                          /* herdado */
  /* background, border, margin, padding NÃO são herdados */
}

/* Forçar herança ou resetar */
.reset { all: unset; }      /* remove TODOS os estilos */
p { color: inherit; }       /* força herança explícita */
p { color: initial; }       /* valor padrão do browser */
🔥 Pro tip: :where() e :is() para controlar especificidade
:where() tem especificidade zero — perfeito para estilos base que devem ser facilmente sobrescritos. :is() assume a especificidade do seletor mais específico dentro dele. Use-os para escrever CSS mais flexível e manutenível.

O Box Model — todo elemento é uma caixa

Todo elemento HTML é representado como uma caixa retangular composta por quatro camadas: content, padding, border e margin. Entender exatamente como essas camadas interagem é fundamental para controlar tamanhos e espaçamentos.

// box model completo e box-sizing
/* ── A propriedade mais importante: box-sizing ─────────────── */

/* Comportamento PADRÃO — content-box (confuso!) */
.elemento {
  box-sizing: content-box; /* padrão */
  width: 300px;
  padding: 20px;
  border: 2px solid red;
  /* Largura REAL = 300 + 20+20 + 2+2 = 344px ← surpresa! */
}

/* Comportamento CORRETO — border-box */
.elemento {
  box-sizing: border-box;
  width: 300px;
  padding: 20px;
  border: 2px solid red;
  /* Largura REAL = 300px — padding e border ficam DENTRO */
}

/* Reset global padrão da indústria — coloque no topo do seu CSS */
*, *::before, *::after {
  box-sizing: border-box;
}

/* ── Margin, Padding, Border ─────────────────────────────────── */

/* Shorthand — sentido horário: top, right, bottom, left */
margin: 10px 20px 10px 20px;   /* 4 valores */
margin: 10px 20px;              /* vertical | horizontal */
margin: 10px;                   /* todos os lados */

/* Valores individuais */
margin-top: 10px;
margin-right: 20px;
margin-bottom: 10px;
margin-left: 20px;

/* margin auto: centraliza elementos block horizontalmente */
.container {
  max-width: 1200px;
  margin: 0 auto; /* 0 vertical, auto horizontal = centralizado */
}

/* Margin collapse: margens verticais se fundem entre elementos block */
/* Se p tem margin-bottom: 20px e o próximo p tem margin-top: 30px,
   o espaço entre eles é 30px (o maior), não 50px! */

/* Padding — igual ao margin, mas NUNCA colapsa */
padding: 16px 24px;

/* Border — shorthand: largura estilo cor */
border: 2px solid #ef4444;
border-radius: 8px;              /* bordas arredondadas */
border-radius: 50%;              /* círculo perfeito */
border-top: 1px dashed #ccc;    /* lado específico */
outline: 2px solid blue;        /* fora do border, não afeta layout */

/* ── Display: como o elemento se comporta no fluxo ─────────── */
display: block;         /* ocupa linha inteira, aceita width/height */
display: inline;        /* flui com texto, ignora width/height, margin V */
display: inline-block;  /* flui com texto MAS aceita width/height */
display: none;          /* remove do fluxo e da acessibilidade */
display: flex;          /* ativa Flexbox (próximo conceito!) */
display: grid;          /* ativa Grid */

/* ── Position: sai ou não do fluxo normal ─────────────────── */
position: static;       /* padrão — no fluxo normal */
position: relative;     /* no fluxo, mas top/right/bottom/left funcionam */
position: absolute;     /* sai do fluxo, posicionado relativo ao pai com position */
position: fixed;        /* sai do fluxo, fixo na viewport (nav sticky) */
position: sticky;       /* combina relative + fixed (nav que gruda no scroll) */

/* ── Width e Height ─────────────────────────────────────────── */
width: 300px;           /* fixo */
width: 50%;             /* relativo ao pai */
width: 100vw;           /* 100% da viewport */
width: min-content;     /* mínimo para caber o conteúdo */
width: max-content;     /* máximo sem quebrar */
width: fit-content;     /* ideal: usa max-content até caber no pai */

min-width: 200px;       /* nunca menor que isso */
max-width: 800px;       /* nunca maior que isso (essencial para responsividade) */

Flexbox — layout em uma dimensão

Flexbox é o sistema de layout ideal para distribuir e alinhar itens em uma direção — linha ou coluna. É perfeito para: barras de navegação, listas de cards, formulários, toolbars e qualquer agrupamento de elementos que precisa de alinhamento flexível.

O modelo tem duas categorias de propriedades: as do container (elemento pai com display: flex) e as dos itens (elementos filhos diretos).

// flex container com gap e justify-content: space-between

Item 2

// propriedades do CONTAINER flex
.container {
  display: flex;

  /* ── Direção dos itens ── */
  flex-direction: row;            /* padrão: esquerda → direita */
  flex-direction: row-reverse;    /* direita → esquerda */
  flex-direction: column;         /* cima → baixo */
  flex-direction: column-reverse;

  /* ── Quebra de linha ── */
  flex-wrap: nowrap;    /* padrão: todos em uma linha, podem vazar */
  flex-wrap: wrap;      /* quebra para próxima linha se necessário */
  flex-wrap: wrap-reverse;

  /* ── Shorthand: direction + wrap ── */
  flex-flow: row wrap;

  /* ── Alinhamento no eixo PRINCIPAL (flex-direction) ── */
  justify-content: flex-start;    /* padrão: itens no início */
  justify-content: flex-end;      /* itens no final */
  justify-content: center;        /* itens centralizados */
  justify-content: space-between; /* espaço igual ENTRE os itens */
  justify-content: space-around;  /* espaço igual ao redor de cada item */
  justify-content: space-evenly;  /* espaço exatamente igual entre tudo */

  /* ── Alinhamento no eixo CRUZADO (perpendicular) ── */
  align-items: stretch;      /* padrão: itens se esticam para preencher */
  align-items: flex-start;   /* itens no topo do container */
  align-items: flex-end;     /* itens no fundo */
  align-items: center;       /* itens verticalmente centralizados ← muito usado */
  align-items: baseline;     /* alinha pela linha de base do texto */

  /* ── Alinhamento de múltiplas linhas (quando wrap ativo) ── */
  align-content: flex-start;
  align-content: center;
  align-content: space-between;

  /* ── Gap entre itens (substitui margin hacks) ── */
  gap: 16px;            /* igual em todas as direções */
  gap: 8px 16px;        /* row-gap | column-gap */
  row-gap: 8px;
  column-gap: 16px;
}
// propriedades dos ITENS flex
.item {
  /* ── Ordem visual (sem alterar o HTML) ── */
  order: 0;       /* padrão: ordem do HTML */
  order: -1;      /* move para antes de todos com order:0 */
  order: 999;     /* move para depois */

  /* ── Crescimento: quanto o item cresce se houver espaço extra ── */
  flex-grow: 0;   /* padrão: não cresce */
  flex-grow: 1;   /* cresce proporcionalmente */
  flex-grow: 2;   /* cresce 2x mais que itens com flex-grow:1 */

  /* ── Encolhimento: quanto o item encolhe se não houver espaço ── */
  flex-shrink: 1; /* padrão: pode encolher */
  flex-shrink: 0; /* nunca encolhe (mantém tamanho base) */

  /* ── Tamanho base antes de crescer/encolher ── */
  flex-basis: auto;   /* padrão: usa width ou conteúdo */
  flex-basis: 200px;  /* tamanho inicial fixo */
  flex-basis: 0;      /* ignora tamanho do conteúdo, cresce do zero */

  /* ── Shorthand flex: grow shrink basis ── */
  flex: 1;          /* equivale a: flex: 1 1 0% — cresce e encolhe igualmente */
  flex: 1 0 200px;  /* cresce, não encolhe, base 200px */
  flex: none;       /* equivale a: 0 0 auto — tamanho fixo */
  flex: auto;       /* equivale a: 1 1 auto */

  /* ── Alinhamento individual (sobrescreve align-items do container) ── */
  align-self: auto;
  align-self: flex-start;
  align-self: center;
  align-self: flex-end;
  align-self: stretch;
}

CSS Grid — layout em duas dimensões

CSS Grid é o sistema de layout mais poderoso do CSS. Enquanto Flexbox trabalha em uma dimensão (linha OU coluna), Grid trabalha em duas dimensões simultaneamente — linhas E colunas. É ideal para layouts de página inteira, dashboards, galerias e qualquer estrutura com alinhamento em ambos os eixos.

// propriedades do CONTAINER grid
.grid {
  display: grid;

  /* ── Definir colunas ── */
  grid-template-columns: 200px 1fr 1fr;       /* fixo + frações */
  grid-template-columns: repeat(3, 1fr);       /* 3 colunas iguais */
  grid-template-columns: repeat(4, minmax(200px, 1fr)); /* mín 200px, máx igual */
  grid-template-columns: auto 1fr auto;        /* conteúdo | flexível | conteúdo */

  /* ── Definir linhas ── */
  grid-template-rows: 64px 1fr 80px;          /* header | main | footer */
  grid-template-rows: auto;                    /* altura pelo conteúdo */

  /* ── Gap entre células ── */
  gap: 24px;
  gap: 16px 24px;   /* row-gap | column-gap */

  /* ── Alinhamento de todas as células ── */
  justify-items: stretch; /* padrão: células preenchem a coluna */
  justify-items: start | end | center;
  align-items: stretch;
  align-items: start | end | center;

  /* ── Alinhamento da grade dentro do container ── */
  justify-content: start | end | center | space-between;
  align-content: start | end | center | space-between;

  /* ── Linhas implícitas (quando itens excedem as definidas) ── */
  grid-auto-rows: 200px;         /* todas as linhas implícitas têm 200px */
  grid-auto-rows: minmax(100px, auto);
  grid-auto-flow: row;           /* padrão: preenche linha a linha */
  grid-auto-flow: column;        /* preenche coluna a coluna */
  grid-auto-flow: row dense;     /* preenche buracos (para galerias) */
}
// posicionamento de itens no grid
/* ── Posicionar itens por linha de grade ── */
.item {
  /* Grid lines: 1-indexed. Uma grade de 3 colunas tem 4 linhas verticais */

  grid-column: 1 / 3;           /* da linha 1 até a 3 (ocupa 2 colunas) */
  grid-column: 1 / -1;          /* da primeira até a última linha (largura total) */
  grid-column: span 2;          /* ocupa 2 colunas a partir da posição atual */
  grid-row: 1 / 3;              /* da linha 1 até a 3 (ocupa 2 linhas) */
  grid-row: span 1;

  /* Shorthand: row-start / column-start / row-end / column-end */
  grid-area: 1 / 1 / 3 / 3;    /* linhas 1-3, colunas 1-3 */

  /* Alinhamento individual */
  justify-self: start | end | center | stretch;
  align-self: start | end | center | stretch;
}

/* ── Template areas — a abordagem mais legível ── */
.page {
  display: grid;
  grid-template-columns: 260px 1fr 300px;
  grid-template-rows: 64px 1fr 80px;
  grid-template-areas:
    "header  header  header"
    "sidebar main    aside"
    "footer  footer  footer";
  min-height: 100vh;
  gap: 0;
}

.page-header  { grid-area: header; }
.page-sidebar { grid-area: sidebar; }
.page-main    { grid-area: main; }
.page-aside   { grid-area: aside; }
.page-footer  { grid-area: footer; }

Grid Responsivo — o padrão mais poderoso

// grade que se adapta sem media queries
/* ── auto-fill vs auto-fit + minmax ─────────────────────────── */

/* auto-fill: cria o máximo de colunas que cabem, mesmo que vazias */
.galeria {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(250px, 1fr));
  gap: 20px;
}
/* Em tela larga → 4 colunas; tablet → 2 colunas; mobile → 1 coluna */
/* Tudo sem media query! */

/* auto-fit: como auto-fill, mas colunas vazias colapsam a 0 */
.cards {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
  gap: 24px;
}
/* Use auto-fit quando quer que os itens se expandam para preencher */

/* ── minmax() — controla mínimo e máximo de uma trilha ── */
/* minmax(min, max):
   min = tamanho mínimo da trilha
   max = tamanho máximo (pode ser fr, %, auto, max-content...) */

grid-template-rows: minmax(100px, auto); /* pelo menos 100px, cresce com conteúdo */
grid-template-columns: minmax(200px, 1fr); /* entre 200px e 1 fração */

Dimensões1D — linha OU coluna2D — linhas E colunas
Ideal paraComponentes: nav, cards, botõesLayouts de página inteira
ControleItens controlam seu próprio tamanhoContainer controla tudo
Alinhamento cruzadoLimitado a uma linha por vezPerfeito alinhamento em 2D
Responsividade sem media queryCom flex-wrap + flex-basisCom auto-fill/fit + minmax
Sobreposição de elementosDifícilNativo (grid-area)
Suporte a browsersExcelenteExcelente (IE11 parcial)
🔥 Pro tip: use os dois juntos
Flexbox e Grid não são concorrentes — são complementares. A regra de ouro: use Grid para o macro-layout (estrutura da página, posições das seções) e Flexbox para o micro-layout (alinhamento dentro de um componente). A maioria dos projetos profissionais usa os dois extensivamente.

Unidades CSS — quando usar cada uma

Escolher a unidade correta é parte fundamental de um CSS manutenível e responsivo. Cada unidade tem seu contexto ideal.

UnidadeRelativa aIdeal paraEvitar em
px Pixel fixo Borders, box-shadow, outline, detalhes precisos Font-size, larguras de layout
rem Font-size do html (padrão 16px) Font-size, padding, margin, gap — respeita preferências do usuário —
em Font-size do elemento pai Espaçamento proporcional ao texto local (ex: padding de botão) Contextos aninhados — se acumula e fica imprevisível
% Dimensão do pai Larguras de layout fluidas, padding vertical em aspect-ratio Font-size (use rem)
vw Largura da viewport Layouts full-width, font-size fluido com clamp() Heights em mobile (usa vh em vez de dvh)
vh Altura da viewport Layouts full-height em desktop Mobile — a barra do browser muda a altura
dvh Altura dinâmica da viewport Substituto moderno de vh no mobile Browsers muito antigos
ch Largura do caractere "0" Largura de campos de texto (width: 40ch) Layouts gerais
fr Fração do espaço disponível Grid — distribui espaço restante Fora de contexto grid/flex
// funções CSS modernas essenciais
/* ── clamp(min, ideal, max) — valor fluido com limites ── */
/* Font-size que cresce com a tela, mas tem limites */
h1 { font-size: clamp(1.5rem, 4vw + 1rem, 3rem); }
/*                     ↑ mín   ↑ fluido         ↑ máx */

/* Container width responsivo */
.container {
  width: clamp(320px, 90%, 1200px);
  margin: 0 auto;
}

/* ── min() e max() ── */
width: min(500px, 100%);   /* o menor dos dois — nunca passa de 500px */
width: max(300px, 50%);    /* o maior dos dois — nunca menor que 300px */

/* ── calc() — cálculos mistos ── */
width: calc(100% - 2 * 24px);     /* 100% menos as margens */
padding: calc(1rem + 2px);
height: calc(100vh - 64px);       /* viewport menos navbar */

/* ── Variáveis CSS (Custom Properties) ── */
:root {
  --spacing-sm: 8px;
  --spacing-md: 16px;
  --spacing-lg: 24px;
  --color-primary: #ef4444;
  --font-size-base: 1rem;
  --radius: 8px;
}

.card {
  padding: var(--spacing-md);
  border-radius: var(--radius);
  border: 1px solid var(--color-primary);
  /* Variáveis com fallback: */
  color: var(--color-text, #e8e8f0); /* usa #e8e8f0 se --color-text não existir */
}

⚠️ Armadilhas Comuns

1. Esquecer box-sizing: border-box

O comportamento padrão content-box adiciona padding e border à largura, causando overflow inesperado. Sempre coloque o reset global *, *::before, *::after { box-sizing: border-box; } no topo do seu CSS.

2. Margin collapse — o susto mais comum

Margens verticais entre elementos block se fundem — a maior vence. Isso não acontece com padding, nem entre flex/grid items. Se precisar de espaço consistente, use gap no container flex/grid ou padding no pai em vez de margin nos filhos.

3. Usar height: 100% sem altura definida no pai

height: 100% em um filho só funciona se o pai tiver uma altura definida. Para layouts full-height, use min-height: 100dvh no body ou no container raiz, e flex: 1 / align-self: stretch nos filhos.

4. !important como solução padrão

!important quebra a cascata e cria CSS impossível de manter. Quando você sente vontade de usar, é sinal de que a especificidade está errada. Refatore o seletor para ser mais específico, ou reestruture o CSS.

5. Usar vh em mobile para alturas críticas

Em dispositivos móveis, 100vh inclui a barra de endereço do browser, causando conteúdo cortado. Use 100dvh (dynamic viewport height) para alturas críticas em mobile, com fallback para browsers antigos.

6. Confundir justify e align em Flexbox

justify-content alinha no eixo principal (direção do flex-direction). align-items alinha no eixo cruzado. Quando você muda flex-direction: column, os eixos invertem — o que alinhava horizontalmente agora alinha verticalmente.

Termos desta Aula

Box Model
Modelo que representa todo elemento HTML como uma caixa com quatro camadas: content, padding, border e margin.
Especificidade (Specificity)
Sistema de pontuação que determina qual regra CSS vence quando múltiplas regras se aplicam ao mesmo elemento.
Flex Container / Flex Item
O elemento com display: flex é o flex container; seus filhos diretos são os flex items.
fr (fraction unit)
Unidade de CSS Grid que representa uma fração do espaço disponível após subtrair gaps e trilhas fixas.
Grid Line / Grid Track / Grid Cell / Grid Area
Grid Line: linha divisória numerada. Track: espaço entre duas linhas (coluna ou linha). Cell: interseção de uma track de linha com uma de coluna. Area: retângulo formado por múltiplas cells.
Margin Collapse
Fusão de margens verticais adjacentes entre elementos block — a maior margem vence, não somam.
clamp(min, ideal, max)
Função CSS que retorna o valor ideal se estiver entre min e max, caso contrário retorna o limite atingido. Usada para values fluidos com limites.

🏁 Checkpoint — Desafio Final

Construa uma landing page de produto usando exclusivamente HTML semântico e CSS (sem frameworks), com:

  • Header fixo com logo à esquerda e nav + CTA à direita (Flexbox, position: sticky)
  • Hero section com texto e imagem lado a lado em telas grandes, empilhados em mobile (Grid ou Flexbox)
  • Seção de features: grade de 3 colunas que vira 1 coluna no mobile com auto-fit + minmax — sem media query
  • Seção de preços: 3 cards usando Grid, o card central com destaque (border, escala ou cor diferente)
  • Footer com layout de 4 colunas que colapsa para 2 e depois 1 — apenas com media queries
  • Todas as unidades de espaçamento em rem, font-sizes com clamp(), cores em variáveis CSS
💬 Dica: comece pelo mobile-first — estilize para tela pequena e use media queries para expandir para telas maiores. min-width em vez de max-width. Isso resulta em CSS mais limpo e componentes mais resilientes.
