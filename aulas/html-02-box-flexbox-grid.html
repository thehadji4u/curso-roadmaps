CSS — Box Model, Flexbox & Grid

⏱ ~35 min de leitura 🎯 Fundamentos 🔧 Hands-on incluído
box-model flexbox css-grid cascade specificity border-box :is() :has()
// 01 — motivação

Por que isso importa?

Todo elemento HTML é uma caixa retangular. Não importa se é um parágrafo, um botão ou um card de produto — o browser enxerga tudo como caixas empilhadas ou enfileiradas. Se você não entende como essas caixas se comportam, o CSS vai parecer mágica negra.

Box Model, Flexbox e Grid são os três pilares do layout CSS. Box Model define o espaçamento interno e externo de cada elemento. Flexbox organiza itens em uma dimensão (linha ou coluna). Grid organiza em duas dimensões simultaneamente. Cascade & Specificity explicam por que um estilo prevalece sobre outro — o debugger mais importante do CSS.

💼 Por que um engenheiro sênior usa isso hoje
Todo time de produto tem debates sobre layout. Sêniors escolhem grid para layouts de página e flexbox para componentes internos — e justificam essa escolha. Eles também debugam conflitos de especificidade sem chutar: calculam o score e sabem exatamente qual regra vai ganhar. Nunca usam !important como solução.
// 02 — analogia

Antes do código: uma analogia

🌍 Mundo Real → Tecnologia
Imagine uma exposição de arte. Cada quadro tem seu próprio suporte embutido (padding), uma moldura que o circunda (border), e a distância que o separa do quadro ao lado (margin). A tela pintada em si é o content.

Os quadros numa parede lado a lado são Flexbox — o curador decide se ficam numa linha ou numa coluna, como se alinham, se o espaço entre eles é igual ou se algum ocupa mais área.

A planta baixa do museu é o Grid — salas definidas em linhas e colunas, onde cada peça pode ocupar uma ou mais salas ao mesmo tempo.

E a Cascade é a hierarquia de decisões do curador: as regras do museu nacional prevalecem sobre as do museu local, que prevalecem sobre a preferência do artista individual.
Tela do quadro → content Suporte embutido → padding Moldura → border Espaço entre quadros → margin Curador alinhando obras → Flexbox Planta baixa do museu → CSS Grid Hierarquia de regras → Cascade & Specificity
// 03 — box model

Box Model — a anatomia de todo elemento

Todo elemento CSS é composto de quatro camadas concêntricas. De dentro para fora: content → padding → border → margin.

margin
border
padding
content (width × height)

content-box vs border-box

O CSS tem dois modelos de cálculo de largura, definidos pela propriedade box-sizing. Este é um dos maiores causadores de bugs em iniciantes:

box-sizing Como width é calculado Comportamento Recomendação
content-box width = apenas o conteúdo. padding e border são adicionados por fora Um div com width:200px; padding:20px ocupa 240px no total ❌ Padrão do browser — evite usar sozinho
border-box width = content + padding + border juntos. Tudo fica dentro do valor declarado Um div com width:200px; padding:20px ocupa exatamente 200px ✅ Use sempre — ative globalmente com *
🔥 Pro tip — ative border-box globalmente
Todo projeto CSS profissional começa com este reset. Coloque-o no topo do seu CSS principal e esqueça os bugs de tamanho para sempre:

*, *::before, *::after { box-sizing: border-box; }

📄 box-model.css CSS 
/* ─── Reset universal: SEMPRE coloque isso primeiro ─── */
*, *::before, *::after {
  box-sizing: border-box;
}

/* ─── Exemplo prático ─── */
.card {
  width: 300px;            /* com border-box: card ocupa exatamente 300px */
  padding: 24px;           /* espaço interno — não aumenta a largura total  */
  border: 2px solid #ef4444; /* borda — incluída nos 300px                 */
  margin: 16px auto;       /* espaço externo + centralizado horizontalmente */
}

/* ─── Colapso de margin: margem entre dois elementos ─── */
/* Quando dois elementos com margin se tocam verticalmente,
   a margem NÃO soma: vence o maior valor (colapso de margin) */
.titulo  { margin-bottom: 32px; }
.subtitulo { margin-top: 16px; }
/* Espaço real entre eles: 32px (não 48px!) */

/* ─── Margin auto: centralizar horizontalmente ─── */
.container {
  width: 860px;
  max-width: 100%;
  margin: 0 auto;   /* left e right auto = centralizado */
}
// 04 — flexbox

Flexbox — layout em uma dimensão

Flexbox (Flexible Box Layout) resolve o problema de distribuir e alinhar itens em uma única direção — linha ou coluna. É ideal para componentes: navbars, cards, botões agrupados, listas de itens.

O modelo tem dois atores: o flex container (pai que recebe display: flex) e os flex items (filhos diretos). Propriedades do container controlam o layout coletivo; propriedades do item controlam o comportamento individual.

// flex-direction: row (padrão) — justify-content: space-between
container
item 1
item 2
item 3
flex-direction: column — align-items: center
item A
item B
item C

Propriedades do container

Propriedade Valores principais O que faz
flex-direction row · column · row-reverse Define o eixo principal (horizontal ou vertical)
justify-content flex-start · center · space-between · space-around Alinha no eixo principal
align-items stretch · center · flex-start · flex-end Alinha no eixo cruzado (perpendicular)
flex-wrap nowrap · wrap · wrap-reverse Quebra ou não os itens em múltiplas linhas
gap 8px · 16px 8px Espaçamento entre itens (substituiu margin hacks)

Propriedades do item

Propriedade Valores O que faz
flex-grow 0 (padrão) · 1 · 2 Quanto o item cresce para preencher espaço livre. 1 = ocupa todo o espaço disponível
flex-shrink 1 (padrão) · 0 Se o item pode encolher quando falta espaço. 0 = nunca encolhe
flex-basis auto · 200px · 30% Tamanho inicial antes de grow/shrink serem aplicados
flex 1 = 1 1 0% Shorthand de grow + shrink + basis. Use sempre o shorthand
order 0 (padrão) · valores negativos e positivos Reordena itens visualmente sem alterar o HTML
align-self mesmos valores de align-items Sobrescreve o align-items do container só para este item
📄 flexbox.css CSS 
/* ─── Navbar clássica com flexbox ─── */
.navbar {
  display: flex;
  align-items: center;
  justify-content: space-between;  /* logo à esquerda, links à direita */
  padding: 0 24px;
  height: 64px;
  gap: 16px;
}

/* ─── Card grid responsivo com flex-wrap ─── */
.cards {
  display: flex;
  flex-wrap: wrap;        /* quebra em nova linha quando necessário */
  gap: 20px;
}

.card {
  flex: 1 1 280px;        /* grow=1, shrink=1, base=280px */
  /* cada card cresce, encolhe, mas nunca fica menor que 280px */
}

/* ─── Centralizar qualquer coisa com flex ─── */
.center-tudo {
  display: flex;
  align-items: center;      /* eixo cruzado: vertical */
  justify-content: center;  /* eixo principal: horizontal */
  min-height: 100vh;        /* ocupa toda a altura da viewport */
}

/* ─── Barra lateral que não encolhe ─── */
.layout {
  display: flex;
  gap: 24px;
}
.sidebar { flex: 0 0 240px; }  /* não cresce (0), não encolhe (0), sempre 240px */
.main    { flex: 1; }           /* ocupa todo o restante */
// 05 — css grid

CSS Grid — layout em duas dimensões

Enquanto Flexbox trabalha em uma direção por vez, CSS Grid define linhas e colunas simultaneamente. É o sistema de layout mais poderoso do CSS e ideal para estruturas de página: header, sidebar, main, footer.

// grid-template-columns: repeat(3, 1fr)
header (span 3)
item 1
item 2
item 3
item 4 (span 2)
item 5
footer (span 3)

Propriedades essenciais do Grid

Propriedade Exemplo O que faz
grid-template-columns 1fr 2fr 1fr Define as colunas. fr = fração do espaço disponível
grid-template-rows auto 1fr auto Define as linhas. auto = tamanho pelo conteúdo
grid-template-areas "header header" "sidebar main" Nomeia áreas para posicionar itens por nome (muito legível)
gap 20px · 16px 24px Espaçamento entre linhas e colunas
repeat() repeat(4, 1fr) Repete padrões — evita repetição manual
auto-fit + minmax() repeat(auto-fit, minmax(200px, 1fr)) Grid responsivo automático: cria quantas colunas couberem no espaço disponível
grid-column 1 / 3 ou span 2 Item ocupa da coluna 1 até a 3 (ou se expande 2 colunas)
subgrid grid-template-columns: subgrid Item filho herda as trilhas do grid pai — alinha conteúdo interno ao grid externo
📄 grid-layout.css CSS 
/* ─── Layout de página clássico com grid-areas ─── */
.app {
  display: grid;
  grid-template-areas:
    "header  header"
    "sidebar main"
    "footer  footer";
  grid-template-columns: 240px 1fr;
  grid-template-rows: 64px 1fr 48px;
  min-height: 100vh;
  gap: 0;
}

.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }
.footer  { grid-area: footer; }

/* ─── Grid de cards totalmente responsivo (sem media queries!) ─── */
.cards-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
  gap: 24px;
  /* cria automaticamente 1, 2, 3 ou mais colunas
     conforme o espaço disponível — cada coluna min 280px */
}

/* ─── Subgrid: alinhar conteúdo interno ao grid pai ─── */
.card-grid-pai {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 16px;
}

.card {
  display: grid;
  grid-template-rows: subgrid;  /* herda linhas do pai */
  grid-row: span 3;             /* ocupa 3 linhas do pai */
  /* resultado: imagem, título e texto de todos os cards
     ficam alinhados horizontalmente, mesmo com alturas diferentes */
}
💡 Flexbox vs Grid — quando usar cada um?
Flexbox: quando o conteúdo dita o layout. "Quero que esses itens fiquem em linha e se distribuam bem." → navbars, grupos de botões, listas de chips, layouts de componente interno.

Grid: quando o layout dita o conteúdo. "Preciso de 3 colunas e o segundo item deve sempre ocupar 2 delas." → página inteira, dashboard, galeria de fotos, card com campos alinhados.
// 06 — cascade & specificity

Cascade & Specificity — quem ganha o conflito?

Quando duas regras CSS tentam estilizar o mesmo elemento, o browser precisa decidir qual vence. Essa decisão segue três critérios em ordem de importância: origem, especificidade e ordem de aparecimento.

Especificidade — o score

Cada seletor tem um score de especificidade representado como três números: (A, B, C).

(1,0,0) Seletor de ID #header, #main-nav
(0,1,0) Seletor de classe, pseudo-classe, atributo .card, :hover, [type="text"]
(0,0,1) Seletor de tag, pseudo-elemento div, p, ::before
(0,0,0) Seletor universal * { ... }
⚠️ Especificidade se soma — cuidado com seletores longos
nav ul li a.active tem score (0,1,4) — uma classe + quatro tags. Isso é muito específico e dificulta sobrescrever depois. Prefira seletores curtos com classes bem nomeadas: .nav-link--active tem score (0,1,0) e é muito mais fácil de manter.

:is(), :where() e :has() — seletores modernos

CSS moderno oferece pseudo-classes que simplificam seletores complexos e controlam a especificidade de forma precisa:

Pseudo-classe Especificidade Uso Exemplo
:is() Herda do seletor mais específico dentro dela Agrupa seletores — reduz repetição de código :is(h1, h2, h3) { margin: 0; }
:where() Sempre zero — não contribui para especificidade Agrupa sem aumentar especificidade. Ideal para resets e utilitários :where(article, section) p { ... }
:has() Herda do seletor interno (como :is()) "Seletor pai" — estiliza um elemento baseado no que ele contém figure:has(figcaption) { ... }
📄 cascade-modern.css CSS 
/* ─── :is() — agrupar seletores sem repetição ─── */
/* antes (repetitivo e sujeito a erro) */
h1, h2, h3, h4, h5, h6 { font-family: 'Syne', sans-serif; }

/* depois (mais limpo) */
:is(h1, h2, h3, h4, h5, h6) { font-family: 'Syne', sans-serif; }

/* ─── :where() — reset de zero especificidade ─── */
/* perfeito para CSS de biblioteca: fácil de sobrescrever */
:where(button, input, select, textarea) {
  font-family: inherit;
  font-size: inherit;
}

/* ─── :has() — o "seletor pai" que o CSS nunca teve ─── */

/* card com imagem: layout diferente de card sem imagem */
.card:has(img) {
  display: grid;
  grid-template-columns: 120px 1fr;
}

/* label de campo inválido muda de cor */
label:has(+ input:invalid) {
  color: #ef4444;
}

/* formulário com campos obrigatórios não preenchidos: botão desabilitado */
form:has(input:required:placeholder-shown) .btn-submit {
  opacity: 0.4;
  pointer-events: none;
}

/* ─── Cascade layers — controle explícito da ordem ─── */
@layer reset, base, components, utilities;

@layer reset {
  *, *::before, *::after { box-sizing: border-box; }
}

@layer base {
  body { font-family: 'JetBrains Mono', monospace; }
}

@layer utilities {
  .mt-auto { margin-top: auto; }
}
/* Regras em "utilities" sempre ganham de regras em "base",
   independente da especificidade — é a ordem da layer que decide */

// 08 — armadilhas

Erros que todo mundo comete

⚠️ Esquecer box-sizing: border-box
Sem esse reset, padding e border aumentam a largura total do elemento de forma inesperada. Qualquer cálculo de layout quebra. Sempre comece com *, *::before, *::after { box-sizing: border-box; }.
⚠️ Usar margin em vez de gap no Flexbox/Grid
margin em flex items causa problemas: o último item não tem margem e o primeiro também não, então você usa margin: 0 8px e tenta compensar. Com gap: 16px no container, o espaçamento é uniforme e sem efeitos colaterais. Sempre prefira gap.
⚠️ Usar !important para resolver conflito de especificidade
!important não resolve o problema — apenas mascara. Quando dois seletores têm !important, o de maior especificidade vence. E agora você tem dois bugs em vez de um. A solução real é entender o score e reescever o seletor.
⚠️ Usar Grid para tudo ou Flexbox para tudo
Grid e Flexbox não são concorrentes — são ferramentas complementares. Navbars e componentes internos → Flexbox. Layout de página e alinhamento multi-eixo → Grid. Projetos reais usam os dois juntos.

🏁 Checkpoint — valide seu aprendizado

Construa um layout de perfil de usuário com as seguintes características:

  • Ative box-sizing: border-box globalmente e use margin/padding de forma consciente em cada elemento
  • Use CSS Grid para o layout geral da página: header, sidebar com info do perfil e área de posts
  • Use Flexbox dentro da sidebar para alinhar avatar + nome + bio verticalmente e centralizados
  • Crie um grid de posts (auto-fit, minmax()) que seja responsivo sem media queries
  • Aplique pelo menos um seletor :has() — por exemplo: card que tem imagem recebe layout diferente
