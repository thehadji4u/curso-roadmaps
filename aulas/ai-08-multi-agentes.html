CrewAI & LangGraph

// por que isso importa?

Um agente sozinho não é suficiente

Um único agente LLM tem limitações claras: contexto limitado, falta de especialização profunda, e dificuldade em manter estado consistente em tarefas longas. Para problemas complexos — como "analise este mercado, escreva um relatório e poste no LinkedIn" — a solução é dividir para conquistar.

Sistemas multi-agente permitem que agentes especializados colaborem: um pesquisa, outro analisa, outro escreve, outro revisa. Cada um faz o que faz melhor, com um orquestrador coordenando o fluxo.

🔥 Por que um engenheiro sênior usa hoje?
Empresas como Deloitte, McKinsey e Bloomberg já usam sistemas multi-agente para análise financeira, geração de relatórios e due diligence. É a arquitetura de IA mais importante de 2025.
// analogia do mundo real

A Equipe de Consultoria

🏢 Analogia
Imagine contratar uma única pessoa para fazer toda uma consultoria: ela pesquisa o mercado, analisa dados, escreve o relatório, revisa, e apresenta ao cliente. Impossível ser excelente em tudo. Agora imagine uma firma de consultoria: um pesquisador de mercado, um analista financeiro, um redator sênior, e um gerente de projeto coordenando tudo. Cada um especialista na sua área, trabalhando em paralelo, entregando qualidade superior. Sistemas multi-agente são essa firma de consultoria.
🎯 Gerente de Projeto
Orchestrator / Manager Agent
🔍 Pesquisador de Mercado
Research Agent (web search tools)
📊 Analista Financeiro
Analyst Agent (pandas/code tools)
✍️ Redator Sênior
Writer Agent (output formatter)
🔎 Revisor de Qualidade
Critic Agent (guardrail checker)

Padrões de Design Multi-Agente

Existem quatro padrões fundamentais em sistemas multi-agente. Entender qual usar em cada situação é a habilidade-chave do engenheiro de IA.

Diagrama — Os 4 Padrões de Orquestração
flowchart TD subgraph P1["🔗 Sequential (Pipeline)"] A1["Agent A"] -->|output| B1["Agent B"] -->|output| C1["Agent C"] end subgraph P2["⭐ Hierarchical (Manager)"] M["Manager Agent"] M --> W1["Worker 1"] M --> W2["Worker 2"] M --> W3["Worker 3"] W1 & W2 & W3 -->|results| M end subgraph P3["🔄 Collaborative (Debate)"] D1["Agent A\n(propose)"] <-->|critique| D2["Agent B\n(critique)"] D2 -->|improved| D3["Consensus\nOutput"] end subgraph P4["📊 LangGraph (State Machine)"] S["State"] --> N1["Node 1"] N1 --> COND{"Condition"} COND -->|path A| N2["Node 2"] COND -->|path B| N3["Node 3"] N2 & N3 -->|update| S end style M fill:#a855f7,color:#fff style S fill:#06b6d4,color:#fff

🔗 Sequential Pipeline

Saída de um agente é entrada do próximo. Simples, previsível. Ideal para: transformações lineares de dados.

⭐ Hierarchical (Manager)

Um Manager delega tarefas e agrega resultados. Ideal para: divisão de trabalho paralela com supervisor.

🔄 Collaborative / Debate

Agentes debatem e criticam uns aos outros até consenso. Ideal para: melhorar qualidade de outputs.

📊 LangGraph State Machine

Estado compartilhado + nós com lógica + arestas condicionais. Ideal para: workflows complexos com branching.

// framework 1 — crewai

CrewAI: Tripulação de Agentes

CrewAI usa metáforas de equipe: você define Agentes com papéis, backstories e objetivos, e Tasks com descrições e outputs esperados. O framework orquestra automaticamente o fluxo.

bash
$ pip install crewai crewai-tools
$ pip install langchain-openai duckduckgo-search
crew_research.py — Time de análise de mercado
import os
from crewai import Agent, Task, Crew, Process
from crewai_tools import SerperDevTool, WebsiteSearchTool
from langchain_openai import ChatOpenAI

# Configuração do LLM
llm = ChatOpenAI(model="gpt-4o", temperature=0.2)

# Ferramentas disponíveis
search_tool = SerperDevTool()  # Google Search via Serper API
web_tool = WebsiteSearchTool()


# ─── AGENTES ────────────────────────────────────────────────────────────────

pesquisador = Agent(
    role="Analista de Mercado Sênior",
    goal=(
        "Coletar dados abrangentes e factuais sobre o mercado de {topic}. "
        "Foco em tendências recentes, players principais, e oportunidades."
    ),
    backstory=(
        "Você tem 15 anos de experiência em pesquisa de mercado. "
        "É meticuloso com fontes, sempre verifica fatos antes de reportar, "
        "e tem experiência no setor de tecnologia."
    ),
    tools=[search_tool, web_tool],
    llm=llm,
    verbose=True,
    allow_delegation=False,
    max_iter=5  # Máximo de iterações para evitar loops
)

analista = Agent(
    role="Analista Estratégico",
    goal=(
        "Analisar os dados coletados sobre {topic} e identificar insights "
        "estratégicos, oportunidades e riscos."
    ),
    backstory=(
        "Consultor estratégico com MBA pela Wharton e experiência em BCG. "
        "Transforma dados em estratégias acionáveis usando frameworks como "
        "SWOT, Porter's Five Forces e análise PESTLE."
    ),
    llm=llm,
    verbose=True,
    allow_delegation=False
)

redator = Agent(
    role="Redator Executivo",
    goal=(
        "Transformar análises técnicas sobre {topic} em relatório executivo "
        "claro, conciso e persuasivo para tomada de decisão."
    ),
    backstory=(
        "Jornalista econômico com 10 anos de experiência na The Economist. "
        "Especialista em traduzir complexidade para clareza executiva."
    ),
    llm=llm,
    verbose=True,
    allow_delegation=False
)


# ─── TASKS ──────────────────────────────────────────────────────────────────

task_pesquisa = Task(
    description=(
        "Pesquise o mercado de {topic} em profundidade. Colete:\n"
        "1. Tamanho do mercado e projeção de crescimento (TAM/SAM/SOM)\n"
        "2. Top 5 players e suas market shares\n"
        "3. Principais tendências de 2024-2025\n"
        "4. Regulamentações relevantes\n"
        "5. Casos de uso mais adotados\n\n"
        "Use fontes recentes (últimos 6 meses). Cite URLs."
    ),
    expected_output=(
        "Relatório de pesquisa estruturado em markdown com dados quantitativos, "
        "URLs das fontes, e organizado nas 5 seções solicitadas."
    ),
    agent=pesquisador
)

task_analise = Task(
    description=(
        "Com base na pesquisa fornecida, realize análise estratégica do mercado de {topic}:\n"
        "1. Análise SWOT completa\n"
        "2. 3 oportunidades de entrada no mercado rankeadas por potencial\n"
        "3. Principais riscos e como mitigá-los\n"
        "4. Recomendação estratégica para uma startup entrando neste mercado"
    ),
    expected_output=(
        "Análise estratégica em markdown com SWOT visual, análise de oportunidades "
        "com critérios objetivos e recomendação final justificada."
    ),
    agent=analista,
    context=[task_pesquisa]  # Recebe output da task anterior
)

task_relatorio = Task(
    description=(
        "Compile pesquisa e análise num relatório executivo profissional sobre {topic}. "
        "O relatório deve:\n"
        "- Ter no máximo 2 páginas\n"
        "- Usar linguagem executiva (não técnica)\n"
        "- Incluir sumário executivo de 3 bullets\n"
        "- Terminar com próximos passos recomendados\n"
        "- Formato: markdown com headers claros"
    ),
    expected_output=(
        "Relatório executivo completo em markdown, pronto para apresentação "
        "a investidores ou board. Máximo 800 palavras."
    ),
    agent=redator,
    context=[task_pesquisa, task_analise],
    output_file="relatorio_mercado.md"  # Salva automaticamente
)


# ─── CREW ───────────────────────────────────────────────────────────────────

crew = Crew(
    agents=[pesquisador, analista, redator],
    tasks=[task_pesquisa, task_analise, task_relatorio],
    process=Process.sequential,  # ou Process.hierarchical
    verbose=2,  # 0=silencioso, 1=basic, 2=detalhado
    memory=True,  # Mantém memória entre runs
    max_rpm=10   # Rate limit para APIs
)


if __name__ == "__main__":
    resultado = crew.kickoff(inputs={"topic": "IA Generativa no Brasil"})
    print("\n" + "="*60)
    print("RELATÓRIO FINAL:")
    print("="*60)
    print(resultado)
💡 Process.hierarchical vs sequential
Sequential: cada task executa em ordem, output vira input da próxima. Hierarchical: um Manager Agent (criado automaticamente) decide qual Worker deve realizar cada task e em que ordem. Use hierarchical quando as tasks podem se beneficiar de ordem dinâmica.

CrewAI com Tools Customizadas

Você pode criar tools próprias com o decorator @tool:

custom_tools.py
from crewai_tools import BaseTool
from pydantic import BaseModel, Field
import httpx


class BuscaEmpresaInput(BaseModel):
    cnpj: str = Field(description="CNPJ da empresa no formato 00.000.000/0000-00")


class BuscaEmpresaTool(BaseTool):
    name: str = "Buscar Dados de Empresa"
    description: str = (
        "Busca dados cadastrais de uma empresa brasileira pelo CNPJ. "
        "Retorna razão social, situação cadastral, porte e atividade principal."
    )
    args_schema: type[BaseModel] = BuscaEmpresaInput

    def _run(self, cnpj: str) -> str:
        """Consulta API pública da Receita Federal."""
        cnpj_limpo = cnpj.replace(".", "").replace("/", "").replace("-", "")
        resp = httpx.get(
            f"https://brasilapi.com.br/api/cnpj/v1/{cnpj_limpo}",
            timeout=10
        )
        if resp.status_code != 200:
            return f"CNPJ não encontrado: {cnpj}"

        data = resp.json()
        return (
            f"Razão Social: {data.get('razao_social')}\n"
            f"Situação: {data.get('descricao_situacao_cadastral')}\n"
            f"Porte: {data.get('porte')}\n"
            f"Atividade: {data.get('cnae_fiscal_descricao')}\n"
            f"Município: {data.get('municipio')}/{data.get('uf')}"
        )


# Usar a tool customizada
busca_empresa = BuscaEmpresaTool()
agente_compliance = Agent(
    role="Analista de Compliance",
    goal="Verificar situação cadastral de empresas parceiras",
    tools=[busca_empresa],
    llm=llm
)
// framework 2 — langgraph

LangGraph: State Machines para Agentes

Enquanto CrewAI é alto nível e orientado a "equipes", LangGraph é baixo nível e orientado a grafos de estado. Você define: State (dados compartilhados), Nodes (funções que transformam o estado) e Edges (conexões, incluindo condicionais).

É a escolha de engenheiros quando precisam de controle total sobre o fluxo do agente.

bash
$ pip install langgraph langchain-openai langchain-community
langgraph_agent.py — Agente de pesquisa com human-in-the-loop
import json
from typing import TypedDict, Annotated, Sequence
from langchain_core.messages import BaseMessage, HumanMessage, AIMessage, ToolMessage
from langchain_openai import ChatOpenAI
from langchain_community.tools.ddg_search import DuckDuckGoSearchRun
from langgraph.graph import StateGraph, END
from langgraph.prebuilt import ToolNode
import operator


# ─── STATE ──────────────────────────────────────────────────────────────────
# Define o estado compartilhado entre todos os nós do grafo

class AgentState(TypedDict):
    messages: Annotated[Sequence[BaseMessage], operator.add]  # acumula mensagens
    next_step: str        # controla o fluxo
    pesquisa_concluida: bool
    relatorio_draft: str
    aprovado_humano: bool
    iteracoes: int


# ─── TOOLS ──────────────────────────────────────────────────────────────────

search = DuckDuckGoSearchRun()
tools = [search]

llm = ChatOpenAI(model="gpt-4o", temperature=0)
llm_com_tools = llm.bind_tools(tools)


# ─── NODES (funções que transformam o estado) ────────────────────────────────

def node_pesquisar(state: AgentState) -> dict:
    """Nó de pesquisa: usa o LLM com tools para buscar informações."""
    print("🔍 Pesquisando...")
    response = llm_com_tools.invoke(state["messages"])
    return {
        "messages": [response],
        "iteracoes": state.get("iteracoes", 0) + 1
    }


def node_executar_tools(state: AgentState) -> dict:
    """Nó executor de tools: roda as tool calls do LLM."""
    tool_node = ToolNode(tools)
    result = tool_node.invoke(state)
    return result


def node_analisar(state: AgentState) -> dict:
    """Nó de análise: sintetiza os dados coletados."""
    print("📊 Analisando dados...")
    msgs = state["messages"]

    prompt = HumanMessage(content=
        "Com base em todas as pesquisas acima, escreva um draft de relatório "
        "em 3 parágrafos: Contexto, Principais Achados e Conclusão. "
        "Seja específico e use os dados encontrados."
    )

    response = llm.invoke(msgs + [prompt])
    return {
        "messages": [response],
        "relatorio_draft": response.content,
        "pesquisa_concluida": True
    }


def node_revisao_humana(state: AgentState) -> dict:
    """
    Nó de revisão humana (Human-in-the-Loop).
    Em produção, isso pausaria o grafo e esperaria input do usuário.
    """
    print("\n" + "="*50)
    print("📝 DRAFT DO RELATÓRIO:")
    print("="*50)
    print(state["relatorio_draft"])
    print("="*50)

    # Simular aprovação (em produção: input real do usuário via UI/webhook)
    aprovado = input("\n✅ Aprovar relatório? (s/n): ").strip().lower() == "s"

    return {"aprovado_humano": aprovado}


def node_finalizar(state: AgentState) -> dict:
    """Nó final: formata e salva o relatório."""
    print("✅ Finalizando relatório...")
    relatorio_final = f"""
# RELATÓRIO DE ANÁLISE
**Gerado automaticamente por LangGraph Agent**

{state["relatorio_draft"]}

---
*Iterações de pesquisa: {state.get('iteracoes', 0)}*
*Status: {'Aprovado pelo Revisor' if state['aprovado_humano'] else 'Publicado sem revisão'}*
    """
    with open("relatorio_final.md", "w") as f:
        f.write(relatorio_final)

    return {"messages": [AIMessage(content="Relatório salvo em relatorio_final.md")]}


def node_refinar(state: AgentState) -> dict:
    """Nó de refinamento: melhora o relatório com base no feedback."""
    print("🔄 Refinando relatório...")
    msgs = state["messages"]

    prompt = HumanMessage(content=
        "O relatório foi rejeitado. Por favor, revise-o tornando-o mais "
        "detalhado, com dados mais específicos e conclusões mais claras."
    )

    response = llm.invoke(msgs + [prompt])
    return {
        "messages": [response],
        "relatorio_draft": response.content,
        "aprovado_humano": False
    }


# ─── EDGES CONDICIONAIS ──────────────────────────────────────────────────────

def decidir_apos_pesquisa(state: AgentState) -> str:
    """Decide o próximo nó após pesquisa."""
    msgs = state["messages"]
    ultima_msg = msgs[-1]

    # Se o LLM chamou tools, executá-las
    if hasattr(ultima_msg, "tool_calls") and ultima_msg.tool_calls:
        return "executar_tools"

    # Máximo de iterações atingido
    if state.get("iteracoes", 0) >= 3:
        return "analisar"

    # Continuar pesquisando
    return "analisar"


def decidir_apos_revisao(state: AgentState) -> str:
    """Decide o próximo nó após revisão humana."""
    if state["aprovado_humano"]:
        return "finalizar"
    return "refinar"


# ─── CONSTRUIR O GRAFO ───────────────────────────────────────────────────────

grafo = StateGraph(AgentState)

# Adicionar nós
grafo.add_node("pesquisar", node_pesquisar)
grafo.add_node("executar_tools", node_executar_tools)
grafo.add_node("analisar", node_analisar)
grafo.add_node("revisao_humana", node_revisao_humana)
grafo.add_node("finalizar", node_finalizar)
grafo.add_node("refinar", node_refinar)

# Definir ponto de entrada
grafo.set_entry_point("pesquisar")

# Arestas condicionais
grafo.add_conditional_edges(
    "pesquisar",
    decidir_apos_pesquisa,
    {
        "executar_tools": "executar_tools",
        "analisar": "analisar"
    }
)

grafo.add_conditional_edges(
    "revisao_humana",
    decidir_apos_revisao,
    {
        "finalizar": "finalizar",
        "refinar": "refinar"
    }
)

# Arestas diretas
grafo.add_edge("executar_tools", "pesquisar")
grafo.add_edge("analisar", "revisao_humana")
grafo.add_edge("refinar", "revisao_humana")
grafo.add_edge("finalizar", END)

# Compilar o grafo
app = grafo.compile()


# ─── EXECUTAR ────────────────────────────────────────────────────────────────

if __name__ == "__main__":
    resultado = app.invoke({
        "messages": [HumanMessage(
            content="Pesquise as principais tendências de IA no Brasil em 2025. "
                   "Foque em adoção corporativa e startups promissoras."
        )],
        "pesquisa_concluida": False,
        "relatorio_draft": "",
        "aprovado_humano": False,
        "iteracoes": 0
    })
    print("\n🏁 Pipeline concluído!")

// guardrails & avaliação

Controlando Qualidade e Segurança

Sistemas multi-agente podem aluciná-los mutuamente — um agente inventa algo e o próximo usa como fato. Guardrails são verificações que interceptam e bloqueiam outputs problemáticos.

guardrails.py — Validação de outputs com Pydantic + LLM
from pydantic import BaseModel, field_validator
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
import re


# ─── GUARDRAIL 1: Pydantic Schema Validation ────────────────────────────────

class RelatorioOutput(BaseModel):
    titulo: str
    sumario_executivo: str  # Máximo 3 bullets
    achados: list[str]       # Pelo menos 3 achados com dados
    conclusao: str
    fontes: list[str]        # Mínimo 2 URLs

    @field_validator("sumario_executivo")
    @classmethod
    def validar_sumario(cls, v):
        bullets = v.count("•") + v.count("-") + v.count("*")
        if bullets > 5:
            raise ValueError("Sumário deve ter no máximo 5 bullets")
        return v

    @field_validator("fontes")
    @classmethod
    def validar_fontes(cls, v):
        urls = [f for f in v if f.startswith("http")]
        if len(urls) < 2:
            raise ValueError("Necessário mínimo 2 URLs como fonte")
        return v

    @field_validator("achados")
    @classmethod
    def validar_achados_tem_dados(cls, v):
        # Verifica se pelo menos metade dos achados tem números
        com_dados = sum(1 for a in v if re.search(r'\d+[%$R$€]?', a))
        if com_dados < len(v) // 2:
            raise ValueError("Achados devem incluir dados quantitativos")
        return v


# ─── GUARDRAIL 2: LLM-as-Judge ──────────────────────────────────────────────

class QualityScore(BaseModel):
    score: int  # 0-10
    problemas: list[str]
    aprovado: bool


llm_judge = ChatOpenAI(model="gpt-4o", temperature=0)

JUDGE_PROMPT = ChatPromptTemplate.from_messages([
    ("system", """Você é um juiz de qualidade de relatórios executivos.
    Avalie o relatório nos seguintes critérios (0-10 cada):
    - Factualidade (sem afirmações sem evidência)
    - Clareza (linguagem executiva, sem jargão desnecessário)
    - Completude (cobre todos os aspectos solicitados)
    - Fontes (cita fontes verificáveis)

    Responda em JSON: {{"score": 7, "problemas": ["..."], "aprovado": true}}
    Aprovado se score >= 7 e sem problemas críticos."""),
    ("human", "Relatório para avaliar:\n\n{relatorio}")
])


async def avaliar_qualidade(relatorio: str) -> QualityScore:
    """Usa LLM como juiz para avaliar qualidade do output."""
    chain = JUDGE_PROMPT | llm_judge.with_structured_output(QualityScore)
    return await chain.ainvoke({"relatorio": relatorio})


# ─── GUARDRAIL 3: PII Detection ─────────────────────────────────────────────

PII_PATTERNS = [
    (r'\b\d{3}\.\d{3}\.\d{3}-\d{2}\b', "CPF"),
    (r'\b\d{2}\.\d{3}\.\d{3}/\d{4}-\d{2}\b', "CNPJ"),
    (r'\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b', "Email"),
    (r'\b(?:\+55\s?)?(?:\(?\d{2}\)?\s?)?\d{4,5}[-\s]?\d{4}\b', "Telefone"),
]


def detectar_pii(texto: str) -> list[dict]:
    """Detecta Informações Pessoais Identificáveis no output."""
    encontrados = []
    for pattern, tipo in PII_PATTERNS:
        matches = re.findall(pattern, texto)
        if matches:
            encontrados.append({"tipo": tipo, "ocorrencias": len(matches)})
    return encontrados


def aplicar_guardrails(output: str) -> tuple[bool, list[str]]:
    """
    Aplica todos os guardrails num output de agente.

    Returns:
        (passou, lista_de_problemas)
    """
    problemas = []

    # 1. PII check
    pii = detectar_pii(output)
    if pii:
        problemas.extend([f"PII detectado: {p['tipo']}" for p in pii])

    # 2. Tamanho mínimo
    if len(output.split()) < 100:
        problemas.append("Output muito curto (< 100 palavras)")

    # 3. Verifica se tem alucinações óbvias (números impossíveis)
    numeros = re.findall(r'\b(\d{1,3}(?:\.\d{3})*(?:,\d+)?)\s*%', output)
    for num in numeros:
        valor = float(num.replace(".", "").replace(",", "."))
        if valor > 100:
            problemas.append(f"Percentual impossível encontrado: {valor}%")

    passou = len(problemas) == 0
    return passou, problemas


# Exemplo de uso integrado com CrewAI
class RelatorioComGuardrail(Task):
    def execute(self, *args, **kwargs):
        output = super().execute(*args, **kwargs)

        passou, problemas = aplicar_guardrails(str(output))
        if not passou:
            print(f"⚠️ GUARDRAIL ACIONADO: {problemas}")
            # Re-executar com instrução de correção
            # ...

        return output

Pitfalls em Sistemas Multi-Agente

⚠️ Alucinação em cascata (Cascading Hallucination)

Agente A alucina um fato → Agente B usa como verdade → Agente C escreve relatório com dado falso. Solução: adicione um Critic Agent entre passos críticos que verifica claims com search tools antes de passar adiante.

⚠️ Context window overflow

Ao passar o output completo de um agente para o próximo, você pode estourar o contexto. Use summarization entre nós: antes de passar para o próximo agente, resuma o output. Em LangGraph, implemente um nó de sumarização se len(state["messages"]) > 20.

⚠️ Custo descontrolado em produção

Um Crew de 5 agentes com Process.hierarchical pode fazer 20+ chamadas ao GPT-4o por run. Monitore com langfuse ou langsmith. Regra: use GPT-4o apenas para tarefas de raciocínio complexo; use gpt-4o-mini para tarefas simples de formatação ou extração.

⚠️ Deadlock por dependências circulares

Em LangGraph, uma aresta condicional que volta para um nó anterior sem condição de saída cria loop infinito. Sempre adicione iteracoes no state com máximo definido como condição de escape.
