// POR QUE ISSO IMPORTA?

O coração de qualquer aplicação real

Toda aplicação JavaScript manipula dados — e dados em JS vivem em objetos e arrays. Uma API REST retorna JSON (objetos). Um carrinho de e-commerce é um array de objetos. Um estado de React é um objeto. Se você não dominar essas estruturas, vai travar exatamente onde o trabalho fica sério.

O ES6+ (ES2015 em diante) transformou completamente como escrevemos JavaScript. Arrow functions, destructuring, modules, classes — hoje esses recursos são obrigatórios para qualquer código profissional. Engenheiros sêniores usam essas features em 100% do código que escrevem no dia a dia.

🔥 Contexto sênior
Saber manipular arrays com map, filter e reduce é uma das habilidades mais testadas em entrevistas técnicas de front e back-end. Dominar objetos e o prototype chain é o que separa quem "usa" JavaScript de quem "entende" JavaScript.

// OBJETOS

Objetos: chave → valor

Um objeto JavaScript é uma coleção de pares chave: valor. As chaves são strings (ou Symbols), os valores podem ser qualquer tipo — inclusive funções (que chamamos de métodos).

javascript — criando e acessando objetos
// Notação literal — a forma mais comum
const usuario = {
  nome: 'Ana Silva',
  idade: 28,
  ativo: true,
  endereco: {              // objeto aninhado
    cidade: 'São Paulo',
    uf: 'SP'
  },
  saudar() {               // método (shorthand ES6)
    return `Olá, sou ${this.nome}`;
  }
};

// Duas formas de acessar propriedades
console.log(usuario.nome);          // 'Ana Silva'   — dot notation
console.log(usuario['nome']);       // 'Ana Silva'   — bracket notation (útil com variáveis)

// Adicionando e removendo propriedades
usuario.email = 'ana@email.com';    // adiciona
delete usuario.ativo;               // remove

// Verificar se propriedade existe
console.log('nome' in usuario);     // true
console.log(usuario.hasOwnProperty('email')); // true

Destructuring de objetos

Destructuring é a forma ES6 de extrair valores de objetos (e arrays) em variáveis de uma vez só. Elimina muito código repetitivo.

javascript — destructuring
const usuario = { nome: 'Ana', idade: 28, cidade: 'SP' };

// Sem destructuring (verbose)
const nome = usuario.nome;
const idade = usuario.idade;

// Com destructuring (ES6)
const { nome, idade } = usuario;

// Renomeando ao extrair
const { nome: nomeCompleto, cidade: uf } = usuario;
// nomeCompleto = 'Ana', uf = 'SP'

// Valor padrão se a propriedade não existir
const { nome, role = 'user' } = usuario;
// role = 'user' (porque não existe no objeto)

// Destructuring em parâmetros de função — muito comum em React!
function exibir({ nome, idade }) {
  return `${nome} tem ${idade} anos`;
}
exibir(usuario); // 'Ana tem 28 anos'

// Spread para copiar/mesclar objetos
const extra = { cargo: 'Dev', seniority: 'Sr' };
const completo = { ...usuario, ...extra };
// { nome, idade, cidade, cargo, seniority }
💡 Spread vs Object.assign
O spread {...obj} faz uma cópia shallow (rasa) do objeto — propriedades de objetos aninhados ainda apontam para a mesma referência. Para clones profundos use structuredClone(obj) (nativo moderno) ou JSON.parse(JSON.stringify(obj)).

Métodos úteis de Object

javascript — Object methods
const produto = { nome: 'Notebook', preco: 3500, estoque: 12 };

// Listar chaves, valores e entradas
Object.keys(produto);    // ['nome', 'preco', 'estoque']
Object.values(produto);  // ['Notebook', 3500, 12]
Object.entries(produto); // [['nome','Notebook'], ['preco',3500], ['estoque',12]]

// Iterar com entries (pattern muito usado)
Object.entries(produto).forEach(([chave, valor]) => {
  console.log(`${chave}: ${valor}`);
});

// Construir objeto a partir de entries
const produtoNovo = Object.fromEntries(
  Object.entries(produto).filter(([, v]) => typeof v === 'number')
);
// { preco: 3500, estoque: 12 }

// Congelar (imutabilidade rasa)
const config = Object.freeze({ apiUrl: 'https://api.exemplo.com' });
config.apiUrl = 'outro'; // silenciosamente ignorado (ou erro em strict mode)
// ARRAYS

Arrays: listas + transformações funcionais

Arrays são listas ordenadas de qualquer tipo de valor. Mas o que torna os arrays JS poderosos são os métodos de ordem superior (higher-order functions): map, filter, reduce, find, some, every. Eles recebem funções como argumento e são a base do estilo funcional em JS.

javascript — o trio essencial: map / filter / reduce
const produtos = [
  { nome: 'Notebook', preco: 3500, ativo: true },
  { nome: 'Mouse', preco: 150, ativo: true },
  { nome: 'Monitor', preco: 1800, ativo: false },
  { nome: 'Teclado', preco: 280, ativo: true },
];

// MAP — transforma cada elemento, retorna novo array do mesmo tamanho
const nomes = produtos.map(p => p.nome);
// ['Notebook', 'Mouse', 'Monitor', 'Teclado']

const comDesconto = produtos.map(p => ({
  ...p,
  precoFinal: p.preco * 0.9  // 10% desconto
}));

// FILTER — filtra por condição, retorna novo array menor (ou igual)
const ativos = produtos.filter(p => p.ativo);
// [Notebook, Mouse, Teclado]

const baratos = produtos.filter(p => p.preco < 500);
// [Mouse, Teclado]

// REDUCE — acumula valor único a partir do array
const totalEstoque = produtos.reduce((acc, p) => acc + p.preco, 0);
// 5730

// Encadeamento — legível e poderoso
const totalAtivos = produtos
  .filter(p => p.ativo)          // só ativos
  .map(p => p.preco)             // pega preços
  .reduce((acc, preco) => acc + preco, 0); // soma
// 3930

Outros métodos fundamentais

javascript — find, some, every, flat, flatMap
const nums = [1, 2, 3, 4, 5, 6];

// FIND — retorna o primeiro elemento que satisfaz a condição
const primeiroMaiorQue3 = nums.find(n => n > 3); // 4

// FINDINDEX — índice do primeiro que satisfaz
const idx = nums.findIndex(n => n > 3); // 3

// SOME — true se ALGUM satisfaz
nums.some(n => n > 5); // true

// EVERY — true se TODOS satisfazem
nums.every(n => n > 0); // true
nums.every(n => n > 3); // false

// FLAT — achata arrays aninhados
const nested = [1, [2, 3], [4, [5, 6]]];
nested.flat();    // [1, 2, 3, 4, [5, 6]]  — 1 nível
nested.flat(2);   // [1, 2, 3, 4, 5, 6]   — 2 níveis
nested.flat(Infinity); // achata tudo

// FLATMAP — map + flat(1) em um passo
const phrases = ['hello world', 'foo bar'];
phrases.flatMap(s => s.split(' ')); // ['hello', 'world', 'foo', 'bar']

// Destructuring de arrays
const [primeiro, segundo, ...resto] = nums;
// primeiro=1, segundo=2, resto=[3,4,5,6]

// Spread para concatenar
const a = [1, 2, 3];
const b = [4, 5, 6];
const c = [...a, ...b]; // [1,2,3,4,5,6]

// PROTOTYPE CHAIN

Como a herança funciona por baixo dos panos

Todo objeto em JavaScript tem uma propriedade interna chamada [[Prototype]] que aponta para outro objeto (ou null). Quando você acessa uma propriedade que não existe no objeto, o motor JS sobe essa cadeia procurando — isso é o prototype chain.

① cachorro (instância) nome: 'Rex', raça: 'Lab' — propriedades próprias
↑ [[Prototype]] Cachorro.prototype latir(), correr() — métodos compartilhados
↑ [[Prototype]] Object.prototype toString(), hasOwnProperty(), valueOf()
↑ [[Prototype]] null fim da cadeia — propriedade não encontrada → undefined
javascript — prototype chain na prática
// Função construtora (pré-ES6 — ainda muito comum em código legado)
function Animal(nome, tipo) {
  this.nome = nome;   // propriedade própria
  this.tipo = tipo;
}

// Método no prototype — compartilhado por TODAS as instâncias (eficiente!)
Animal.prototype.descrever = function() {
  return `${this.nome} é um ${this.tipo}`;
};

const gato = new Animal('Felix', 'gato');
const cao = new Animal('Rex', 'cachorro');

gato.descrever(); // 'Felix é um gato'

// Verificar de onde vem a propriedade
gato.hasOwnProperty('nome');     // true — é própria
gato.hasOwnProperty('descrever'); // false — está no prototype

// Object.create — herança explícita sem new
const animal = { respirar() { return 'respirando...'; } };
const peixe = Object.create(animal);
peixe.nome = 'Nemo';
peixe.respirar(); // 'respirando...' — vem do prototype

// Inspecionar a cadeia
Object.getPrototypeOf(peixe) === animal; // true
// CLASSES (ES6)

Classes: syntactic sugar sobre prototypes

Classes ES6 são uma sintaxe mais limpa para criar funções construtoras e configurar o prototype. Internamente, o mecanismo é exatamente o mesmo — prototypes. Classes não introduzem um modelo de herança diferente.

ES5 — Função Construtora
function Conta(dono, saldo) {
  this.dono = dono;
  this.saldo = saldo;
}
Conta.prototype.depositar = function(v) {
  this.saldo += v;
};
Conta.prototype.ver = function() {
  return `${this.dono}: R$${this.saldo}`;
};
var c = new Conta('Ana', 1000);
ES6 — Class
class Conta {
  constructor(dono, saldo) {
    this.dono = dono;
    this.saldo = saldo;
  }
  depositar(valor) {
    this.saldo += valor;
  }
  ver() {
    return `${this.dono}: R$${this.saldo}`;
  }
}
const c = new Conta('Ana', 1000);
javascript — herança com extends e campos privados
class Conta {
  #saldo = 0; // campo privado (ES2022) — não acessível fora da classe

  constructor(dono, saldoInicial = 0) {
    this.dono = dono;
    this.#saldo = saldoInicial;
  }

  depositar(valor) {
    if (valor <= 0) throw new Error('Valor inválido');
    this.#saldo += valor;
    return this;  // permite encadeamento: conta.depositar(100).depositar(50)
  }

  get saldo() { return this.#saldo; }  // getter — acesso read-only

  toString() {
    return `[Conta] ${this.dono}: R$${this.#saldo}`;
  }

  static criar(dono) {  // factory method estático
    return new Conta(dono);
  }
}

// Herança
class ContaPoupanca extends Conta {
  #taxaJuros;

  constructor(dono, saldo, taxa) {
    super(dono, saldo);  // OBRIGATÓRIO chamar super() antes de usar this
    this.#taxaJuros = taxa;
  }

  aplicarJuros() {
    this.depositar(this.saldo * this.#taxaJuros);
    return this;
  }
}

const poupanca = new ContaPoupanca('Bob', 5000, 0.005);
poupanca.aplicarJuros();
console.log(poupanca.saldo); // 5025

// Verificar hierarquia
poupanca instanceof ContaPoupanca; // true
poupanca instanceof Conta;          // true

Modules: organizando código em arquivos

Antes dos modules ES6, JavaScript não tinha sistema nativo de módulos — você dependia de soluções como CommonJS (require) ou AMD. Com ES6, a linguagem ganhou import / export nativos, que hoje são o padrão universal (navegador + Node.js + Deno).

📊 Diagrama — Sistema de Módulos
flowchart TD A["utils/math.js\nexport { soma, multiplica }"] --> C["app.js"] B["utils/api.js\nexport default fetchUser\nexport { parseData }"] --> C D["config.js\nexport const API_URL"] --> C C --> E["bundle final (Vite/webpack)"]
javascript — exports e imports
// ──── utils/math.js ────
// Named exports — pode ter vários
export const PI = 3.14159;

export function soma(a, b) {
  return a + b;
}

export function multiplica(a, b) {
  return a * b;
}

// ──── utils/api.js ────
// Default export — só um por arquivo
export default async function fetchUsuarios(url) {
  const res = await fetch(url);
  return res.json();
}

// Pode combinar default + named no mesmo arquivo
export const BASE_URL = 'https://api.exemplo.com';


// ──── app.js ────
// Importar named exports
import { soma, multiplica, PI } from './utils/math.js';

// Importar default export (nome é livre)
import buscarUsuarios from './utils/api.js';

// Importar tudo como namespace
import * as Math from './utils/math.js';
Math.soma(2, 3); // 5

// Renomear ao importar
import { soma as somar } from './utils/math.js';

// Re-exportar (útil em index.js de pasta)
export { soma, multiplica } from './utils/math.js';

produto.js — Classe Produto
// produto.js
export class Produto {
  #preco;

  constructor(nome, preco, categoria) {
    this.nome = nome;
    this.#preco = preco;
    this.categoria = categoria;
    this.id = crypto.randomUUID(); // UUID nativo (Node 19+ / browsers)
  }

  get preco() { return this.#preco; }

  aplicarDesconto(percentual) {
    if (percentual < 0 || percentual > 100) {
      throw new RangeError('Desconto deve ser entre 0 e 100');
    }
    this.#preco = +(this.#preco * (1 - percentual / 100)).toFixed(2);
    return this;
  }

  toJSON() {
    return {
      id: this.id,
      nome: this.nome,
      preco: this.#preco,
      categoria: this.categoria
    };
  }

  toString() {
    return `[${this.categoria}] ${this.nome} — R$${this.#preco}`;
  }
}

estoque.js — Gerenciamento com arrays funcionais
// estoque.js
export class Estoque {
  #produtos = [];

  adicionar(produto) {
    this.#produtos.push(produto);
    return this;
  }

  // Busca com optional chaining
  buscarPorNome(termo) {
    const t = termo.toLowerCase();
    return this.#produtos.filter(p =>
      p.nome.toLowerCase().includes(t)
    );
  }

  porCategoria(cat) {
    return this.#produtos.filter(p =>
      p.categoria.toLowerCase() === cat.toLowerCase()
    );
  }

  // Relatório com reduce
  resumo() {
    return this.#produtos.reduce((acc, p) => {
      const cat = p.categoria;
      acc[cat] ??= { total: 0, quantidade: 0 }; // nullish assignment
      acc[cat].total += p.preco;
      acc[cat].quantidade += 1;
      return acc;
    }, {});
  }

  maisBaratos(n = 5) {
    return [...this.#produtos]
      .toSorted((a, b) => a.preco - b.preco)  // ES2023 — não muta original
      .slice(0, n);
  }

  get total() {
    return this.#produtos.reduce((s, p) => s + p.preco, 0);
  }
}

⚠️ Erros que todo dev comete no começo

1. Mutar arrays dentro do map/filter

Os métodos funcionais como map e filter devem ser puros. Não mute o objeto original dentro deles — crie um novo com spread ou Object.assign.

// ❌ Errado — muta o objeto original
const novos = produtos.map(p => {
  p.preco *= 0.9; // modifica o original!
  return p;
});

// ✅ Correto — cria novo objeto
const novos = produtos.map(p => ({ ...p, preco: p.preco * 0.9 }));
2. sort() muta e ordena como string por padrão

Array.sort() muta o array original E, sem comparador, ordena lexicograficamente (como string). [10, 9, 2].sort() retorna [10, 2, 9] — não o que você espera. 

// ❌ Perigoso — muta o original e ordem errada
[10, 9, 2].sort(); // [10, 2, 9]

// ✅ Correto — comparador numérico + não muta (ES2023)
[10, 9, 2].toSorted((a, b) => a - b); // [2, 9, 10]

// Ou copiar antes de ordenar
[...[10, 9, 2]].sort((a, b) => a - b);
3. this em arrow functions dentro de classes

Arrow functions não têm this próprio — herdam do escopo léxico. Isso é útil em callbacks mas pode confundir em métodos de objetos.

class Timer {
  constructor() { this.segundos = 0; }

  // ❌ Não funciona — this dentro do callback será undefined (strict) ou window
  iniciarErrado() {
    setInterval(function() {
      this.segundos++; // ❌ this errado!
    }, 1000);
  }

  // ✅ Arrow function herda this do método
  iniciar() {
    setInterval(() => {
      this.segundos++; // ✅ this é a instância
    }, 1000);
  }
}
4. Referência vs. valor em objetos e arrays

Objetos e arrays são passados por referência. Copiar com = não cria um novo objeto — cria outro apontador para o mesmo lugar na memória.

const a = { x: 1 };
const b = a;         // b aponta para o MESMO objeto
b.x = 99;
console.log(a.x);    // 99 — a também mudou!

// ✅ Cópia rasa
const c = { ...a };  // ou Object.assign({}, a)
// ✅ Cópia profunda
const d = structuredClone(a); // nativo e seguro
5. Esquecer super() em subclasses

Em uma subclasse ES6, você DEVE chamar super() antes de usar this no constructor. Caso contrário, um ReferenceError é lançado.

class Animal { constructor(nome) { this.nome = nome; } }

class Gato extends Animal {
  constructor(nome, cor) {
    // this.cor = cor; // ❌ ReferenceError: Must call super before accessing 'this'
    super(nome);     // ✅ chamar super primeiro
    this.cor = cor;
  }
}
