// 01 — motivação

Por que isso importa?

Todo programa útil manipula grupos de dados — listas de usuários, registros de vendas, configurações. Python oferece quatro estruturas nativas: list, tuple, dict e set. Escolher a errada não quebra o programa — mas pode torná-lo dez vezes mais lento do que precisa ser.

Dados também precisam persistir em disco. JSON e CSV são os formatos mais comuns de troca entre sistemas. E quando algo dá errado nesse processo, o tratamento de erros com try/except é a linha que separa um script que crasha de um serviço robusto.

💼 Por que um engenheiro sênior usa isso hoje
Toda API retorna JSON. Todo relatório gera CSV. Todo sistema lê configs de arquivos. Sêniors tratam erros corretamente (FileNotFoundError, JSONDecodeError), usam context managers para fechamento seguro, e criam exceções customizadas que comunicam erros de domínio com clareza para quem chamar o código.
// 02 — analogia

Antes do código: uma analogia

🌍 Mundo Real → Tecnologia
Pense numa cozinha profissional. Cada recipiente existe por um motivo.

A bandeja numerada e reordenável é a list. A embalagem lacrada de fábrica que nunca muda é a tuple — imutável por design, mais leve. O fichário indexado por nome onde você busca "molho pesto" diretamente sem folhear é o dict. O cesto de ingredientes únicos onde duplicatas são descartadas automaticamente é o set.

O livro na estante é o arquivo em disco — persiste quando a cozinha fecha. E o chef treinado para imprevistos que sabe o que fazer quando falta ingrediente sem paralisar tudo é o try/except.

// 04 — tratamento de erros

try / except / finally, raise e custom exceptions

Em Python, erros são objetos — instâncias de classes que herdam de BaseException. O bloco try/except captura esses objetos e permite reagir sem encerrar o programa. O finally garante execução de limpeza independente do resultado — com ou sem erro.

// fluxo try / except / else / finally
flowchart TD A([bloco try executa]) --> B{Exceção lançada?} B -- Não --> C[else: roda se sem erro] B -- Sim --> D{except captura?} D -- Sim --> E[except: trata o erro] D -- Não --> F[🔴 propaga para cima] C --> G[finally: SEMPRE executa] E --> G G --> H([programa continua]) style C fill:#0f3020,stroke:#10b981,color:#6ee7b7 style E fill:#3d2f0f,stroke:#f59e0b,color:#fde68a style F fill:#3d0f0f,stroke:#ef4444,color:#fca5a5
📄 erros.pyPython
# ── try / except / else / finally ─────────────────────────────
try:
    valor = int("42")
except ValueError:
    print("Conversão falhou")
else:
    print(f"Conversão ok: {valor}")   # executa SÓ se sem erro
finally:
    print("Sempre executa")           # ideal para limpeza/fechamento

# ── Capturar e inspecionar ────────────────────────────────────
try:
    dados = {"nome": "Ana"}
    print(dados["email"])             # KeyError
except KeyError as e:
    print(f"Chave ausente: {e}")      # 'email'

# ── finally: garantir fechamento de recurso ───────────────────
conexao = None
try:
    conexao = abrir_db()
    dados = conexao.query("SELECT * FROM usuarios")
except Exception as e:
    print(f"Erro: {e}")
finally:
    if conexao:
        conexao.fechar()              # sempre fecha, mesmo com erro

# ── raise: lançar exceção explicitamente ─────────────────────
def sacar(saldo: float, valor: float) -> float:
    if valor <= 0:
        raise ValueError(f"Valor deve ser positivo, recebi {valor}")
    if valor > saldo:
        raise ValueError(f"Saldo insuficiente: {saldo:.2f} < {valor:.2f}")
    return saldo - valor

# Encadear exceções com contexto adicional
try:
    sacar(100, 200)
except ValueError as original:
    raise RuntimeError("Falha na operação bancária") from original

# ── Custom exceptions — hierarquia de domínio ────────────────
class AppError(Exception):
    """Base para todos os erros da aplicação."""
    pass

class ValidationError(AppError):
    def __init__(self, campo: str, mensagem: str):
        self.campo = campo
        self.mensagem = mensagem
        super().__init__(f"[{campo}] {mensagem}")

class NotFoundError(AppError):
    def __init__(self, recurso: str, id):
        super().__init__(f"{recurso} id={id} não encontrado")

# Nunca deixe KeyError/ValueError vazar — envolva em domínio
def buscar_usuario(user_id: int) -> dict:
    if not isinstance(user_id, int) or user_id <= 0:
        raise ValidationError("user_id", "deve ser inteiro positivo")
    usuario = db.get(user_id)
    if not usuario:
        raise NotFoundError("Usuário", user_id)
    return usuario

# Captura hierárquica: mais específico antes do mais geral
try:
    user = buscar_usuario(-1)
except ValidationError as e:
    print(f"Dado inválido: [{e.campo}] {e.mensagem}")
except NotFoundError as e:
    print(f"Não encontrado: {e}")
except AppError as e:
    print(f"Erro genérico: {e}")

// 05 — arquivos

Leitura/escrita, context manager, JSON e CSV

O context manager — with open(...) as f: — é a única forma correta de trabalhar com arquivos. Ele garante fechamento automático ao sair do bloco, mesmo que uma exceção ocorra. Arquivos abertos com f = open(...) solto esgotam file descriptors do sistema em produção se a exceção ocorrer antes do f.close().

📄 arquivos.pyPython
from pathlib import Path
import json, csv

# ── Context manager — sempre use with ────────────────────────
with open("notas.txt", "w", encoding="utf-8") as f:
    f.write("Linha 1\n")
    f.writelines(["Linha 2\n", "Linha 3\n"])

with open("notas.txt", "r", encoding="utf-8") as f:
    conteudo = f.read()               # string completa

with open("notas.txt", "r", encoding="utf-8") as f:
    for linha in f:                   # linha a linha — eficiente
        print(linha.strip())

# Modos: "r" ler · "w" escrever (sobrescreve) · "a" append · "x" novo

# ── pathlib — caminhos seguros ────────────────────────────────
caminho = Path("dados") / "usuarios" / "perfis.json"
caminho.parent.mkdir(parents=True, exist_ok=True)
caminho.exists()      # True/False
caminho.suffix        # ".json"

# Carregar

    dados = json.load(f)              # → lista de dicts

# String ↔ Python (sem arquivo)
obj  = json.loads('{"nome":"Carlos"}')  # string → dict

# ── CSV ───────────────────────────────────────────────────────
cols  = ["nome", "depto", "salario"]

with open("func.csv", "w", newline="", encoding="utf-8") as f:
    w = csv.writer(f)
    w.writerow(cols)

# Ler como lista de dicts (DictReader — mais ergonômico)
with open("func.csv", "r", encoding="utf-8") as f:
    funcionarios = list(csv.DictReader(f))
    # [{"nome": "Ana", "depto": "Eng", "salario": "8500"}, ...]

# ── Erros comuns em arquivo ───────────────────────────────────
def ler_config(caminho: str) -> dict:
    try:
        with open(caminho, "r", encoding="utf-8") as f:
            return json.load(f)
    except FileNotFoundError:
        return {}                     # config ausente → defaults
    except json.JSONDecodeError as e:
        raise ValueError(f"JSON inválido em {caminho}: {e}") from e
🔥 Pro tip — sempre especifique encoding="utf-8"
O encoding padrão do open() é UTF-8 no Linux/macOS e cp1252 no Windows. Um arquivo gerado no Windows sem encoding explícito vai corromper acentos no Linux. Sempre escreva encoding="utf-8".

📄 pipeline.pyPython
import csv, json
from pathlib import Path
from collections import defaultdict


class DadoInvalidoError(Exception):
    pass


def processar_linha(linha: dict, num: int) -> dict:
    try:
        qtd   = int(linha["quantidade"])
        preco = float(linha["preco_unit"])
    except ValueError as e:
        raise DadoInvalidoError(f"Linha {num}: {e}") from e
    if qtd <= 0 or preco <= 0:
        raise DadoInvalidoError(f"Linha {num}: valores devem ser positivos")
    return {**linha, "quantidade": qtd, "preco": preco, "total": round(qtd * preco, 2)}


def ler_vendas(caminho: str) -> tuple[list, list]:
    validas, erros = [], []
    with open(caminho, "r", encoding="utf-8") as f:
        for i, linha in enumerate(csv.DictReader(f), start=2):
            try:
                validas.append(processar_linha(linha, i))
            except DadoInvalidoError as e:
                erros.append(str(e))
    return validas, erros


def analisar(vendas: list) -> dict:
    por_cat  = defaultdict(float)
    por_vend = defaultdict(float)
    for v in vendas:
        por_cat[v["categoria"]]  += v["total"]
        por_vend[v["vendedor"]]  += v["total"]

    top3 = sorted(por_vend.items(), key=lambda x: x[1], reverse=True)[:3]
    return {
        "receita_total":   round(sum(v["total"] for v in vendas), 2),
        "por_categoria":   dict(por_cat),
        "por_vendedor":    dict(por_vend),
        "produtos_unicos": sorted({v["produto"] for v in vendas}),  # set!
        "top_vendedores":  [{"nome": n, "total": t} for n, t in top3],
    }


if __name__ == "__main__":
    try:
        vendas, erros = ler_vendas("vendas.csv")
    except FileNotFoundError:
        print("vendas.csv não encontrado")
        raise SystemExit(1)

    if erros:
        print(f"⚠️  {len(erros)} linha(s) ignorada(s):")
        for e in erros: print(f"   • {e}")

    rel = analisar(vendas)
    rel["linhas_processadas"] = len(vendas)
    rel["linhas_com_erro"]    = len(erros)

    Path("saida").mkdir(exist_ok=True)
    with open("saida/relatorio.json", "w", encoding="utf-8") as f:
        json.dump(rel, f, ensure_ascii=False, indent=2)

    print(f"\n📊 Receita total: R$ {rel['receita_total']:,.2f}")
    print(f"   Processadas: {len(vendas)} · Com erro: {len(erros)}")
    print(f"✅ saida/relatorio.json salvo")

// 07 — armadilhas

Erros que todo mundo comete

⚠️ Capturar Exception genérica e silenciar
except Exception: pass silencia bugs reais junto com erros esperados. Sempre capture a exceção mais específica e nunca use pass sem ao menos um log.
⚠️ Busca linear em list quando dict seria O(1)
next((u for u in usuarios if u["id"] == alvo), None) é O(n). Se você faz isso repetidamente, construa um dict de lookup uma vez: por_id = {u["id"]: u for u in usuarios}. Depois por_id[alvo] é O(1).
⚠️ Abrir arquivo sem context manager
f = open("x.txt") solto: se uma exceção ocorrer antes de f.close(), o arquivo fica aberto indefinidamente. Sempre use with open(...) as f:.
⚠️ Modificar dict/list enquanto itera
for k in d: del d[k] → RuntimeError: dictionary changed size during iteration. Solução: for k in list(d.keys()):

Construa um gerenciador de biblioteca de livros que persiste dados em JSON:

  • Crie um dict indexado por ISBN — cada livro tem título, autor, ano, gêneros (set convertido para list ao salvar) e disponível (bool)
  • Implemente funções para adicionar livro, buscar por ISBN, listar por gênero e marcar como emprestado/devolvido
