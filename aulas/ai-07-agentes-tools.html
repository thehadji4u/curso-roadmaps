Por que isso importa?

LLMs convencionais respondem perguntas. Agentes fazem coisas: buscam informações em tempo real, executam código, chamam APIs externas, tomam decisões iterativas e completam tarefas complexas de múltiplos passos — sem intervenção humana em cada etapa.

Essa é a diferença entre um chatbot e um sistema de IA autônomo. Um engenheiro sênior usa agentes para substituir scripts frágeis por pipelines que raciocinam sobre o que fazer a seguir.

🔍

Pesquisa Autônoma

Agente busca, lê e sintetiza informações de múltiplas fontes sob demanda

💻

Execução de Código

Gera, executa e itera sobre código Python para resolver problemas analíticos

🔗

Integração de APIs

Consulta CRMs, calendários, bancos de dados e serviços externos de forma autônoma

🧠

Memória Persistente

Lembra de preferências, contexto anterior e histórico de interações entre sessões

💡 Chain vs Agente
Uma chain executa passos fixos em ordem definida em tempo de design. Um agente decide dinamicamente quais ferramentas usar, em que ordem, e quantas vezes — com base no raciocínio sobre o problema.
// analogia

Analogia do Mundo Real

🎯 Assistente Executivo vs Secretária de Script
Um sistema de IA tradicional é como uma secretária que segue um roteiro fixo: "recepcione o cliente, anote o nome, transfira para o ramal 201." Qualquer situação fora do roteiro a paralisa.

Um agente de IA é como um assistente executivo sênior: você diz "resolva o problema do cliente" e ele decide por conta própria se precisa ligar, pesquisar, consultar o manual, escalar para outro departamento ou fazer tudo isso em sequência.
🎯 Objetivo do chefe
input do usuário
🛠 Ferramentas (telefone, email, CRM)
Tools (funções, APIs, vector stores)
🧠 Raciocínio do assistente
ReAct loop (Thought → Action → Observation)
📓 Bloco de notas do assistente
Memória (buffer, summary, vector)
⏱ Limite de horas de trabalho
max_iterations / timeout
// conceito central

Function Calling — O LLM chama funções reais

Function Calling (OpenAI) / Tool Use (Anthropic) é o mecanismo pelo qual o LLM decide, com base na conversa, quando e como invocar funções externas que você definiu. O LLM não executa a função — ele gera um JSON estruturado com o nome da função e os argumentos; seu código faz a chamada real e devolve o resultado.

// fluxo de function calling
sequenceDiagram participant U as Usuário participant LLM as LLM (GPT-4o) participant App as Seu Código participant F as Função/API U->>App: "Qual o clima em SP e quanto é 15% de R$890?" App->>LLM: mensagens + definições das tools LLM->>App: tool_call: buscar_clima(cidade="São Paulo") App->>F: executa buscar_clima("São Paulo") F-->>App: {"temp": 28, "cond": "nublado"} App->>LLM: tool_result + tool_call: calcular("0.15 * 890") LLM->>App: tool_call: calcular("0.15 * 890") App->>F: executa calcular("0.15 * 890") F-->>App: {"resultado": 133.5} App->>LLM: tool_result LLM->>U: "São Paulo: 28°C nublado. 15% de R$890 = R$133,50"

Implementação com OpenAI

python — function_calling_openai.py
import json, os
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

# 1. Definir as ferramentas disponíveis
tools = [
    {
        "type": "function",
        "function": {
            "name": "buscar_clima",
            "description": "Retorna o clima atual de uma cidade brasileira",
            "parameters": {
                "type": "object",
                "properties": {
                    "cidade": {
                        "type": "string",
                        "description": "Nome da cidade, ex: São Paulo, Curitiba"
                    },
                    "unidade": {
                        "type": "string",
                        "enum": ["celsius", "fahrenheit"],
                        "description": "Unidade de temperatura (padrão: celsius)"
                    }
                },
                "required": ["cidade"]
            }
        }
    },
    {
        "type": "function",
        "function": {
            "name": "calcular",
            "description": "Calcula expressões matemáticas. Input: expressão como '2 + 2 * 10'",
            "parameters": {
                "type": "object",
                "properties": {
                    "expressao": {"type": "string", "description": "Expressão matemática"}
                },
                "required": ["expressao"]
            }
        }
    }
]

# 2. Implementações reais das funções
def buscar_clima(cidade: str, unidade: str = "celsius") -> dict:
    # Em produção: chame a API OpenWeatherMap, WeatherAPI etc.
    mock = {
        "São Paulo": {"temp": 28, "condicao": "parcialmente nublado", "umidade": 72},
        "Curitiba":  {"temp": 16, "condicao": "chuvoso",              "umidade": 88},
        "Rio de Janeiro": {"temp": 33, "condicao": "ensolarado",     "umidade": 65},
    }
    return mock.get(cidade, {"temp": 25, "condicao": "dados indisponíveis"})

def calcular(expressao: str) -> dict:
    try:
        # ATENÇÃO: use ast.literal_eval ou numexpr em produção
        resultado = eval(expressao, {"__builtins__": {}})
        return {"resultado": resultado, "expressao": expressao}
    except Exception as e:
        return {"erro": str(e)}

# 3. Loop do agente
def executar_agente(pergunta: str) -> str:
    mensagens = [{"role": "user", "content": pergunta}]

    while True:
        resposta = client.chat.completions.create(
            model="gpt-4o",
            messages=mensagens,
            tools=tools,
            tool_choice="auto"   # LLM decide se e qual tool usar
        )

        msg = resposta.choices[0].message

        # Sem tool calls = resposta final em linguagem natural
        if not msg.tool_calls:
            return msg.content

        # Adiciona a mensagem do assistente (com tool_calls) ao contexto
        mensagens.append(msg)

        # Executa cada tool call e adiciona o resultado
        for tc in msg.tool_calls:
            nome = tc.function.name
            args = json.loads(tc.function.arguments)

            if nome == "buscar_clima":
                resultado = buscar_clima(**args)
            elif nome == "calcular":
                resultado = calcular(**args)
            else:
                resultado = {"erro": f"Tool '{nome}' não encontrada"}

            mensagens.append({
                "role": "tool",
                "tool_call_id": tc.id,
                "content": json.dumps(resultado, ensure_ascii=False)
            })

# Uso
print(executar_agente("Qual o clima em São Paulo e Curitiba? E quanto é 15% de R$890?"))

Implementação com Anthropic (Tool Use)

python — tool_use_anthropic.py
import anthropic, json, os

client = anthropic.Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])

tools = [
    {
        "name": "buscar_clima",
        "description": "Retorna o clima atual de uma cidade",
        "input_schema": {
            "type": "object",
            "properties": {
                "cidade": {"type": "string", "description": "Nome da cidade"}
            },
            "required": ["cidade"]
        }
    }
]

mensagens = [{"role": "user", "content": "Como está o clima no Rio de Janeiro?"}]

while True:
    resp = client.messages.create(
        model="claude-opus-4-5",
        max_tokens=1024,
        tools=tools,
        messages=mensagens
    )

    if resp.stop_reason == "end_turn":
        for bloco in resp.content:
            if hasattr(bloco, "text"):
                print(bloco.text)
        break

    if resp.stop_reason == "tool_use":
        mensagens.append({"role": "assistant", "content": resp.content})

        resultados = []
        for bloco in resp.content:
            if bloco.type == "tool_use":
                resultado = buscar_clima(bloco.input["cidade"])
                resultados.append({
                    "type": "tool_result",
                    "tool_use_id": bloco.id,
                    "content": json.dumps(resultado, ensure_ascii=False)
                })

        mensagens.append({"role": "user", "content": resultados})

Padrão ReAct — Reasoning + Acting

ReAct é um loop de raciocínio onde o agente alterna entre pensar (Thought), agir (Action) e observar (Observation) — repetindo até ter confiança suficiente para dar a resposta final. Cada ciclo adiciona informação ao contexto.

// ciclo react
flowchart TD A([💬 Pergunta do usuário]) --> B B[💭 Thought\nAnalisa o que precisa fazer] --> C C[⚡ Action\nEscolhe e invoca uma tool] --> D D[👁 Observation\nRecebe resultado da tool] --> E{Tenho informação\nsuficiente?} E -->|Não — preciso de mais dados| B E -->|Sim| F([✅ Final Answer]) style B fill:#1e1030,stroke:#a855f7,color:#e8e8f0 style C fill:#0f2a1e,stroke:#10b981,color:#e8e8f0 style D fill:#0a1f2a,stroke:#06b6d4,color:#e8e8f0 style F fill:#1e1030,stroke:#a855f7,color:#e8e8f0

Exemplo de trace ReAct para a pergunta "Quem criou Python e há quantos anos?":

💭 Thought 1
Preciso saber quem criou Python. Vou buscar na Wikipedia.
⚡ Action
wikipedia("Python programming language creator")
👁 Observation
"Python foi criado por Guido van Rossum. A primeira versão foi lançada em 1991."
💭 Thought 2
Sei o criador (1991). Agora preciso calcular quantos anos se passaram até 2025.
⚡ Action
calcular("2025 - 1991")
👁 Observation
{"resultado": 34}
✅ Final Answer
Python foi criado por Guido van Rossum e lançado em 1991 — há 34 anos.

Agente ReAct com LangChain

python — react_agent_langchain.py
from langchain.agents import create_react_agent, AgentExecutor
from langchain_openai import ChatOpenAI
from langchain.tools import Tool
from langchain import hub
import wikipedia, os

llm = ChatOpenAI(model="gpt-4o", temperature=0)

# Ferramentas
def buscar_wikipedia(query: str) -> str:
    """Busca definições e fatos na Wikipedia em português."""
    wikipedia.set_lang("pt")
    try:
        return wikipedia.summary(query, sentences=4)
    except wikipedia.exceptions.DisambiguationError as e:
        return f"Ambíguo. Opções: {e.options[:3]}"
    except Exception:
        return "Informação não encontrada."

def calcular(expressao: str) -> str:
    """Calcula expressões matemáticas simples. Ex: '2025 - 1991'"""
    try:
        return str(eval(expressao, {"__builtins__": {}}))
    except Exception as e:
        return f"Erro no cálculo: {e}"

tools = [
    Tool(
        name="wikipedia",
        func=buscar_wikipedia,
        description="Útil para buscar fatos, definições, histórico e informações gerais. "
                    "Input: termo ou frase de busca em português."
    ),
    Tool(
        name="calculadora",
        func=calcular,
        description="Para cálculos matemáticos. Input: expressão como '2025 - 1991' ou '(15/100) * 890'."
    ),
]

# Prompt ReAct padrão (do LangChain Hub)
prompt = hub.pull("hwchase17/react")

agent = create_react_agent(llm, tools, prompt)
executor = AgentExecutor(
    agent=agent,
    tools=tools,
    verbose=True,        # mostra o trace Thought/Action/Observation
    max_iterations=6,    # evita loops infinitos
    handle_parsing_errors=True
)

resposta = executor.invoke({
    "input": "Quem criou Python e em que ano? Quantos anos se passaram desde então até 2025?"
})

print("\n=== RESPOSTA FINAL ===")
print(resposta["output"])

Memória de Agentes

LLMs são stateless por natureza — cada chamada começa do zero. Memória é a camada que preserva contexto entre turnos de conversa ou entre sessões distintas.

Tipo Duração Armazenamento Tokens consumidos Melhor para
Window Buffer Sessão RAM Médio (N últimas msgs) Chat simples, suporte
Summary Memory Sessão RAM Baixo (resumo automático) Conversas longas
Entity Memory Sessão RAM Médio Lembrar pessoas/entidades
VectorDB Memory Persistente Vector Store Variável (busca semântica) Histórico longo e rico
Redis/Postgres Persistente BD externo Baixo Produção multi-usuário

Window Buffer Memory — prática

python — memoria_agente.py
from langchain.memory import ConversationBufferWindowMemory
from langchain.chains import ConversationChain
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o-mini", temperature=0.7)

# Guarda apenas as últimas 5 mensagens (janela deslizante)
memory = ConversationBufferWindowMemory(k=5, return_messages=True)

chain = ConversationChain(llm=llm, memory=memory, verbose=False)

# Conversa com memória
chain.predict(input="Olá! Meu nome é Ana e trabalho com análise de dados.")
chain.predict(input="Estou tentando aprender Python para automação.")
chain.predict(input="Tenho 3 anos de experiência com Excel.")
resposta = chain.predict(input="Com base no que sabe sobre mim, qual curso você recomendaria?")
print(resposta)
# Vai mencionar: Ana, análise de dados, Python, Excel

# Ver o histórico em memória
print("\n--- Histórico ---")
for msg in memory.chat_memory.messages:
    role = "Usuário" if msg.type == "human" else "IA"
    print(f"{role}: {msg.content[:80]}...")

Memória com Vector Store (busca semântica no histórico)

python — memoria_vetorial.py
from langchain.memory import VectorStoreRetrieverMemory
from langchain_community.vectorstores import Chroma
from langchain_openai import OpenAIEmbeddings

# Vector store para guardar memórias importantes
embeddings = OpenAIEmbeddings(model="text-embedding-3-small")
vectorstore = Chroma(
    embedding_function=embeddings,
    persist_directory="./agent_memory_db"
)
retriever = vectorstore.as_retriever(search_kwargs={"k": 3})
memory = VectorStoreRetrieverMemory(retriever=retriever)

# Salvar contexto importante
memory.save_context(
    {"input": "Meu framework favorito é FastAPI"},
    {"output": "Anotei! FastAPI com Pydantic é excelente para APIs REST."}
)
memory.save_context(
    {"input": "Prefiro código limpo com type hints"},
    {"output": "Ótimo hábito! Type hints facilitam manutenção e IDEs."}
)
memory.save_context(
    {"input": "Trabalho num projeto de e-commerce"},
    {"output": "Entendido. E-commerce tem desafios de escalabilidade interessantes."}
)

# Recuperar memórias relevantes semanticamente
contexto = memory.load_memory_variables(
    {"prompt": "Qual tecnologia usar para construir a API do meu projeto?"}
)
print(contexto["history"])
# Vai retornar as memórias sobre FastAPI e e-commerce — as mais relevantes

OpenAI Assistants API

A Assistants API da OpenAI oferece um agente fully managed: você não gerencia o loop, as tools ou a memória manualmente. A API cuida de tudo, persistindo threads de conversa na infraestrutura da OpenAI.

💡 Quando usar Assistants API vs LangChain?
Use Assistants API para MVPs rápidos, quando não precisa de flexibilidade máxima e quer zero infraestrutura. Use LangChain/LlamaIndex quando precisa de controle total sobre tools, memória customizada, vector stores proprietários ou rodar 100% self-hosted.
python — assistants_api.py
from openai import OpenAI
import time, os

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

# Criar assistente (persistido na OpenAI — faça apenas uma vez)
assistente = client.beta.assistants.create(
    name="Tutor de Python",
    instructions="""Você é um tutor especializado em Python e engenharia de software.
    Explique conceitos com exemplos práticos e código executável.
    Sempre responda em português brasileiro.
    Use code_interpreter para demonstrar e testar código quando útil.""",
    model="gpt-4o",
    tools=[
        {"type": "code_interpreter"},  # executa Python real
        {"type": "file_search"}        # busca em arquivos enviados
    ]
)
print(f"Assistente criado: {assistente.id}")

# Criar thread (uma por conversa/usuário)
thread = client.beta.threads.create()

# Adicionar pergunta
client.beta.threads.messages.create(
    thread_id=thread.id,
    role="user",
    content="Mostre-me como calcular a série de Fibonacci com e sem memoização. Compare a performance."
)

# Disparar execução
run = client.beta.threads.runs.create(
    thread_id=thread.id,
    assistant_id=assistente.id
)

# Aguardar conclusão com polling
while run.status in ["queued", "in_progress", "requires_action"]:
    time.sleep(1)
    run = client.beta.threads.runs.retrieve(thread_id=thread.id, run_id=run.id)
    print(f"  Status: {run.status}")

# Exibir resposta
msgs = client.beta.threads.messages.list(thread_id=thread.id, order="desc")
for msg in msgs.data:
    if msg.role == "assistant":
        for bloco in msg.content:
            if bloco.type == "text":
                print(bloco.text.value)
        break

# Continuar a conversa (thread persiste)
client.beta.threads.messages.create(
    thread_id=thread.id,
    role="user",
    content="Agora mostre como fazer o mesmo com generators."
)

// armadilhas

⚠️ Armadilhas Comuns

🔁 Loop infinito sem max_iterations
Se o agente não consegue completar a tarefa, continua tentando ferramentas indefinidamente — consumindo tokens e dinheiro. Sempre defina max_iterations=5 e max_execution_time=30.
📝 Tool description vaga
O LLM escolhe qual tool usar baseado APENAS na description. Descrição ruim = escolha errada. Seja preciso: "quando usar", "formato do input", "o que retorna".
🔑 Memory session_id compartilhado
Sem session_id único por usuário, todos compartilham a mesma memória. Ana vai receber contexto do João. Use session_id=str(user_id) ou session_id=str(uuid4()).
⚡ Ferramentas destrutivas sem confirmação
Nunca exponha tools como deletar_registro(), enviar_email_em_massa() ou dropar_tabela() sem um passo de confirmação humana. Use human-in-the-loop para ações irreversíveis.
💥 Tool levanta Exception
Se sua função Python lança uma exceção, o agente para. Sempre retorne um dict com "erro" em vez de levantar — o agente pode se recuperar e tentar outra abordagem.
