n8n — Integrações Reais

⏱ ~55 min de leitura 🎯 Intermediário 🔧 Hands-on incluído
Google Sheets Telegram Slack Email / Gmail PostgreSQL Notion Airtable OAuth2 webhooks workflow completo
// motivação

Por que isso importa?

Saber criar workflows no n8n é apenas o começo. O verdadeiro poder da ferramenta aparece quando você conecta sistemas reais — Google Sheets que sua equipe já usa, Telegram para notificações instantâneas, bancos de dados onde seus dados vivem, Notion para documentação dinâmica. É aí que a automação sai do papel e economiza horas por semana.

Nesta aula vamos cobrir as 6 categorias de integração mais usadas em produção: planilhas, mensageiros, e-mail, banco de dados, gerenciadores de projetos e ferramentas no-code. Para cada uma, você vai entender a autenticação, os nodes principais, os padrões mais usados e as armadilhas mais comuns.

💼 Por que um engenheiro sênior domina isso
Automação sem integração é script isolado. Engenheiros sêniors constroem pipelines de dados que capturam leads em formulários → salvam no Sheets → notificam no Slack → criam tickets no Notion → disparam e-mails de boas-vindas. Todo esse fluxo automatizado significa menos erros manuais, mais velocidade e dados sempre sincronizados. O n8n é o hub que cola tudo.
// analogia

Antes do código: uma analogia

🌍 Mundo Real → Tecnologia
Pense no n8n como o adaptador universal de viagem. Quando você viaja entre países, cada tomada tem um formato diferente — Brasil usa padrão NBR, EUA usa plug tipo A, Europa usa tipo C. O adaptador universal recebe qualquer plug e entrega energia no formato que o aparelho precisa, sem que o aparelho saiba nada sobre a tomada original.

No mundo digital, cada serviço fala uma linguagem própria: Google Sheets usa OAuth + API REST, Telegram usa Bot Token, PostgreSQL usa TCP com credenciais. O n8n é o adaptador — você conecta qualquer serviço de qualquer lado e ele faz a tradução. Os dados fluem sem que os sistemas precisem conhecer uns aos outros.
🔌 Tomada de energia do país A → API do Google Sheets (OAuth2) 🔌 Tomada do país B → Bot API do Telegram (Token) 🔌 Tomada do país C → PostgreSQL (host + credenciais) 🔌 Adaptador universal → n8n (gerencia todas as auth)

Autenticação: o pré-requisito de tudo

Antes de usar qualquer integração, você precisa configurar as Credentials no n8n. Cada serviço usa um método diferente. Entender esses padrões evita 80% das frustrações com integrações.

Serviço Tipo de Auth O que você precisa Onde obter
Google Sheets / Gmail OAuth2 Client ID + Client Secret Google Cloud Console → APIs & Services
Telegram Bot Token Token do bot @BotFather no Telegram
Slack OAuth2 Bot Token (xoxb-...) api.slack.com → Apps → OAuth
PostgreSQL / MySQL Credenciais Host, Porta, DB, User, Password Seu provedor de banco de dados
Notion API Key Integration Token notion.so/my-integrations
Airtable API Key Personal Access Token airtable.com/create/tokens
SMTP (Email) Credenciais Host SMTP, Porta, User, Password Configurações do provedor de e-mail

Google Sheets — a planilha que vira banco

📊
Google Sheets Node
Auth: OAuth2 — Categoria: Produtividade

O node do Google Sheets é um dos mais usados no n8n. Ele permite ler, escrever, atualizar e deletar dados em planilhas do Google, sem nenhuma linha de código.

Operações principais

Append Row

Adicionar linha

Insere uma nova linha com os dados do item atual. Ideal para capturar respostas de formulários, logs e leads.

Get All Rows

Ler todas as linhas

Retorna todas as linhas da planilha como items n8n. Perfeito para sincronizar dados ou processar listas em loop.

Update Row

Atualizar linha

Atualiza células específicas usando um campo como chave (ex: ID ou e-mail). Usado para marcar status, atualizar timestamps.

Delete Row

Deletar linha

Remove uma linha pelo número ou por uma condição. Use com cuidado — não tem undo nativo no n8n.

Configuração passo a passo

🔥 Como configurar OAuth2 com Google
1. Acesse console.cloud.google.com → crie um projeto
2. Ative a API: Google Sheets API e Google Drive API
3. Em "Credenciais" → Criar Credenciais → OAuth 2.0 Client ID
4. Tipo: Aplicativo da Web → adicione https://SEU-N8N/rest/oauth2-credential/callback como URI de redirecionamento
5. No n8n: Credentials → Google Sheets OAuth2 → cole Client ID e Client Secret → clique em "Sign in with Google"
exemplo: estrutura de dados para Append Row
// No node Google Sheets → Append Row:
// Mapeie os campos da planilha com expressões n8n

Telegram & Slack — notificações em tempo real

✈️
Telegram Node
Auth: Bot Token — Categoria: Comunicação

O Telegram é a escolha número 1 para notificações de automação por ser simples, gratuito e ter bots sem custo. Em 2 minutos você cria um bot e começa a receber alertas de qualquer workflow.

Criar um bot Telegram

1
Fale com o @BotFather
No Telegram, busque @BotFather → envie /newbot → escolha um nome e username para o bot
2
Guarde o token
O BotFather vai te enviar um token no formato 1234567890:ABCdefGHIjklMNOpqrsTUVwxyz — guarde-o
3
Obtenha seu Chat ID
Envie uma mensagem para o seu bot. Depois acesse: https://api.telegram.org/botSEU_TOKEN/getUpdates — o chat.id aparece no JSON de resposta
4
Configure no n8n
Credentials → Telegram API → cole o token → no node Telegram: operação "Send Message", Chat ID e o texto com expressões
exemplo: mensagem formatada com HTML no Telegram
// No node Telegram → Send Message:
// Parse Mode: HTML (suporta bold, italic, links)

// Chat ID: {{ $env.TELEGRAM_CHAT_ID }}

👤 Nome: {{ $json.nome }}

🕐 Data: {{ $now.toFormat('dd/MM/yyyy HH:mm') }}

Slack — alternativa corporativa

Para ambientes corporativos, o Slack é mais comum. A configuração usa OAuth2 com um Bot Token. O padrão de uso é similar ao Telegram, mas com recursos adicionais como blocks para mensagens formatadas.

🔥 Slack Blocks vs. Texto simples
Para notificações simples, texto funciona. Para dashboards e alertas bonitos, use Slack Block Kit Builder (app.slack.com/block-kit-builder) para criar templates visuais e cole o JSON no campo "Blocks" do node Slack.

Email — SMTP e Gmail

Envio de e-mail é uma das automações mais pedidas. O n8n suporta dois nodes principais: Send Email (SMTP genérico) e Gmail (OAuth2 com funcionalidades extras como ler, mover e rotular e-mails).

Send Email (SMTP)

SMTP Genérico

Funciona com qualquer provedor: Gmail, Outlook, SendGrid, Mailgun, Zoho. Configure host, porta (587/465), TLS e credenciais.

Gmail Node

Gmail via OAuth2

Além de enviar, permite ler e-mails, buscar por label, mover para pastas e criar drafts. Ideal para automações de inbox.

Configuração SMTP para Gmail

⚠️ Gmail exige App Password
O Google bloqueou o uso de senha normal para SMTP. Para usar SMTP com Gmail você precisa de uma App Password:
1. Ative 2-Step Verification na conta Google
2. Acesse: myaccount.google.com → Security → App Passwords
3. Gere uma senha específica para o n8n
4. No n8n: Use smtp.gmail.com, porta 587, TLS: STARTTLS, e a App Password como senha

Banco de Dados — PostgreSQL, MySQL e MongoDB

Quando os dados precisam persistir de forma estruturada — com relações, consultas complexas e transações — o caminho é o banco de dados. O n8n tem nodes nativos para PostgreSQL, MySQL, MariaDB, SQLite e MongoDB.

🗄️
PostgreSQL Node
Auth: Host + Porta + Credenciais — O mais usado em produção
OperaçãoDescriçãoExemplo de uso
Execute Query Roda SQL puro — SELECTs complexos, JOINs, agregações Relatórios, dashboards, auditorias
Insert Insere um ou vários registros via mapeamento visual Salvar leads, logs, eventos
Update Atualiza registros usando uma coluna como chave Mudar status, atualizar campos
Delete Remove registros por condição Limpeza de dados expirados
exemplo: insert + query encadeados no workflow
-- Node 1: Execute Query — verificar se lead já existe
SELECT id, email, created_at
FROM leads
WHERE email = '{{ $json.email }}'
LIMIT 1;

-- Node 2 (após IF verificar se veio resultado):
-- Se não existe → Insert
INSERT INTO leads (nome, email, telefone, origem, created_at)
VALUES (
  '{{ $json.nome }}',
  '{{ $json.email }}',
  '{{ $json.telefone }}',
  '{{ $json.utm_source }}',
  NOW()
)
RETURNING id;

-- Se já existe → Update
UPDATE leads
SET ultimo_contato = NOW(),
    total_visitas = total_visitas + 1
WHERE email = '{{ $json.email }}';

Notion & Airtable — gerenciar projetos com automação

Notion

Base de conhecimento viva

Crie páginas, database entries e blocos automaticamente. Ideal para logs de auditoria, documentação gerada por IA e kanban populado por workflows.

Airtable

Planilha + banco híbrido

Insira e atualize records em Bases do Airtable. As views do Airtable (Grid, Gallery, Kanban) ficam automaticamente atualizadas com os dados dos seus workflows.

Notion: configurar a integração

1
Criar Integration no Notion
Acesse notion.so/my-integrations → New Integration → dê um nome → copie o Internal Integration Token
2
Conectar ao Database
Abra o Database no Notion → clique nos 3 pontinhos (⋯) → Add connections → selecione sua integration
3
Pegar o Database ID
A URL do database tem o formato notion.so/WORKSPACE/DATABASE_ID?v=VIEW_ID — copie o DATABASE_ID (32 chars)
exemplo: criar página no Notion via n8n
// Node Notion → Create → Database Page

Mãos na massa: Workflow de Lead Capture completo

Vamos construir um workflow real e completo: um formulário web captura um lead → salva no Google Sheets → notifica no Telegram → envia e-mail de boas-vindas → cria entrada no Notion. Tudo automático.

1
Criar o Trigger — Webhook
Node: Webhook → HTTP Method: POST → Path: /lead-capture → copie a URL de produção
Testar com cURL
$ curl -X POST https://SEU-N8N/webhook/lead-capture \
  -H "Content-Type: application/json" \
  -d '{"nome":"João Silva","email":"joao@email.com","telefone":"11999999999","utm_source":"google"}'
2
Validar dados — IF Node
Adicione um IF para garantir que o e-mail está presente: {{ $json.email }} is not empty. Caso inválido → retorne erro 400 com "Respond to Webhook".
3
Salvar no Google Sheets
Node: Google Sheets → Append Row → selecione sua planilha → mapeie os campos (Nome, Email, Telefone, Data, Origem)
4
Notificar no Telegram
Node: Telegram → Send Message → Chat ID do seu grupo/canal → mensagem formatada com os dados do lead
5
Enviar e-mail de boas-vindas
Node: Send Email → From: seu@email.com → To: {{ $json.email }} → Subject: "Bem-vindo(a), {{ $json.nome }}!" → HTML Body com template
6
Criar página no Notion
Node: Notion → Create → Database Page → mapeie as propriedades → retorne 200 OK com "Respond to Webhook"

Erros comuns nas integrações

⚠️ Rate Limits — a integração para de funcionar aleatoriamente

Google Sheets tem limite de 100 requests/100 segundos. Telegram tem limite de 30 mensagens/segundo. Se você processar uma lista grande em loop, vai estourar o rate limit.

✅ Solução: adicione um Wait node entre as iterações (1–2 segundos) ou use o modo "Batch" disponível em alguns nodes.

⚠️ Credencial OAuth2 expira — workflow para às 3h da manhã

Tokens OAuth2 do Google expiram. O n8n renova automaticamente usando o refresh token, mas se a credencial foi criada sem o scope correto, a renovação falha silenciosamente.

✅ Solução: ao criar a credencial Google, certifique-se de que o consent screen inclui os scopes necessários. Adicione um Error Trigger no workflow para ser avisado por Telegram quando algo falhar.

⚠️ Tipos de dados — números chegam como string

Webhooks e Google Sheets frequentemente retornam números como strings. Uma operação matemática como {{ $json.preco * 1.1 }} pode resultar em "10.00...1.1" (concatenação) em vez de 11.

✅ Solução: converta explicitamente: {{ Number($json.preco) * 1.1 }} ou use o Code Node para transformar os tipos antes de operar.

⚠️ Notion: propriedades com tipos errados causam 400

O Notion é estrito com tipos. Se você tentar escrever texto em um campo de número ou uma string onde espera-se um objeto select, a API retorna 400.

✅ Solução: use sempre a estrutura correta da API do Notion para cada tipo de propriedade (title, rich_text, select, date, email, number). Teste no Notion API Explorer antes de integrar.

🏁 Checkpoint — valide seu aprendizado

Construa um sistema de monitoramento de formulário de contato completo em n8n:

  • Trigger via Webhook recebendo nome, email, mensagem e assunto
  • Validar que email e mensagem não estão vazios (IF Node)
  • Salvar no Google Sheets com timestamp automático
  • Enviar notificação formatada no Telegram com todos os dados
  • Responder ao webhook com JSON {"status": "ok", "id": "{{ $json.id }}"}
  • Adicionar Error Trigger que avisa no Telegram se qualquer node falhar
💬 Dica: use o node "Set" para preparar os dados antes de enviar para múltiplos destinos — isso evita repetir expressões e torna o workflow mais legível.
