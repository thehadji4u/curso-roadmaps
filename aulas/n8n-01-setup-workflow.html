// MOTIVAÇÃO

Por que isso importa?

Quantas horas por semana você gasta em tarefas repetitivas? Copiar dados de uma planilha para um sistema, enviar e-mails de relatório toda segunda-feira, monitorar uma API e notificar o time quando algo muda, sincronizar contatos entre CRMs? Cada uma dessas tarefas é um workflow esperando para ser automatizado — e a maioria delas não exige uma linha de código.

O n8n (nodemation) é uma plataforma de automação open-source, self-hosted e com interface visual que conecta centenas de serviços e APIs em fluxos de trabalho. Diferente de Zapier ou Make, você pode rodar na sua própria infraestrutura, ter controle total dos dados e estender com JavaScript quando precisar de lógica customizada.

💼 Por que um engenheiro usa n8n (e não só código)
n8n não substitui código — ele complementa. Para integrações entre APIs conhecidas (Slack + Google Sheets + GitHub + Notion), construir em n8n é 10x mais rápido que escrever código do zero. Para automações de negócio que mudam frequentemente, uma interface visual é mais fácil de manter. Engenheiros seniores sabem quando usar a ferramenta certa: n8n para glue logic entre serviços, código para lógica de domínio complexa.
// ANALOGIA

Antes do código: uma analogia

🌍 Mundo Real → Tecnologia
Pense num encanamento inteligente da sua casa. A água (dados) entra por diferentes fontes — torneira da cozinha, chuveiro, jardim. Cada cômodo tem registros e válvulas (lógica) que controlam o fluxo. Alguns canos levam a um filtro (transformação), outros para o aquecedor (processamento), e finalmente a água chega onde precisa — pia, máquina de lavar, jardim — no formato certo.

O n8n é esse sistema de encanamento para dados. Os triggers são as torneiras que iniciam o fluxo. Os nodes são as válvulas, filtros e transformadores que processam os dados ao longo do caminho. As connections são os canos que ligam tudo. E o workflow completo garante que a informação chegue ao destino certo, no formato certo, na hora certa — automaticamente, sem você ter que abrir nenhuma torneira manualmente.
🚿 Torneira que inicia o fluxo → Trigger node (Webhook, Schedule, etc.) 🔧 Válvula que controla o fluxo → IF / Switch node (lógica condicional) 🔄 Filtro que transforma a água → Set / Code node (transformar dados) 🚰 Cano que liga dois pontos → Connection entre nodes 🏁 Destino final da água → Action node final (Slack, Email, DB) 📋 Projeto do encanamento completo → Workflow salvo no n8n

Estrutura de dados — items e JSON

Todo node no n8n trabalha com uma lista de items. Cada item é um objeto JSON com dois campos: json (os dados) e opcionalmente binary (arquivos). Quando um node produz múltiplos items (ex: busca 10 registros no banco), o próximo node processa cada item individualmente por padrão.

Estrutura de um item no n8n
// O que flui entre os nodes é sempre um array de items:
[
  {
    "json": {
      "id": 42,
      "title": "Fix bug in auth module",
      "author": "maria.dev",
      "labels": ["bug", "priority-high"],
      "created_at": "2024-01-15T10:30:00Z"
    }
  },
  {
    "json": {
      "id": 43,
      "title": "Add unit tests for payment service",
      "author": "joao.eng",
      "labels": ["enhancement"],
      "created_at": "2024-01-15T11:00:00Z"
    }
  }
]
💡 Expressions — acessando dados entre nodes
Para usar dados do node anterior, use a sintaxe de expression com duplas chaves: {{ $json.title }} acessa o campo title do item atual. {{ $node["GitHub Trigger"].json.author }} acessa dados de um node específico pelo nome. Você pode usar qualquer expressão JavaScript válida dentro das chaves.
// TRIGGERS

Tipos de Triggers — como um workflow começa

Todo workflow precisa de um ponto de entrada. Os triggers definem quando e por que o workflow é iniciado. Escolher o trigger certo é a primeira decisão ao criar qualquer automação.

⏰
Schedule Trigger
Executa o workflow em horários definidos via cron ou intervalos. Não precisa de evento externo.
Exemplos: relatório todo dia às 9h, limpeza semanal, sync horária de dados
🌐
Webhook Trigger
Expõe uma URL HTTP que dispara o workflow quando recebe uma requisição POST/GET. Trigger mais versátil.
Exemplos: receber eventos de GitHub, Stripe, formulários web, qualquer serviço com webhook
📧
Email Trigger (IMAP)
Monitora uma caixa de e-mail e dispara quando novos e-mails chegam, com filtros por assunto ou remetente.
Exemplos: processar pedidos por email, alertas de monitoramento, suporte ao cliente
🔔
App Triggers nativos
Triggers específicos de cada integração — GitHub, Slack, Notion, Airtable. Gerenciam polling ou webhooks automaticamente.
Exemplos: novo PR no GitHub, nova linha no Airtable, mensagem em canal do Slack
▶️
Manual Trigger
Disparado clicando em "Execute workflow" na interface. Essencial para desenvolvimento e testes.
Usos: testar workflows em desenvolvimento, executar ações únicas sob demanda
🔗
Execute Workflow Trigger
Permite que outro workflow chame este como sub-rotina, passando dados como parâmetros.
Modularizar automações grandes em sub-workflows reutilizáveis
🔥 Webhook Trigger é o mais poderoso
Com o Webhook Trigger, você expõe uma URL única no formato https://seu-n8n.com/webhook/uuid. Qualquer serviço que suporta webhooks (GitHub, Stripe, Typeform, HubSpot, praticamente tudo) pode chamar essa URL e disparar seu workflow com os dados do evento. É a cola universal entre qualquer SaaS e sua automação.

Instalando o n8n — 3 formas

🐳 Docker

Recomendado para dev

Um comando, zero configuração, fácil de atualizar e remover. Persistência via volume nomeado.

📦 npm

Global no sistema

Instala como pacote global Node.js. Boa para uso local sem Docker. Precisa de Node 18+.

☁️ n8n Cloud

Sem infraestrutura

Versão gerenciada pelo próprio n8n. 14 dias grátis. Boa para validar antes de self-host.

Instalação via Docker (recomendada)

Terminal
# Forma simples — roda e abre no browser
$ docker run -it --rm \
--name n8n \
-p 5678:5678 \
-v n8n_data:/home/node/.n8n \
docker.n8n.io/n8nio/n8n

info: Editor is now accessible via: http://localhost:5678/
info: Initializing n8n process...

Instalação via Docker Compose (persistente e configurável)

docker-compose.yml para n8n self-hosted
services:
  n8n:
    image: docker.n8n.io/n8nio/n8n:latest
    restart: unless-stopped
    ports:
      - "5678:5678"
    environment:
      # URL base (importante para webhooks funcionarem)
      - N8N_HOST=localhost
      - N8N_PORT=5678
      - N8N_PROTOCOL=http
      - WEBHOOK_URL=http://localhost:5678/

      # Autenticação básica (habilite em produção!)
      - N8N_BASIC_AUTH_ACTIVE=true
      - N8N_BASIC_AUTH_USER=admin
      - N8N_BASIC_AUTH_PASSWORD=senha_segura

      # Timezone
      - GENERIC_TIMEZONE=America/Sao_Paulo

      # Executores (important para performance)
      - EXECUTIONS_PROCESS=main

    volumes:
      - n8n_data:/home/node/.n8n

volumes:
  n8n_data:

Passo a Passo — Construindo o Primeiro Workflow

Vamos construir um workflow real: toda hora, buscar uma frase motivacional de uma API pública e enviar para um webhook de teste. Você vai aprender Schedule Trigger, HTTP Request node, Set node e como visualizar os dados em cada etapa.

1
Subir o n8n e criar um novo workflow
Terminal
$ docker run -it --rm -p 5678:5678 -v n8n_data:/home/node/.n8n docker.n8n.io/n8nio/n8n
Editor disponível em: http://localhost:5678/

Acesse http://localhost:5678, complete o cadastro inicial e clique em "+ New Workflow". Renomeie para "Frases Motivacionais".

2
Adicionar o Schedule Trigger
Clique no "+" no centro do canvas → busque "Schedule" → selecione "Schedule Trigger". Configure:
  • ✦ Trigger Interval: Hours
  • ✦ Hours Between Triggers: 1

Para testar agora sem esperar 1 hora, clique em "Execute Node" para disparar manualmente.

3
Adicionar o HTTP Request node
Clique no "+" na saída do Schedule Trigger → busque "HTTP Request". Configure:
  • ✦ Method: GET
  • ✦ URL: https://zenquotes.io/api/random
  • ✦ Response Format: JSON

Clique em "Execute Node". Você verá o Output com a frase retornada pela API no painel de dados.

Adicionar o Set node para formatar os dados
Clique no "+" na saída do HTTP Request → busque "Set". Configure os campos:

Essas expressões são avaliadas dinamicamente. O $json se refere ao output do node imediatamente anterior. O $now é uma variável built-in do n8n para a data/hora atual.

// EXPRESSÕES

Expressões — a linguagem do n8n

Expressões permitem referenciar dados dinâmicos de nodes anteriores, usar funções JavaScript e acessar variáveis do sistema. Elas são escritas dentro de {{ }} em qualquer campo de texto do n8n.

Expressões mais usadas — referência rápida
// ── Acessar dados do node anterior ──────────────────────────────
{{ $json.campo }}                    // Campo do item atual
{{ $json.usuario.nome }}             // Campo aninhado
{{ $json.tags[0] }}                  // Primeiro item de array

// ── Referenciar node específico pelo nome ──────────────────────
{{ $node["Nome do Node"].json.campo }}
{{ $("HTTP Request").item.json.status }}   // Sintaxe alternativa

// ── Variáveis built-in ─────────────────────────────────────────
{{ $now }}                           // DateTime atual (objeto Luxon)
{{ $now.toISO() }}                   // "2024-01-15T10:30:00.000Z"
{{ $now.toFormat('dd/MM/yyyy') }}    // "15/01/2024"
{{ $workflow.id }}                   // ID do workflow
{{ $execution.id }}                  // ID da execução atual
{{ $itemIndex }}                     // Índice do item atual no loop

// ── JavaScript dentro das expressões ──────────────────────────
{{ $json.nome.toUpperCase() }}
{{ $json.valor * 1.1 }}              // Cálculos
{{ $json.email.includes('@') ? 'válido' : 'inválido' }}
{{ $json.tags.join(', ') }}          // Array para string
{{ JSON.stringify($json) }}          // Converter para string JSON

// ── Acessar dados de múltiplos items ──────────────────────────
{{ $items().length }}                // Total de items na execução
{{ $items("HTTP Request").length }}  // Items de um node específico
✅ n8n com expressões
// No campo "Message" do Slack node:
"🚀 Novo PR de {{ $json.user.login }}: 
{{ $json.pull_request.title }}
Branch: {{ $json.pull_request.head.ref }} → {{ $json.pull_request.base.ref }}
Link: {{ $json.pull_request.html_url }}"
⚙️ Equivalente em código Python
message = (
    f"🚀 Novo PR de {data['user']['login']}: "
    f"{data['pull_request']['title']}\n"
    f"Branch: {data['pull_request']['head']['ref']} "
    f"→ {data['pull_request']['base']['ref']}\n"
    f"Link: {data['pull_request']['html_url']}"
)
slack_client.chat_postMessage(
    channel="#dev", text=message
)

⚠️ Armadilhas Comuns

1. WEBHOOK_URL errada — webhooks não funcionam externamente

A variável WEBHOOK_URL define o endereço base das URLs de webhook geradas pelo n8n. Se você configurar como localhost, os webhooks só funcionam de dentro do mesmo servidor. Para receber eventos externos (GitHub, Stripe), configure com o domínio público do seu servidor: WEBHOOK_URL=https://n8n.seudominio.com/. Para desenvolvimento local, use ngrok para expor o localhost.

2. Dados binários passando por Set node — perda de arquivos

O Set node por padrão não preserva dados binários (arquivos, imagens) do node anterior. Se o seu workflow manipula arquivos, ative a opção "Keep Only Set" como falso, ou use o campo binary explicitamente. Nodes que trabalham com binários têm configurações específicas para isso.

3. Expressions com erro silencioso retornam string vazia

Se uma expression como {{ $json.campo_que_nao_existe }} falha, o n8n retorna string vazia por padrão — sem erro, sem aviso. Isso pode causar dados incorretos passando silenciosamente pelo workflow. Use o modo de debug (clique no node após executar) para verificar sempre os dados de saída.

4. Execuções acumulando sem limpeza — disco lotando

Por padrão, n8n salva o histórico de todas as execuções com todos os dados de entrada/saída. Em workflows que rodam frequentemente (a cada minuto, por exemplo), isso pode consumir gigabytes em semanas. Configure EXECUTIONS_DATA_SAVE_ON_SUCCESS=none ou EXECUTIONS_DATA_MAX_AGE=168 (horas) para limitar a retenção.

5. Credenciais compartilhadas entre workflows — risco de segurança

As credenciais criadas em um workflow ficam disponíveis para todos os workflows da instância n8n. Em ambientes compartilhados (múltiplos usuários), use o n8n com múltiplos workspaces ou instâncias separadas. Revise quem tem acesso à sua instância n8n — quem acessa o editor tem acesso implícito a todas as credenciais configuradas.

// GLOSSÁRIO

Termos desta Aula

Workflow
Conjunto de nodes conectados que define uma automação completa. Pode estar ativo (rodando automaticamente) ou inativo (apenas editável). Salvo como JSON internamente.
Node
Bloco funcional individual que realiza uma operação — buscar dados, transformar, enviar para serviço externo, tomar uma decisão. Cada node tem inputs e outputs.
Trigger Node
Node especial que inicia o workflow. Todo workflow tem exatamente um trigger. Define quando e com quais dados o workflow começa a executar.
Item
Unidade de dados que flui entre nodes. Um array de items. Cada item tem um campo json com os dados e opcionalmente binary para arquivos.
Expression
Sintaxe {{ }} usada em campos de texto para referenciar dados dinâmicos — do node anterior ($json), de nodes específicos ($node), ou variáveis do sistema ($now, $workflow).
Credentials
Configurações de autenticação para APIs externas (tokens, OAuth, usuário/senha). Armazenadas de forma criptografada no banco do n8n e reutilizáveis entre workflows.
Execution
Uma instância de execução do workflow — do disparo do trigger até o último node. Cada execução tem ID único, status (success/error) e log dos dados em cada node.
Canvas
Área de trabalho visual onde nodes são posicionados e conectados com setas. Os nodes são arrastáveis e as conexões são desenhadas clicando no ponto de saída de um node e arrastando para a entrada do próximo.
self-hosted
Modalidade de rodar o n8n na sua própria infraestrutura (servidor, VPS, Docker local) em vez de usar o serviço cloud gerenciado. Gratuito, com controle total dos dados e sem limites de execução.

  • Workflow 1 — Schedule + API: A cada 30 minutos, busque a cotação do dólar em uma API pública (ex: https://economia.awesomeapi.com.br/json/last/USD-BRL), extraia o valor com o Set node e envie para um webhook de teste (webhook.site)
  • Workflow 2 — Webhook: Crie um endpoint POST /contato que recebe um JSON com nome, email e mensagem. Use o IF node para validar se o email contém "@". Na saída true, responda com {"status": "ok", "recebido": true}. Na saída false, responda com {"status": "erro", "motivo": "email inválido"}
  • Verifique o histórico em Executions e inspecione os dados de saída de cada node
  • Bônus: Adicione um Code node após o Set node do Workflow 1 que calcula o preço de 100 dólares em reais usando $json.bid
💬 Dica: para testar o Workflow 2 sem sair do terminal, use curl -X POST http://localhost:5678/webhook-test/contato -H "Content-Type: application/json" -d '{"nome":"João","email":"joao@test.com","mensagem":"Olá"}'. A URL de teste (webhook-test) só funciona enquanto você está com o editor aberto no node com o botão "Listening".
